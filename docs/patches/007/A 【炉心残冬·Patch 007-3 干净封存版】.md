## 2. 承诺任务定义  
承诺任务是玩家在事件中选择“我会在期限内解决”的结果。  
承诺任务不是立即生效按钮。 承诺任务是限时目标。  
示例：  
```
承诺补足口粮
承诺扩容医疗
承诺处理遗体
承诺安置儿童
承诺降低旧城派人数
承诺恢复信任
承诺降低恐慌
承诺补足煤炭
承诺降低炉心压力

```
承诺任务的核心是：  
```
玩家获得一个处理窗口。
系统记录截止日和目标。
到期后检查目标是否达成。
成功则奖励。
失败则惩罚。

```
   
⸻  
   
## 3. 承诺任务不是持续增益  
承诺任务本身不提供持续效果。  
规则：  
```
承诺任务不会每天提高信任。
承诺任务不会每天降低恐慌。
承诺任务不会暂停原本的日结惩罚。
承诺任务不会自动生成资源、住房、医疗容量或煤炭。
承诺任务不会自动修复旧城派、医疗、食物、住房或炉心压力问题。

```
说明：  
```
承诺只是“给我一点时间”。
不是“我说了会修，所以问题现在先自己变好”。

```
   
⸻  
   
## 4. 承诺任务数据结构  
建议字段：  
```
promise_id: string
promise_type: string
promise_source_event: string
promise_created_day: int
promise_deadline_day: int
promise_target: dict
promise_success_effect: dict
promise_fail_effect: dict
promise_status: active / success / failed / expired

```
字段说明：  
```
promise_id：承诺唯一 ID。
promise_type：承诺类型，如 food / medical / housing / old_city / trust / panic。
promise_source_event：来源事件。
promise_created_day：承诺开启日。
promise_deadline_day：最后可完成日。
promise_target：目标条件。
promise_success_effect：成功效果。
promise_fail_effect：失败效果。
promise_status：当前状态。

```
   
⸻  
   
## 5. 承诺任务结算顺序  
承诺任务统一在新一天开始时检查。  
顺序：  
```
新一天开始
→ 检查 active 承诺任务
→ 若目标已达成，结算成功
→ 若 current_day > promise_deadline_day 且目标未达成，结算失败
→ 移除已结算承诺
→ 释放承诺槽
→ 再检查当天事件触发
→ 再显示当天面板与事件

```
禁止：  
```
在 end_day 时结算一次承诺。
新一天开始再结算一次承诺。
承诺到期后同时触发多次成功 / 失败。
承诺失败当天连环弹出多个升级事件。

```
说明：  
```
承诺任务是日切时检查的限时目标，不是 end_day 中途偷袭玩家的暗刀。

```
   
⸻  
   
## 6. 承诺截止日含义  
promise_deadline_day 表示最后一个仍可完成目标的日期。  
规则：  
```
玩家可以在 promise_deadline_day 当天进行操作。
promise_deadline_day 当天 end_day 后，系统进入下一天。
下一天开始时，按上一日日结后的最终状态检查承诺是否完成。

```
示例：  
```
第 20 天开启承诺。
promise_deadline_day = 23。

玩家可在第 20、21、22、23 天完成目标。
第 23 天 end_day 后完成日结。
第 24 天开始时检查承诺目标。

```
说明：  
```
避免“截止日当天还没结束就被判失败”的偷袭感。

```
   
⸻  
   
## 7. 承诺任务可提前完成  
承诺任务不必等到截止日才结算成功。  
规则：  
```
新一天开始时，检查所有 active 承诺任务。
若目标已经达成，则立即结算成功。
成功后从 active 承诺列表移除。
成功后释放承诺槽。
若目标未达成，且 current_day > promise_deadline_day，则结算失败。

```
说明：  
```
玩家提前完成承诺后，不应继续占用承诺槽直到截止日。

```
   
⸻  
   
## 8. 承诺任务上限  
全局承诺任务上限：  
```
同一时间最多存在 2 个 active 承诺任务。

```
同类承诺上限：  
```
同一时间最多存在 1 个同类承诺任务。

```
示例：  
```
已有“承诺补足口粮”。
则不能再开启第二个食物类承诺。

```
承诺槽满时：  
```
承诺类选项不可用。
系统显示：当前承诺任务已满。
不可用原因必须显示。
不可用选项不得进入正式 A / B / C 编号。

```
玩家强行输入不可用承诺选项时：  
```
返回失败原因。
不改变状态。
不生成承诺。
不结算效果。
不结算代价。

```
   
⸻  
   
## 9. 承诺任务不能无代价取消  
V1 默认不提供“取消承诺”按钮。  
规则：  
```
承诺任务开启后，不能无代价取消。
若后续需要提供取消功能，则取消视为主动放弃，并立即按失败结算。

```
说明：  
```
避免玩家在快失败时取消承诺来逃避惩罚。
守炉人不能把锅塞进雪里假装没许过。

```
   
⸻  
   
## 10. 承诺任务类型  
V1 承诺任务分为以下类型：  
```
food：食物承诺
medical：医疗承诺
housing：住房承诺
body：遗体处理承诺
children：儿童安置承诺
labor：工时 / 过劳承诺
coal：煤炭承诺
furnace：炉心压力承诺
old_city：旧城派承诺
trust：信任承诺
panic：恐慌承诺

```
说明：  
```
同一类型不能重复挂多个承诺，防止玩家把所有危机都塞进“以后再说”箱子里。

```
   
⸻  
   
## 11. 承诺任务常见期限  
默认期限方向：  
```
轻度承诺：2～3 天
中度承诺：3～5 天
重度承诺：5～7 天
终局前承诺：按剩余天数缩短

```
示例方向：  
```
补足口粮：3 天
扩容医疗：4 天
补足住房：4～5 天
处理遗体：3 天
安置儿童：4 天
降低旧城派人数：5 天
补足煤炭：3～5 天
恢复信任：4 天
降低恐慌：4 天

```
说明：  
```
上述期限是方向 / 示例。
具体天数由后续数值表确认。
未封存前不得由代码窗猜测为最终数值。

```
   
⸻  
   
## 12. 终局前承诺限制  
第七霜落临近时，承诺任务不得生成超出游戏剩余时间的截止日。  
规则：  
```
若承诺期限会超过第 48 天，则自动缩短到第 48 天。
第 49 天进入第七霜落后，不再开启新的普通承诺任务。

```
说明：  
```
第七霜落阶段是终局验收，不是继续开长期承诺任务的阶段。

```
   
⸻  
   
## 13. 承诺目标写法  
承诺目标必须是可检查的状态，不写成纯文案。  
示例：  
```
food 储备达到 X。
熟食储备达到 X。
医疗容量 ≥ 当前病患人数。
无住所人数 = 0。
未处理遗体数量 = 0。
旧城派人数 ≤ X。
信任 ≥ X。
恐慌 ≤ X。
煤炭储备 ≥ X。
炉心压力 ≤ X。

```
禁止：  
```
承诺让居民满意。
承诺改善城市。
承诺尽快处理。
承诺想办法。

```
说明：  
```
承诺目标必须能被代码窗判断成功或失败。

```
   
⸻  
   
## 14. 承诺目标显示规则  
玩家接受承诺前，必须看到：  
```
承诺目标
截止日
成功方向
失败方向
当前已有承诺数量
同类承诺是否已存在

```
显示示例：  
```
承诺补足口粮：
目标：第 23 天结束前，让熟食储备达到安全线。
截止：第 23 天。
成功：信任上升，恐慌下降。
失败：信任下降，恐慌上升。

```
   
⸻  
   
## 15. 承诺成功规则  
承诺成功可以：  
```
提高信任。
降低恐慌。
降低旧城派人数。
降低对应风险。
终止对应承诺链。
移除 active 承诺。
释放承诺槽。

```
承诺成功不得：  
```
凭空生成大量资源。
直接删除旧城派系统。
直接清空所有危机。
直接改变第七霜落日期。

```
   
⸻  
   
## 16. 承诺失败规则  
承诺失败可以：  
```
降低信任。
提高恐慌。
提高旧城派人数。
提高危机风险。
推动事件链进入下一阶段。
让对应问题更容易再次触发。

```
承诺失败不得：  
```
直接秒杀玩家。
直接造成大量死亡。
直接触发旧城派最终出走。
直接扣除出走人口。
直接提前第七霜落。
直接触发炉心爆炸。
直接绕过危机倒计时。

```
说明：  
```
承诺失败是惩罚，不是系统偷袭处决。
严重后果仍应通过已有系统、倒计时、事件阶段或日结风险继续推进。

```
   
⸻  
   
## 17. 承诺失败与死亡  
承诺失败不得直接造成死亡。  
规则：  
```
食物承诺失败：提高饥饿风险，但死亡仍由食物 / 饥饿系统结算。
医疗承诺失败：提高医疗压力，但死亡仍由疾病 / 重症系统结算。
住房承诺失败：提高寒冷暴露风险，但死亡仍由寒冷系统结算。
遗体承诺失败：提高信任 / 恐慌惩罚，但不直接造成新死亡。

```
   
⸻  
   
## 18. 承诺任务与旧城派  
旧城派承诺任务用于降低旧城派人数或稳定旧城派风险。  
旧城派承诺成功：  
```
旧城派人数下降。
信任上升。
恐慌下降。
可降低旧城派风险或终止对应承诺链。

```
旧城派承诺失败：  
```
旧城派人数上升。
信任下降。
恐慌上升。
旧城派风险提高。
但不提前触发离城倒计时。
不跳过离城倒计时。
不立即触发最终出走结算。
不直接扣人口。
不直接造成死亡。

```
   
⸻  
   
## 19. 承诺任务与信任 / 恐慌危机  
信任 / 恐慌危机承诺用于处理危机倒计时。  
信任承诺方向：  
```
在截止日前将信任恢复到安全线以上。
成功：信任危机解除或降级。
失败：危机倒计时继续推进。

```
恐慌承诺方向：  
```
在截止日前将恐慌降低到安全线以下。
成功：恐慌危机解除或降级。
失败：危机倒计时继续推进。

```
终火誓约 / 最高戒令生效后：  
```
对应失败轴不再直接触发失败倒计时。
但信任 / 恐慌数值仍继续存在。
信任 / 恐慌仍可参与事件、旧城派与终局评价。

```
   
⸻  
   
## 20. 承诺任务与炉心压力  
炉心压力承诺用于处理炉心过载风险。  
目标方向：  
```
降低炉心压力。
关闭过载并等待压力恢复。
完成炉心减压相关科技。
避免压力持续爆表。

```
规则：  
```
承诺任务不自动降低炉心压力。
不自动关闭 overload。
不自动改变 furnace。
不自动开启 woodfuel。
玩家仍需手动执行对应命令或完成科技。

```
   
⸻  
   
## 21. 连锁升级定义  
连锁升级用于处理长期未解决的问题。  
V1 连锁升级最多 3 层：  
```
第 1 层：status 预警
第 2 层：事件 / 承诺 / 选择
第 3 层：危机或严重后果

```
规则：  
```
同一问题最多升级 3 层。
第 3 层之后不再无限升级。
连锁升级不直接凭空生成资源。
连锁升级不直接跳过已有死亡、医疗、食物或住房系统。

```
   
⸻  
   
## 22. 连锁升级适用类型  
适用类型：  
```
食物长期不足
医疗长期不足
遗体长期未处理
住房长期不足
旧城派风险持续上升
信任 / 恐慌危机未处理
炉心压力长期过高

```
不适用类型：  
```
隐藏成就
非阻塞氛围提示
第 48 天霜落前夜
一次性叙事提示

```
   
⸻  
   
## 23. 连锁升级状态字段  
推荐字段：  
```
chain_id: string
chain_type: string
chain_stage: int
chain_last_trigger_day: int
chain_active: bool
chain_resolved: bool

```
示例：  
```
chain_type = food
chain_stage = 2
chain_active = true

```
   
⸻  
   
## 24. 连锁升级触发规则  
规则：  
```
同一问题持续存在，且超过触发条件时，进入下一层。
承诺失败可推动连锁升级。
被压下的事件不自动升级，除非底层状态本身继续恶化。
事件冷却不阻止状态继续恶化。

```
承诺失败升级不得同日连环爆炸：  
```
承诺失败后，先结算失败效果。
若失败导致对应问题升级，则标记升级状态。
升级事件进入当日事件检查流程。
同一日仍遵守事件数量限制。
若已有更高优先级重大事件，则升级事件进入队列或转为 status 预警。

```
   
⸻  
   
## 25. 连锁升级示例  
食物：  
```
第 1 层：status 提示熟食不足。
第 2 层：空锅请愿。
第 3 层：饥饿危机或死亡风险。

```
医疗：  
```
第 1 层：status 提示病床不足。
第 2 层：病床告急。
第 3 层：重症积压或死亡风险。

```
遗体：  
```
第 1 层：status 提示遗体未处理。
第 2 层：第一具遗体 / 雪下尸列。
第 3 层：信任 / 恐慌严重惩罚或旧城派风险上升。

```
旧城派：  
```
旧城派使用自身阶段结构。
不叠普通 1～3 层连锁升级。

```
   
⸻  
   
## 26. 被高优先级事件压下时  
若事件因更高优先级事件被压下：  
```
不弹完整事件。
不结算选项后果。
不偷偷替玩家选择选项。
不偷偷生成承诺。
不偷偷触发承诺失败。
不偷偷消耗资源。
不偷偷改变信任 / 恐慌。

```
后续处理：  
```
若次日问题消失，事件取消。
若次日问题仍存在，重新检查触发。
若问题升级，触发更高阶段事件。
若暂不弹窗，则至少转为 status 预警。

```
   
⸻  
   
## 27. 与全局持续时间 Patch 的关系  
承诺任务、事件冷却、主动行动冷却、临时效果必须区分。  
规则：  
```
承诺任务是限时目标。
事件冷却只限制同类事件弹窗。
主动行动冷却只限制再次使用。
临时事件效果必须写明持续时间。
持续状态如 ration 必须由玩家手动命令修改。

```
禁止：  
```
把事件冷却当成效果持续。
把承诺任务当成持续增益。
把事件选项当成永久命令修改。

```

## 第七霜落预警事件  
## 28. 第七霜落固定时间  
第七霜落固定从第 49 天开始。  
终局阶段：  
```
第 49～55 天：第七霜落

```
规则：  
```
第七霜落不会提前。
第七霜落不会推迟。
事件选择不会改变第七霜落日期。
科技、城约、誓言 / 铁腕也不会改变第七霜落日期。

```
说明：  
```
007-A7 只做预警，不做天气随机。

```
   
⸻  
   
## 29. 终局预警节点
