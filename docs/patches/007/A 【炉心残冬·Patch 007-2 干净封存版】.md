## 1. 本 Patch 定位  
007-2 负责基础民生危机与核心系统危机事件。  
本 Patch 处理：  
```
食物短缺事件
医疗压力事件
死亡 / 遗体事件
儿童相关事件
工时 / 过劳事件
煤炭危机事件
炉心压力 / 过载危机事件
寒冷暴露危机事件
信任危机事件
恐慌危机事件

```
本 Patch 不处理：  
```
旧城派完整主线
固定增员事件
第七霜落完整阶段
隐藏成就完整系统
具体数值平衡表

```

## 基础民生危机事件  
## 2. 基础民生危机总原则  
基础民生危机事件用于提示玩家城市底层生存系统正在失衡。  
基础民生危机包括：  
```
食物不足
医疗容量不足
重症积压
死亡冲击
遗体积压
儿童保护不足
儿童劳动风险
长班 / 过劳压力

```
基础民生事件通常是普通事件。  
但以下情况可升级为重大事件：  
```
第一具遗体
雪下尸列达到高阈值
冻红的手
加班后的空位
医疗系统完全崩溃时触发的病床告急
儿童群体性危机

```
重大事件必须处理后才能 end_day。  
普通事件可提示后跳过，但跳过不取消日结惩罚。  
   
⸻  
   
## 3. 基础民生事件通用规则  
基础民生事件不会替代原系统。  
规则：  
```
食物事件不凭空生成食物。
医疗事件不凭空治疗病人。
死亡事件不自动处理遗体。
儿童事件不绕过儿童炉律。
工时事件不自动重排 workers。
事件冷却不暂停日结惩罚。
事件次数上限不关闭对应问题。

```
事件只负责：  
```
提示问题。
提供选择。
提供承诺任务。
提供一次性临时缓和。
推动事件链升级。

```
   
⸻  
   
## 4. 不可用选项显示规则  
所有需要前置条件的选项，若条件不满足，不进入正式 A / B / C 编号。  
需要检查前置的内容包括：  
```
熟食
资源
建筑运行
社会适配城约
工程科技树
誓言 / 铁腕炉约
路线主动行动冷却
承诺任务槽位

```
显示格式：  
```
【当前不可用的额外手段】
- 优先儿童熟食：需要当前拥有儿童，且熟食大于 0。
- 启用更低配给：需要玩家手动执行 ration 命令。
- 留城劝诫：需要誓言路线、守炉堂运行、行动不在冷却中。

```
玩家强行输入不可用选项时：  
```
返回失败原因。
不改变状态。
不结算效果。
不消耗资源。
不生成承诺。
不消耗冷却。

```
   
⸻  
   
## 5. 触发延迟与 status 预警  
部分基础民生事件设置 1 天容错。  
规则：  
```
问题出现第 1 天：不弹事件，只在 status 显示预警。
问题持续到第 2 天：若仍未缓解，触发对应事件。

```
适用事件：  
```
空锅请愿
病床告急
生食争议

```
说明：  
```
给玩家一天补救空间，但不能让问题完全隐形。

```
   
⸻  
   
## 6. 食物事件一：空锅请愿  
事件名：  
```
空锅请愿

```
类型：  
```
食物短缺普通事件

```
触发条件方向：  
```
熟食不足、总食物可维持天数过低、轻度饥饿人口上升等问题持续至少 1 天。

```
第 1 天只显示 status 预警。  
若次日仍未缓解，触发事件。  
描述方向：  
```
食堂外有人举起空碗。

他们没有闹事，只是站在那里。
碗底被刮得很干净，像一圈冻住的月亮。

有人说：我们不是要更多，只是想知道明天还有没有。

```
选项 A：承诺补足口粮  
```
开启食物承诺任务。
成功：信任上升，恐慌下降。
失败：信任下降，恐慌上升。

```
选项 B：维持当前配给  
```
不改变食物策略。
信任小幅下降。
恐慌小幅上升。
饥饿系统继续按日结规则处理。

```
选项 C：提示调整配给 / 当日临时压低分配  
```
事件不得永久自动切换 ration。
系统可以提示玩家手动执行 ration 命令。
也可以作为当日一次性临时分配效果结算。
不修改玩家当前 ration 设置。

```
边界：  
```
不凭空生成食物。
不绕过 ration 命令。
不免除当日饥饿日结。
不永久修改配给模式。

```
重复规则：  
```
可重复。
同类冷却 4 天。

```
   
⸻  
   
## 7. 食物事件二：生食争议  
事件名：  
```
生食争议

```
类型：  
```
食物 / 医疗交叉普通事件

```
触发条件方向：  
```
连续多日居民主要食用生食。
或
食堂停工导致熟食供应连续中断。

```
单日吃生食不立刻触发。  
描述方向：  
```
有人在炉边啃半冻的生肉。

孩子看着那块肉，没有说话。
大人也没有。

第二天，医疗站多了几张发冷的脸。

```
选项 A：承诺恢复熟食供应  
```
开启食堂 / 熟食承诺任务。
成功：信任上升，恐慌下降，生食争议事件链结束。
失败：信任下降，恐慌上升，患病风险继续按既有规则提高。

```
选项 B：暂时允许继续食用生食  
```
不消耗额外资源。
信任小幅下降。
恐慌小幅上升。
居民继续按生食惩罚规则承担患病风险。

```
选项 C：优先供应儿童熟食  
前置：  
```
当前儿童人数 > 0。
当前熟食 > 0。

```
效果方向：  
```
作为当日一次性临时分配效果。
信任小幅上升。
恐慌小幅下降。
儿童相关风险下降。
食物消耗仍按既有食物系统结算。
若成人食物不足，成人饥饿风险仍正常结算。

```
边界：  
```
不永久修改 ration 模式。
不改变玩家当前配给设置。
不生成额外熟食。
不取消成人饥饿问题。

```
触发次数：  
```
每局最多 1 次。

```
触发后若居民继续长期吃生食，不再重复弹同名事件，但长期生食惩罚仍按日结系统继续。

## 8. 医疗事件一：病床告急  
事件名：  
```
病床告急

```
类型：  
```
医疗压力普通事件；医疗系统完全崩溃时可升级为重大事件

```
触发条件方向：  
```
病患人数超过医疗容量，并持续至少 1 天。
或
未收治病患达到高阈值。
或
医疗建筑因低温停工造成明显收治中断。

```
普通超容量给 1 天 status 预警。  
以下情况不需要延迟：  
```
未收治病患达到高阈值。
医疗建筑因低温停工造成明显收治中断。
重症人数达到高阈值。

```
描述方向：  
```
医疗站门口排着人。

有人坐在雪里，有人靠着墙睡着。
医生没有抬头，只说了一句：

下一个床位空出来之前，别再有人倒下了。

```
选项 A：承诺扩容医疗  
```
开启医疗承诺任务。
目标可为提高医疗容量、建成医疗站 / 医院、恢复医疗建筑运行。
成功：信任上升，恐慌下降。
失败：信任下降，恐慌上升。

```
选项 B：临时腾挪床位  
```
需要已解锁相关医疗炉律或扩容能力。
短期缓和未收治压力。
信任小幅上升或不变。
恐慌小幅下降。
可能增加医疗风险或资源消耗。

```
选项 C：维持现状  
```
不消耗资源。
信任下降。
恐慌上升。
病患继续按医疗系统规则恶化。

```
边界：  
```
临时腾挪床位不能凭空治疗病人。
不新增医疗系统。
不绕过医疗建筑与医疗炉律。

```
重复规则：  
```
可重复。
同类冷却 4 天。

```
   
⸻  
   
## 9. 医疗事件二：重症积压  
事件名：  
```
重症积压

```
类型：  
```
医疗升级事件

```
触发条件方向：  
```
重症人数达到阈值。
或
连续多日重症未能下降。

```
描述方向：  
```
医疗站里安静得不正常。

重症病人不再呻吟。
他们只是看着炉光，好像那是某种很远的东西。

医生说：我们还能救一些人。
但不是所有人。

```
选项 A：投入额外医疗配给  
```
需要已解锁额外医疗配给。
消耗熟食或指定资源。
优先缓和轻症 / 部分重症压力。
信任小幅上升。
恐慌下降。

```
选项 B：承诺扩大重症收治能力  
```
开启医疗承诺任务。
成功：信任上升，恐慌下降。
失败：信任下降，恐慌上升，死亡风险继续按医疗系统结算。

```
选项 C：接受现状  
```
不消耗资源。
信任下降。
恐慌上升。
重症继续按医疗系统结算死亡 / 伤残风险。

```
边界：  
```
不新增重症救治系统。
不直接治愈重症。
只调用已存在医疗规则。

```
触发次数：  
```
每局最多 1 次，或作为医疗类升级事件。

```

## 10. 死亡事件一：第一具遗体  
事件名：  
```
第一具遗体

```
类型：  
```
死亡处理入口重大事件

```
触发条件：  
```
本局出现第一例死亡。

```
描述方向：  
```
他们把那个人抬到炉边时，雪还粘在他的袖口上。

没人知道该把他放在哪里。
也没人愿意第一个说：

他已经不需要床位了。

```
选项 A：公开悼念  
```
信任小幅上升。
恐慌小幅下降或不变。
若死亡处理路线未建立，仍需后续处理遗体。

```
选项 B：低调处理  
```
信任小幅下降或不变。
恐慌小幅上升。
遗体仍按死亡处理系统记录。

```
选项 C：暂时搁置  
```
信任下降。
恐慌上升。
未处理遗体压力继续累积。

```
提示：  
```
死亡处理可通过社会适配城约中的墓园 / 冷藏坑路线解决。
若遗体长期未处理，将持续影响信任与恐慌。

```
触发次数：  
```
每局只触发 1 次。

```
边界：  
```
第一具遗体是死亡处理系统入口提醒。
不替代墓园 / 冷藏坑。
不自动处理遗体。

```
   
⸻  
   
## 11. 死亡事件二：雪下尸列  
事件名：  
```
雪下尸列

```
类型：  
```
死亡 / 遗体压力事件；高阈值时为重大事件

```
触发条件方向：  
```
未处理遗体达到阈值。
或
连续死亡且死亡处理能力不足。

```
描述方向：  
```
雪把遗体盖住了一半。

这本来让他们看起来安静些。
但风一吹，露出来的手指又提醒所有人：

他们还在这里。

```
选项 A：承诺处理遗体  
```
开启遗体处理承诺任务。
成功：信任上升，恐慌下降。
失败：信任下降，恐慌上升，遗体压力继续累积。

```
选项 B：举行临时悼念  
```
需要墓园 / 追思仪式，或誓言路线相关能力。
短期降低恐慌。
不自动清除遗体。

```
选项 C：继续搁置  
```
不消耗资源。
信任下降。
恐慌上升。
旧城派人数可能增长。

```
边界：  
```
悼念可以缓和恐慌，但不能替代遗体处理。
不让一场讲话把尸体讲没。

```
重复规则：  
```
可重复。
同类冷却 4 天。

```

## 12. 儿童事件一：孩子们的请求  
事件名：  
```
孩子们的请求

```
类型：  
```
儿童保护普通事件；儿童群体性危机时可升级为重大事件

```
触发条件方向：  
```
儿童数量达到一定规模。
且
儿童保护所 / 学堂缺失或停工。

```
描述方向：  
```
几个孩子站在炉边，没有靠得太近。

他们问守炉人：

我们今天要去哪儿？

没人回答。
因为城市还没决定，孩子在这里算未来，还是算劳动力。

```
选项 A：承诺安置儿童  
```
开启儿童安置承诺任务。
目标可为建成儿童保护所、恢复儿童保护所运行、建成学堂。
成功：信任上升，恐慌下降。
失败：信任下降，恐慌上升。

```
选项 B：暂时维持现状  
```
不消耗资源。
信任小幅下降。
恐慌小幅上升。
儿童仍按当前路线和状态规则处理。

```
选项 C：安排炉边杂务  
```
不需要儿童辅工路线。
不提供正式劳动力收益。
短期缓和儿童无处安置的问题。
信任小幅下降或不变。
恐慌小幅下降或不变。
儿童长期风险不解决。

```
说明：  
```
炉边杂务是临时安置，不是儿童劳动制度。
不会绕过儿童辅工路线，也不会凭空生成可用劳动力。

```
触发次数：  
```
每局最多 1 次。

```
   
⸻  
   
## 13. 儿童事件二：冻红的手  
事件名：  
```
冻红的手

```
类型：  
```
儿童劳动风险重大事件

```
触发条件方向：  
```
儿童参与劳动后出现患病 / 冻伤。
或
儿童辅工路线下儿童风险累计。

```
描述方向：  
```
一个孩子把手藏在袖子里。

医生让他伸出来。
他不肯。

后来人们才看见，那双手已经冻得发红，指节肿得像小块木炭。

```
选项 A：暂停儿童高风险劳动  
```
当日及次日，儿童辅工不可分配到高风险岗位。
若儿童正在相关岗位，自动移出并进入未分配状态。
信任小幅上升。
恐慌小幅下降。
儿童劳动风险下降。

```
数据字段方向：  
```
child_high_risk_work_suspended_until_day

```
选项 B：提供额外防寒照料  
```
消耗熟食。
儿童风险下降。
信任小幅上升。
恐慌小幅下降或不变。

```
选项 C：继续维持安排  
```
不消耗资源。
信任下降。
恐慌上升。
儿童风险继续按既有规则累积。

```
边界：  
```
该事件不禁止儿童辅工路线。
该事件只提醒其代价。
儿童死亡或严重伤残仍按已有健康 / 死亡规则处理。

```
触发次数：  
```
每局最多 1 次。

```

## 14. 工时事件一：长班后的倒下  
事件名：  
```
长班后的倒下

```
类型：  
```
工时 / 过劳普通事件

```
触发条件方向：  
```
连续使用长班。
或
工人疲劳、伤病、事故风险持续上升。

```
选项 A：暂停长班一天  
```
次日全城长班制临时暂停 1 天。
第 1 天结束后，长班制恢复到玩家原先设置，或由玩家重新调整。
信任小幅上升。
恐慌小幅下降或不变。

```
选项 B：提供熟食补偿  
```
消耗熟食。
信任小幅上升。
恐慌小幅下降。
不改变长班设置。

```
选项 C：继续长班  
```
不消耗资源。
信任下降。
恐慌上升。
过劳风险继续按既有规则累积。

```
暂停规则：  
```
暂停期间，系统不得自动把受限人群重新分配回对应岗位。
玩家强行分配时，系统返回失败原因。
暂停结束后，不自动重新分配 workers。

```
触发次数：  
```
每局最多 1 次。

```
   
⸻  
   
## 15. 工时事件二：加班后的空位  
事件名：  
```
加班后的空位

```
类型：  
```
工时 / 过劳重大事件

```
触发条件方向：  
```
长期加班导致工伤、死亡、岗位空缺或严重过劳后果。

```
选项方向：  
```
A：承诺补足人手或降低过劳压力。
B：提供熟食补偿。
C：继续维持安排。

```
规则：  
```
加班补偿只消耗熟食。
不凭空恢复劳动力。
不自动分配新增人口。
不关闭工时系统。

```
触发次数：  
```
每局最多 2 次。

```
工时 / 过劳类重大提醒事件全局最多触发：  
```
3 次

```

## 核心系统危机事件  
## 17. 核心系统危机总原则  
核心系统危机事件用于提示城市核心运行系统进入危险区。  
核心系统危机包括：  
```

炉心压力 / 过载危机
寒冷暴露危机
信任危机
恐慌危机

```
核心系统危机事件不新增 2.0 系统。  
不做：  
```
炉心故障长线
炉心爆炸随机线
炉心工程师维修线
建筑损坏线
观测站
灯塔
维修系统
提前暴雪
煤矿坍塌长线
独立暴乱镇压系统
独立罢工系统

```
   
⸻  
   
## 18. A5 与 A4 的区别  
A4 处理民生压力。  
A5 处理核心系统压力。  
区别：  
```
A4 是居民生活层面的危机。
A5 是城市核心运行层面的危机。

```
A5 事件优先级通常高于 A4。  
若 A5 与 A4 同日触发，按事件优先级与数量限制处理。  
被压下的 A4 事件不偷偷结算。

## 21. 核心事件二：炉心红线  
事件名：  
```
炉心红线

```
类型：  
```
炉心压力 / 过载重大事件

```
触发条件方向：  
```
任意过载等级使用后，炉心压力接近危险线。
连续多日使用过载 I 或过载 II。
炉心压力达到高风险阈值。
炉心压力达到 100。

```
说明：  
```
过载 I 与过载 II 都会推动炉心压力。
区别只在压力增长速度与风险强度，不允许过载 I 逃出红线检测。

```
炉心压力达到 100 时：  
```
第一次达到 100：触发强制红线警告。
玩家必须处理该事件，才能 end_day。
若玩家选择关闭过载，炉心进入危险恢复状态，不立刻失败。
若玩家继续维持过载，并在下一次日结后压力仍处于爆表状态，则触发炉心崩毁失败。

```
显示提示：  
```
炉心压力已达到极限。
这是最后一次手动关闭过载的机会。
若继续维持过载，下一次日结可能导致炉心崩毁。

```
选项 A：立即关闭过载的命令提示  
```
事件不自动关闭 overload。
系统提示玩家可执行 overload off。
玩家手动执行后，按过载系统正常结算压力变化、供暖变化与寒冷风险。

```
选项 B：承诺完成炉心减压方案  
前置：  
```
工程科技树中存在可研究的炉心减压 / 过载稳定 / 终局炉心稳定相关科技。
或
玩家可以通过关闭过载并等待压力自然恢复来完成目标。

```
若没有可用目标：  
```
该选项不进入正式 A / B / C。
只显示为不可用提示。

```
成功方向：  
```
信任上升。
恐慌下降。
炉心红线事件链降级。

```
失败方向：  
```
信任下降。
恐慌上升。
炉心压力继续按既有过载规则结算。
不会凭空降低炉心压力。

```
选项 C：维持过载 / 暂不处理  
```
不消耗额外资源。
信任下降。
恐慌上升。
炉心压力继续按过载系统结算。
若已处于压力 100，下一次日结仍爆表可能触发炉心崩毁失败。

```
重复规则：  
```
可重复。
冷却 4 天。
压力 100 强制红线警告不受普通冷却屏蔽。

```

描述方向：  
```
夜里，有人把炉灰抹在墙缝上。

```

```
事件不自动提高 furnace 档位。
系统提示玩家可执行 furnace <更高档位>。

若玩家手动提高炉心档位，则按炉心消耗、供暖、寒冷暴露系统正常结算。

```
选项 C：维持现状  
```
不消耗额外煤炭或材料。
信任下降。
恐慌上升。
寒冷暴露继续按既有规则结算。

```
边界：  
```
住宅不能被 heat 单独应急加热。
事件不自动建造住房。
事件不自动升级住宅。
事件不允许对住宅使用 heat。
承诺任务只提供时间窗口，不免除当日寒冷结算。
```

重复规则：
