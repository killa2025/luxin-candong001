# A 【炉心残冬·Patch 006C-1 干净封存版】  
主题：誓言与铁腕炉约——页面开放、路线互斥、核心建筑、主动行动与系统边界 状态：干净封存版 归属：006C 誓言与铁腕炉约  
   
⸻  
   
## 1. 命名  
006C 页面名为：  
```
誓言与铁腕

```
系统归属为：  
```
炉约

```
单条仍称：  
```
炉律

```
完整称呼：  
```
誓言与铁腕炉约

```
玩家可见路线名统一为：  
```
誓言路线
铁腕路线

```
内部分类可保留为：  
```
精神路线
秩序路线

```
说明：  
```
誓言对应精神路线。
铁腕对应秩序路线。

```
玩家提示、面板、选择确认、路线说明中，统一使用“誓言路线 / 铁腕路线”。 “精神 / 秩序”只作为内部分类或开发注释使用。  
006C 不是普通社会适配城约，也不是工程科技。  
   
⸻  
   
## 2. 系统定位  
006C 处理城市在中后期压力下选择何种方式维持内部稳定。  
它回答的问题是：  
```
当死亡、饥饿、寒冷和旧城派传言不断撕裂炉城时，居民为什么还愿意留下？

```
两条路线：  
```
誓言路线：用誓言、悼亡、共同忍耐与意义感维系城市。
铁腕路线：用巡查、登记、纪律、留置与强制管控维系城市。

```
006C 主要影响：  
```
信任
恐慌
危机倒计时
内部冲突
旧城派危机
终局评价
最终失败轴改写

```
006C 不直接解决：  
```
煤炭不足
食物不足
住房不足
医疗崩溃
炉心熄灭
第七霜落本身

```
说明：  
```
誓言不能把煤喊出来。
铁腕也不能把冻死的人巡查回来。

```
   
⸻  
   
## 3. 开放时机  
誓言与铁腕炉约不开局开放。  
页面开放条件采用：  
```
阶段进度 + 社会适配推进度

```
而不是：  
```
点完全部社会适配城约

```
原因：  
```
006A 社会适配城约存在互斥路线。
若要求“全部点完”，可能导致页面永远无法开放。

```
006C 的具体开放天数、社会适配炉律数量、保底开放时机，由 006C-4 统一确定。  
当前 006C-4 草案口径为：  
```
第 30 天 + 已签署社会适配炉律 8 条开放。
第 35 天保底开放。

```
若后续 006C-4 对开放阈值进行数值复核，以 006C-4 最终封存版为准。  
   
⸻  
   
## 4. 开放条件  
誓言与铁腕炉约在中后期开放。  
开放判断只使用：  
```
当前阶段进度
已签署社会适配炉律数量
保底开放天数

```
社会适配推进度统计规则：  
```
统计玩家实际签署的社会适配炉律数量。
互斥后被锁定、不可选择的炉律不计入总数。
不要求玩家点完所有社会适配炉律。

```
以下内容不作为 006C 页面直接开放条件：  
```
首次大量死亡
医疗压力明显恶化
旧城派传言出现
信任进入危险区间
恐慌进入危险区间

```
说明：  
```
大量死亡已有保护期兜底。
医疗压力应由医疗建筑、医疗炉律与玩家调度解决。
旧城派是中期主线危机，不是誓言与铁腕页面开启按钮。
信任 / 恐慌危险区间已有危机倒计时机制，不需要重复触发 006C。

```
   
⸻

当前保底开放方向：  
```
若玩家迟迟未达到普通开放条件，则在第 35 天保底开放。

```
说明：  
```
誓言与铁腕是中后期稳定系统。
不能拖到第七霜落才刚刚开放。
第七霜落是终局验收，不是意识形态系统的新手教程。

```
若后续 006C-4 对保底天数进行数值复核，以 006C-4 最终封存版为准。

## 7. 页面显示规则  
誓言与铁腕页面不默认展示完整路线细节。  
显示规则：  
```
第一条誓言炉律：显示完整描述、影响方向与路线锁定提示。
第一条铁腕炉律：显示完整描述、影响方向与路线锁定提示。
后续未解锁炉律：只显示名称、所属分支、效果方向标签、未解锁状态。
已解锁可签署炉律：显示完整效果说明。
最终条未解锁前：只显示名称或模糊方向，不显示完整失败轴改写细节。

```
示例：  
```
未解锁：
悼亡钟声｜誓言路线｜方向：死亡安抚 / 降低恐慌
未解锁：
炉心登记牌｜铁腕路线｜方向：旧城派管控 / 稳定秩序

```
说明：  
```
玩家需要知道大致方向，但不需要提前看到整条路线的全部精确效果。
这样能降低面板负担，也避免 AI 玩家过度计算未来收益。

```
   
⸻  
   
## 8. 路线互斥规则  
誓言路线与铁腕路线全局互斥。  
规则：  
```
签署第一条誓言炉律后，铁腕路线永久锁定。
签署第一条铁腕炉律后，誓言路线永久锁定。

```
禁止：  
```
先签誓言前期炉律，再切铁腕路线。
先签铁腕前期炉律，再切誓言路线。
同时拥有誓言路线与铁腕路线。
同时解锁终火誓约与最高戒令。

```
说明：  
```
第一条路线炉律就是城市方向选择。
誓言和铁腕可以都被看见，但不能都被走完。

```

## 12. 核心建筑与炉旁设施需求  
誓言 / 铁腕路线都需要制度承接物。  
006C 路线承接物分为两类：  
```
路线核心建筑
炉旁设施 / 行动承接物

```
路线核心建筑：  
```
誓言路线：守炉堂
铁腕路线：巡查所

```
守炉堂 / 巡查所是占槽建筑。  
规则：  
```
守炉堂 / 巡查所占普通区域槽位。
守炉堂 / 巡查所不使用特殊核心槽。
守炉堂 / 巡查所需要玩家手动 build。
守炉堂 / 巡查所不会在签署路线入口后自动生成。
守炉堂 / 巡查所不会自动占槽。
守炉堂 / 巡查所上限均为 1 座。

```
路线入口签署后，只解锁对应建筑的建造权限：  
```
签署誓言路线入口后，解锁守炉堂建造权限。
签署铁腕路线入口后，解锁巡查所建造权限。

```
建筑口径：  
```
守炉堂：占 1 槽，上限 1，推荐内环 / 中环。
巡查所：占 1 槽，上限 1，推荐内环 / 中环。

```
代码化方向：  
```
守炉堂 / 巡查所：
allowed_zones = ["inner", "middle"]
recommended_zone = ["inner", "middle"]
slot_cost = 1
max_count = 1

```
说明：  
```
守炉堂 / 巡查所是路线建筑，不是炉旁行动效果。
路线入口炉律解锁核心建筑建造资格，但不会自动建成。

```
   
⸻

## 15. 核心建筑运行规则  
核心建筑运行条件：  
```
核心建筑已建成。
核心建筑有人值守。
核心建筑处于运行状态。

```
值守条件：  
```
至少 1 名可工作人口值守即可运行。
普通儿童不可作为工作人员。
儿童辅工 / 学徒不能独立运行。
若使用儿童辅工 / 学徒辅助，至少需要成人兜底。

```
核心建筑不按人手数量计算效率。  
只判断：  
```
是否已建成。
是否有人值守。
是否满足成人兜底规则。
是否处于运行状态。

```
核心建筑未建成、无人值守、停工或未运行时：  
```
对应路线主动行动不可用。
对应路线后续终章前置检查不满足。
已签署的永久炉律不失效。
面板显示对应异常状态。

```
对应主动行动限制：  
```
守炉堂未运行：守炉宣誓、敲悼亡钟、共食誓餐、留城劝诫等主动行动不可用。
巡查所未运行：巡查、发布公告、临时留置、登记审查等主动行动不可用。

```
永久炉律规则：  
```
已签署的永久炉律不会因为守炉堂 / 巡查所未建成、停工或无人值守而失效。
建筑运行状态只影响主动行动与后续终章前置检查，不回滚已经签署的炉律。

```
   
⸻  
   
## 16. 主动行动规则方向  
誓言 / 铁腕路线可解锁主动行动。  
誓言行动方向：  
```
守炉宣誓
敲悼亡钟
共食誓餐
留城劝诫

```
铁腕行动方向：  
```
巡查
发布公告
临时留置
登记审查

```
主动行动规则：  
```
必须由对应炉律解锁。
需要对应核心建筑已建成并处于运行状态。
使用后进入行动冷却。
行动冷却与路线签署冷却分开记录。
不同主动行动分别记录冷却。

```
主动行动不会因为路线入口签署而自动执行。  
主动行动仍需满足：  
```
对应路线已选择。
对应行动已由炉律或路线解锁。
对应核心建筑已建成。
对应核心建筑处于运行状态。
行动不在冷却中。
满足行动目标条件。

```
路线入口签署 ≠ 核心建筑自动建成。 核心建筑建成 ≠ 主动行动自动执行。  
玩家仍需手动使用主动行动。  
具体行动效果与冷却由 006C-2 / 006C-3 / 006C-4 确定。

## 17. 炉心关闭与誓言 / 铁腕  
炉心关闭时，誓言与铁腕路线的已签署永久炉律不因炉心关闭而自动失效。  
规则：  
```
炉心关闭只通过既有温度、寒冷暴露、信任下降、恐慌上升等系统产生负面影响。
不额外关闭已签署的誓言 / 铁腕永久炉律。
不额外删除炉旁设施状态。
不额外回滚路线选择。

```
但核心建筑若因无人值守、停工或运行条件不满足而未运行，对应路线主动行动仍不可用。  
说明：  
```
炉心关闭本身已经是重大危机。
不再叠加“已签署炉律失效”惩罚，避免后期连锁雪崩过强。

```
结论：  
```
炉心关闭影响城市状态。
城市状态再影响信任、恐慌、死亡、旧城派等系统。
誓言与铁腕不因炉心关闭单独回滚已签署炉律。

```
   
⸻  
   
## 18. 最终条与失败轴改写  
誓言路线最终条：  
```
终火誓约

```
效果方向：  
```
信任降至 0 时，不立刻失败。
恐慌达到失败条件时，仍然失败。

```
铁腕路线最终条：  
```
最高戒令

```
效果方向：  
```
恐慌达到上限时，不立刻失败。
信任降至 0 时，仍然失败。

```
终章炉律不占槽。  
```
终火誓约是终章炉律，不是建筑，不占槽。
最高戒令是终章炉律，不是建筑，不占槽。

```
终章炉律前置要求：  
```
终火誓约需要守炉堂存在并运行，同时满足誓言路线完整前置。
最高戒令需要巡查所存在并运行，同时满足铁腕路线完整前置。

```
被改写的失败轴仍可在 status 中显示为低状态提示，但不再作为直接失败警告。  
说明：  
```
最终条只改写一个失败轴。
不能同时免疫信任归零与恐慌爆满。
不能让城市无条件胜利。

```
   
⸻  
   
## 19. 失败轴改写后的内部运算规则  
誓言 / 铁腕最终条只改写对应失败轴的“直接失败判定”，不删除该数值本身。  
规则：  
```
信任仍然继续计算。
恐慌仍然继续计算。
```
