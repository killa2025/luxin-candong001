```

## 四、006A 总原则  
**4.1 签署炉律**  
```
签署炉律只产生一次签署瞬间影响。
签署后不每天重复结算签署影响。

```
签署炉律可能解锁：  
```
建筑
公共设施
附属功能
行动
模式
岗位权限
医疗 / 食物 / 工时 / 死亡处理规则

```
不能把所有炉律都当成建筑解锁。  
   
⸻  
   
**4.2 行动与模式**  
```
签署炉律 = 解锁行动 / 模式 / 权限。
解锁后不会自动执行。
玩家必须主动输入命令或点击按钮。

```
例如：  
```
签署“粗粮汤”只解锁粗粮汤配给模式。
玩家输入“配给 粗粮汤”后，当前配给才切换为粗粮汤。

签署“分级救治”只解锁分级救治行动。
玩家输入“分级救治 <建筑编号>”并确认后，才执行分级救治。

```
   
⸻  
   
**4.3 情绪结算**  
信任 / 恐慌分层结算：  
```
签署瞬间影响
行动使用影响
每日持续影响
事件影响
死亡 / 伤残 / 饥荒 / 医疗崩溃等系统影响

```
规则：  
```
签署影响不重复结算。
行动影响不等于签署影响。
重大事件由社会情绪系统统一处理。
死亡、伤残、放弃治疗等后果不得重复扣信任 / 重复加恐慌。

```
   
⸻  
   
**4.4 数值处理**  
006A 只记录方向，不补最终数值。  
```
具体产出倍率
具体风险概率
具体容量
具体冷却天数
具体信任 / 恐慌数值
具体治疗 / 死亡 / 伤残比例

```
全部留给数值表、医疗表、情绪表、疾病风险表或试玩平衡。

## 五、006A 分支总览  
**5.1 儿童 / 人力统筹线**  
主线互斥：  
```
儿童辅工线 ↔ 儿童保护线

```
分支树：  
```
儿童辅工线
├─ 儿童辅工：低危岗位
└─ 儿童辅工：全岗位

儿童保护线
└─ 儿童保护所
   └─ 学堂
      ├─ 医疗学徒
      └─ 工程学徒

```
核心口径：  
```
儿童辅工线提供前期有限辅助劳动力，但提高儿童风险并影响终局评价。
儿童保护线不提供短期普通劳动力，但降低儿童风险，提高长期稳定。
医疗学徒与工程学徒互斥。

```
已确认修正：  
```
儿童辅工低危岗位删除“儿童保护所辅助 / 学堂辅助”。
儿童可参与散落煤炭 / 散落木材 / 散落钢铁采集。
V1 无散落食物点。
暴雪、黑霜、第七霜落期间禁止或强限制儿童外勤。

```
学堂岗位口径：  
```
学堂必须至少 1 名工程师运行。
工人不可进入学堂，且不能替代工程师。
儿童、医疗学徒、工程学徒也不能替代工程师。

```
学徒固定名单：  
```
医疗学徒：医疗站、医院。
工程学徒：研究所。

```
修造工坊已从 V1 正式建筑中删除，不再作为工程学徒岗位。  
儿童与娱乐建筑：  
```
儿童、儿童辅工、医疗学徒、工程学徒不可被分配到小酒馆 / 大赌场工作。
小酒馆 / 大赌场全城娱乐效果默认影响所有居民，包括儿童。

```
   
⸻

**5.5 医疗处置线**  
分支树：  
```
基础医疗法
├─ 稳定疗法 ↔ 激进疗法
├─ 扩容收治
│  └─ 养护所
├─ 额外医疗配给
└─ 分级救治

```
基础医疗法：  
```
开局只有极低容量临时救护。
临时救护不是建筑，不占槽，不能替代医疗站。
基础医疗法解锁正式医疗站。
医疗站不是开局默认可建建筑。
医疗站最多 4 座。
医院最多 2 座，但医院不由 006A 直接解锁。

```
稳定疗法 / 激进疗法：  
```
稳定疗法降低重症死亡压力，但治疗更慢或占床更久。
激进疗法提高重症处理速度，但伤残风险上升。
二者互斥。

```
扩容收治：  
```
扩容收治不是新建筑。
它是医疗站 / 医院的收治规则变化。
增加容量，但降低治疗质量或恢复速度。

```
额外医疗配给：  
```
主动医疗行动。
只消耗熟食。
不新增干净布料、恢复物资、药品包、医疗物资等资源字段。
按实际治疗 / 推进人数消耗熟食，不对空气扣粮。
不复活死者，不直接消除伤残。
3 / 20 / 60 为已登记暂定示例，待后续平衡表复核；未完全封存，不进入代码窗最终实现清单。

```
分级救治：  
```
签署只解锁行动。
签署瞬间信任无变化，恐慌无变化。
启动时必须指定一座医疗站或医院。
不能指定养护所。
不能覆盖全城医疗系统。

```
分级救治启动效果方向：  
```
一部分轻症或高生还重症被优先治愈或脱离危险。
一部分濒危、低生还重症或低优先级病人被延后、放弃或粗暴处理。
被放弃或低优先级处理的病人可能死亡或转为伤残。
死亡、伤残、放弃治疗结果由死亡系统、伤残系统、社会情绪系统和固定事件系统统一结算。
启动过分级救治进入终局统计 / 终局文案。

```
养护所：  
```
独立建筑。
用于安置伤残人口。
不恢复伤残者劳动能力。
不是释放医疗床位的唯一钥匙。
伤残者完成急救后会离开医疗床位。

```

```

加班日 <建筑编号>

```
V1 删除工坊 / 修造工坊作为正式建筑。
所有旧文中涉及修造工坊的建筑、岗位、科技前置、加班日目标、学徒岗位口径作废。

```
   
⸻  
   
**9.3 火化场**  
```
火化场旧口径作废。
V1 死亡处理路线统一为墓园线 ↔ 冷藏坑线。
旧火化 / 快速清理功能并入冷藏坑路线后续炉律或文案。

```
   
⸻  
   
**9.4 医疗站**  
```
医疗站不是开局默认可建建筑。
基础医疗法解锁正式医疗站。
开局只有极低容量临时救护。
临时救护不是建筑，不占槽，不能替代医疗站。

```
   
⸻  
   
**9.5 学堂**  
```
学堂属于准关键建筑。
学堂低温停课。
学堂必须至少 1 名工程师运行。
工人不可进入学堂，不能替代工程师。
儿童 / 学徒也不能替代工程师。

```
   
⸻  
   
**9.6 食物与应急口粮**  
```
V1 无散落食物点。
应急口粮不新增虫子、蠕虫或其他库存资源。
应急口粮不额外提高患病风险。

```
   
⸻  
   
**9.7 分级救治**  
```
分级救治不是全城医疗总开关。
分级救治必须指定医疗站或医院。
分级救治不能指定养护所。
分级救治签署瞬间不直接改变信任 / 恐慌。
启动结果交由死亡 / 伤残 / 社会情绪系统结算。

```
   
⸻  
   
**9.8 娱乐减压风险**  
```
娱乐减压线不拆分传染病、皮肤病、斗殴受伤、轻伤等独立风险。
所有相关旧风险表述统一收口为：全局生病风险轻微上升。

```
