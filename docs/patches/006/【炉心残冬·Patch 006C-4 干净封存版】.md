# 【炉心残冬·Patch 006C-4 干净封存版】  
主题：誓言与铁腕炉约——开放节奏、签署冷却与主动行动冷却 归属：006C 誓言与铁腕炉约 状态：干净封存版  
   
⸻  
   
## 1. 本 Patch 定位  
006C-4 负责确定誓言与铁腕炉约的推进节奏。  
包含：  
```
页面开放条件
社会适配推进度统计
誓言 / 铁腕炉律签署冷却
主动行动冷却
同日签署规则
终章可达性
核心建筑运行与行动可用关系
冷却失败反馈
AI 玩家提示规则

```
不包含：  
```
具体信任 / 恐慌变化数值
具体熟食消耗数值
旧城派出走事件后果
旧城派人数降低具体数值
006A 社会适配城约完整冷却重算
最终数值平衡表
建筑成本表
终局评价权重

```
说明：  
```
006C-4 只负责“什么时候开放、什么时候能签、什么时候能用行动”。
不负责把所有信任、恐慌、资源与旧城派数值一次性算完。

```
   
⸻  
   
## 2. 不强制签署炉律  
炉律冷却结束后，玩家可以签署炉律，但不强制签署。  
规则：  
```
冷却结束后，玩家可选择签署。
未签署不会自动惩罚。
系统只提示“有可签署炉律”，不强迫玩家选择。

```
说明：  
```
守炉人应当是决策者，不是被系统按头打卡的冻土公务员。

```
   
⸻  
   
## 3. 页面开放条件  
誓言与铁腕炉约不开局开放。  
页面满足以下任一条件时开放：  
```
条件一：
当前天数 ≥ 第 30 天
且
已签署社会适配炉律 ≥ 8 条

条件二：
当前天数 ≥ 第 35 天
无论社会适配推进度如何，保底开放

```
说明：  
```
第 30 天开放给推进快的玩家。
第 35 天保底开放，避免玩家漏点社会适配导致 006C 永久不开。

```
   
⸻  
   
## 4. 社会适配推进度统计  
社会适配推进度只统计 006A 社会适配炉律。  
计入：  
```
玩家本局实际签署的 006A 社会适配炉律

```
不计入：  
```
被互斥路线锁定、无法再选择的炉律
006B 工程科技
006C 誓言 / 铁腕炉律
主动行动
临时命令
应急口粮、医疗配给、加班日等可点击行动

```
数据字段方向：  
```
social_laws_signed_count: int

```
开放判断：  
```
if day >= 30 and social_laws_signed_count >= 8:
    unlock_oath_order_page = true

if day >= 35:
    unlock_oath_order_page = true

```
说明：  
```
被互斥锁掉的 006A 炉律不强行计入，也不要求玩家点完所有 006A。
006C 的开放依赖“实际推进过社会适配”，不是要求不可能完成的全收集。

```
   
⸻

玩家尚未选择路线时，可见：  
```
守炉誓言｜誓言路线入口｜完整描述
炉城巡令｜铁腕路线入口｜完整描述

```
签署其中一条后：  
```
签署守炉誓言 → 关闭铁腕路线

```
第一条路线炉律必须二次确认。  
誓言路线确认：  
```
你即将选择誓言路线。

```
铁腕路线确认：  
```
你即将选择铁腕路线。
签署后，誓言路线将永久关闭。

confirm sign 炉城巡令

```

## 7. 006C 签署冷却  
誓言 / 铁腕炉律使用独立冷却。  
固定冷却：  
```
006C 炉律签署冷却：2 天

```
结算方式：  
```
若玩家在第 D 天签署 006C 炉律，
下一条 006C 炉律最早可在第 D+2 天签署。

```
示例：  
```
第 30 天签署守炉誓言。
第 31 天不可签署下一条 006C 炉律。
第 32 天可签署下一条 006C 炉律。

```
数据字段方向：  
```
oath_order_law_next_available_day: int

```
判断方式：  
```
if current_day >= oath_order_law_next_available_day:
    006C 炉律可签署
else:
    006C 炉律冷却中

```
   
⸻  
   
## 8. 006A 与 006C 冷却独立  
006A 社会适配炉律与 006C 誓言 / 铁腕炉律冷却独立。  
规则：  
```
同一天最多签署 1 条 006A 社会适配炉律。
同一天最多签署 1 条 006C 誓言 / 铁腕炉律。
若两边冷却都可用，同一天可以各签署 1 条。

```
说明：  
```
中后期允许玩家同时推进社会适配与誓言 / 铁腕。
但不允许单日连续点穿同一个炉约页面。

```
禁止：  
```
同一天连续签署多条 006C 炉律。
用 006A 冷却阻塞 006C。
用 006C 冷却阻塞 006A。

```
   
⸻  
   
## 9. 未冷却完成时的签署反馈  
如果玩家尝试签署 006C 炉律，但冷却未完成，系统返回明确提示。  
格式：  
```
执行失败：誓言与铁腕炉约仍在冷却中。
下一条 006C 炉律可在第 X 天签署。
当前是第 Y 天。

```
示例：  
```
执行失败：誓言与铁腕炉约仍在冷却中。
下一条 006C 炉律可在第 32 天签署。
当前是第 31 天。

```
冷却失败规则：  
```
操作失败。
不消耗冷却。
不改变状态。
不结算效果。
不结算代价。
不推进炉律。
不触发路线互斥变化。

```
说明：  
```
冷却失败只是失败反馈，不是一次“半成功操作”。

```
   
⸻

```
第 30 天：第 1 条，路线入口

第 34 天：第 3 条
第 36 天：第 4 条
第 38 天：第 5 条
第 40 天：第 6 条，路线终章

```
第 35 天：第 1 条，路线入口
第 37 天：第 2 条
第 39 天：第 3 条
第 41 天：第 4 条
第 43 天：第 5 条
第 45 天：第 6 条，路线终章

```
结论：  
```
专注推进的玩家，即使第 35 天才开放，也能在第七霜落前完成路线终章。

```
注意：  
```
上述示例只表示签署冷却上的理论最早路线。
终章实际签署还需要满足对应路线建筑存在并运行。

```

## 11. 终章可达性目标  
006C 的设计目标不是让玩家必定点完终章，而是让专注玩家有机会点完。  
规则方向：  
```
早开玩家可在黑霜中段前后完成终章。
保底开放玩家可在第七霜落前完成终章。
拖延或不投入的玩家可能无法完成终章。

```
说明：  
```
终火誓约 / 最高戒令是强力终章，不应免费保证拿到。
但也不能晚到第七霜落才刚能启动。

```
   
⸻  
   
## 12. 终章前置规则  
终火誓约是誓言路线终章炉律。  
前置：  
```
必须已签署誓言路线前 5 条炉律。
必须已建成守炉堂。
守炉堂必须处于运行状态。
必须满足 006C 签署冷却。

```
即誓言路线前五条为：  
```
守炉誓言
悼亡钟声
共食誓餐
余烬名册
留城誓约

```
最高戒令是铁腕路线终章炉律。  
前置：  
```
必须已签署铁腕路线前 5 条炉律。
必须已建成巡查所。
巡查所必须处于运行状态。
必须满足 006C 签署冷却。

```
即铁腕路线前五条为：  
```
炉城巡令
晨钟点名
统一公告
临时留置
户籍清查

```
说明：  
```
终章不是单分支尽头。
玩家需要完成本路线主要结构后，才能签署最终炉律。

```

## 13. 核心建筑建造与运行时机  
守炉堂 / 巡查所均为路线核心建筑。  
建造规则：  
```
签署守炉誓言后，解锁守炉堂建造权限。
签署炉城巡令后，解锁巡查所建造权限。
守炉堂 / 巡查所需要玩家手动 build。
守炉堂 / 巡查所不会自动生成。
守炉堂 / 巡查所不会自动占槽。

```
运行生效规则：  
```
守炉堂 / 巡查所建成后，若满足值守条件，当日即可视为运行。
不需要 D+1 天等待。
不新增施工后等待日。
不新增修缮期。

```
也就是说：  
```
第 D 天建成核心建筑。
第 D 天分配满足条件的人手。
第 D 天即可视为运行。
第 D 天即可使用已解锁且不在冷却中的对应主动行动。

```
说明：  
```
006C 不通过 D+1 运行延迟防止连招。
若需要限制推进速度，应通过签署冷却、建筑成本、槽位、人手、主动行动冷却与行动条件限制。

```
作废旧口径：  
```
第 D 天建成并分配人手，第 D+1 天开始视为运行。
第 D+1 天起可使用对应主动行动。

```
   
⸻  
   
## 14. 核心建筑运行条件  
守炉堂 / 巡查所均为非生产型路线建筑。  
统一运行规则：  
```
至少 1 名可工作人口值守即可运行。
普通儿童不可作为工作人员。
儿童辅工 / 学徒不能独立运行。
若使用儿童辅工 / 学徒辅助，至少需要成人兜底。
无人值守时，建筑未运行。
不按人手数量计算效率。

```
核心建筑未运行时：  
```
对应路线主动行动不可使用。
对应路线终章炉律前置检查不满足。
已签署的永久炉律不因此失效。
最终条失败轴改写不因此反复开关。

```
说明：  
```
建筑运行状态只管“能不能使用主动行动”和“能不能签终章”。
不回滚路线选择，不撤销已签署炉律。

```
   
⸻  
   
## 15. 核心建筑温度口径  
誓言 / 铁腕路线建筑不参与温度停工系统。  
规则：  
```
守炉堂不受低温、黑霜、第七霜落导致的停工影响。
巡查所不受低温、黑霜、第七霜落导致的停工影响。
守炉堂 / 巡查所不需要 heat。
守炉堂 / 巡查所不可 heat。
炉心关闭不额外影响守炉堂 / 巡查所。

```
运行判断只看：  
```
是否已建成。
是否有人值守。
是否满足成人兜底规则。
是否被后续特殊事件明确禁用。

```
不看：  
```
有效温度
黑霜温度
第七霜落温度
heat 覆盖
炉心当前档位
炉心是否关闭

```
说明：  
```
守炉堂 / 巡查所是制度承接建筑，不是生产建筑。
寒冷会通过城市整体状态影响信任、恐慌、死亡和危机，但不让这两个路线建筑进入温度停工系统。

```
   
⸻  
   
## 16. 主动行动冷却总规则  
主动行动冷却与炉律签署冷却分开。  
规则：  
```
签署炉律冷却不影响主动行动冷却。
主动行动冷却不影响签署炉律冷却。
不同主动行动分别记录冷却。
主动行动冷却只限制再次使用，不代表效果持续。

```
数据字段方向：  
```
action_oath_declaration_next_available_day
action_mourning_bell_next_available_day
action_shared_meal_next_available_day
action_stay_persuasion_next_available_day

action_patrol_next_available_day
action_announcement_next_available_day
action_detain_next_available_day
action_registry_check_next_available_day

```
判断方式：  
```
if current_day >= action_x_next_available_day:
    行动可用
else:
    行动冷却中

```
   
⸻  
   
## 17. 誓言路线主动行动冷却  
誓言路线主动行动冷却如下：  
```
守炉宣誓：3 天
敲响悼亡钟：4 天
共食誓餐：4 天
留城劝诫：3 天

```
结算方式：  
```
若玩家在第 D 天使用主动行动，
该行动下一次最早可在第 D+冷却天数 天使用。

```
示例：  
```
第 32 天使用守炉宣誓。
守炉宣誓冷却 3 天。
下一次可在第 35 天使用。

```
行动效果持续规则：  
```
守炉宣誓释放时结算一次。
敲响悼亡钟释放时结算一次。
共食誓餐释放时结算一次。
留城劝诫释放时结算一次。
冷却期间不每日重复结算上一次行动效果。

```
   
⸻  
   
## 18. 铁腕路线主动行动冷却  
铁腕路线主动行动冷却如下：  
```
巡查：3 天
发布公告：3 天
临时留置：5 天
登记审查：3 天

```
结算方式：  
```
若玩家在第 D 天使用主动行动，
该行动下一次最早可在第 D+冷却天数 天使用。

```
示例：  
```
第 34 天使用临时留置。
临时留置冷却 5 天。
下一次可在第 39 天使用。

```
行动效果持续规则：  
```
巡查释放时结算一次。
发布公告释放时结算一次。
临时留置释放时结算一次。
登记审查释放时结算一次。
冷却期间不每日重复结算上一次行动效果。

```
说明：  
```
临时留置是较强恐慌压制行动，因此冷却更长。

```
   
⸻  
   
## 19. 主动行动使用条件  
主动行动必须满足：  
```
对应路线已选择。
对应炉律已签署。
对应核心建筑已建成并运行。
对应行动不在冷却中。
满足行动自身条件。

```
誓言路线特殊条件：  
```
敲响悼亡钟：近期存在死亡事件时可用。
共食誓餐：熟食足够时可用。
留城劝诫：旧城派危机激活且旧城派人数 > 0 时可用。

```
铁腕路线特殊条件：  
```
登记审查：旧城派危机激活且旧城派人数 > 0 时可用。

```
说明：  
```
核心建筑运行只决定主动行动是否可用。
它不会让主动行动自动执行，也不会让行动效果每日自动持续。

```
   
⸻  
   
## 20. 主动行动失败反馈  
若主动行动冷却未完成，系统返回明确提示。  
格式：  
```
执行失败：<行动名>仍在冷却中。
下一次可在第 X 天使用。
当前是第 Y 天。

```
示例：  
```
执行失败：临时留置仍在冷却中。
下一次可在第 39 天使用。
当前是第 37 天。

```
若主动行动条件不足，系统返回缺失条件。  
示例：  
```
无法使用共食誓餐：
- 需要守炉堂已建成并运行
- 需要熟食足够
- 当前熟食不足
无法使用登记审查：
- 需要巡查所已建成并运行
- 需要旧城派危机已激活
- 当前旧城派危机尚未激活

```
行动失败规则：  
```
不消耗行动冷却。
不改变状态。
不结算效果。
不结算代价。
不消耗熟食。
不降低旧城派人数。
不改变信任 / 恐慌。

```
   
⸻  
   
## 21. 主动行动一次性口径  
006C 主动行动默认都是一次性效果，并进入自身冷却。  
包括：  
```
守炉宣誓
敲响悼亡钟
共食誓餐
留城劝诫
巡查
发布公告
临时留置
登记审查

```
规则：  
```
行动释放时结算一次。
行动效果不每日持续。
行动进入自身冷却。
冷却结束后可再次使用。

```
说明：  
```
006C 主动行动不是持续光环。
守炉堂 / 巡查所运行只决定能不能使用主动行动，不让行动效果每天自动生效。

```
   
⸻  
   
## 22. 终火誓约 / 最高戒令持续规则  
终火誓约签署成功后：  
```
永久改写信任失败轴。
信任归零不再直接触发失败倒计时。
信任数值仍存在。
信任仍参与旧城派、事件、行动、居民状态和终局评价。

```
最高戒令签署成功后：  
```
永久改写恐慌失败轴。
恐慌爆满不再直接触发失败倒计时。
恐慌数值仍存在。
恐慌仍参与旧城派、事件、行动、居民状态和终局评价。

```
最终炉律持续规则：  
```
最终炉律签署成功后是永久路线效果。
不会因一天结束自动取消。

## 25. 冷却与日结顺序  
V1 采用每日回合制。  
推荐顺序：  
```
玩家查看面板
玩家执行签署 / 主动行动 / 建筑 / 分配等操作
end_day
日结
推进到下一天
新一天开始时，根据当前天数判断可用操作

```
签署或使用行动后，不需要每日递减字段。  
系统直接记录：  
```
next_available_day

```
判断方式：  
```
if current_day >= next_available_day:
    操作可用
else:
    操作冷却中

```
说明：  
```
使用“下一次可用日”比使用“剩余冷却天数”更不容易出错。

```

## 27. 与温度停工预警的关系  
全局日结预警会检查关键建筑是否因有效温度过低停工。  
但守炉堂 / 巡查所不参与温度停工系统。  
规则：  
```
守炉堂不需要 heat。
巡查所不需要 heat。
守炉堂不可 heat。
巡查所不可 heat。
守炉堂 / 巡查所不会因低温、黑霜、第七霜落触发温度停工。

```
因此，end_day 前“建筑温度停工预警”不应把守炉堂 / 巡查所列为温度停工对象。  
但仍应提示：  
```

守炉堂 / 巡查所无人值守。
守炉堂 / 巡查所被特殊事件禁用。

```
说明：  
```
它们不吃热量系统，但仍吃建造、槽位、人手和特殊状态系统。

```
