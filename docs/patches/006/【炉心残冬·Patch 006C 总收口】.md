```
条件一：

```

## 4. 路线互斥口径  
誓言路线与铁腕路线永久互斥。  
规则：  
```
签署守炉誓言 → 锁定誓言路线 → 永久关闭铁腕路线
签署炉城巡令 → 锁定铁腕路线 → 永久关闭誓言路线

```
禁止：  
```
同时拥有誓言路线与铁腕路线
先签誓言再切铁腕
先签铁腕再切誓言
同时解锁终火誓约与最高戒令
选择誓言后建造巡查所
选择铁腕后建造守炉堂

```
第一条路线炉律必须二次确认。  
   
⸻

## 7. 核心建筑口径  
006C 路线核心建筑：  
```
誓言路线：守炉堂
铁腕路线：巡查所

```
核心建筑统一规则：  
```
守炉堂 / 巡查所是占槽建筑。
守炉堂 / 巡查所占普通区域槽位。
守炉堂 / 巡查所不使用特殊核心槽。
守炉堂 / 巡查所需要玩家手动 build。
守炉堂 / 巡查所不会随路线入口签署自动生成。
守炉堂 / 巡查所不会自动占槽。
守炉堂 / 巡查所占 1 槽。
守炉堂 / 巡查所上限均为 1 座。

```
路线入口只解锁建造权限：  
```
守炉誓言 → 解锁守炉堂建造权限
炉城巡令 → 解锁巡查所建造权限

```
   
⸻  
   
## 8. 建造区域口径  
守炉堂：  
```
推荐内环 / 中环。
可建区域按建筑表执行。
推荐区域不是硬性限制。

```
巡查所：  
```
推荐内环 / 中环。
可建区域：内环 / 中环 / 外环。
推荐区域不是硬性限制。

```
通用规则：  
```
除非建筑表明确写“只能建造于某区域 / 某资源点”，否则推荐区域只代表建议位置。

```
代码化方向：  
```
守炉堂：
slot_cost = 1
max_count = 1
recommended_zone = ["inner", "middle"]

巡查所：
allowed_zones = ["inner", "middle", "outer"]
recommended_zone = ["inner", "middle"]
slot_cost = 1
max_count = 1

```
   
⸻  
   
## 9. 核心建筑运行口径  
守炉堂 / 巡查所均为非生产型路线建筑。  
运行条件：  
```
建筑已建成。
建筑有人值守。
建筑处于运行状态。

```
值守规则：  
```
至少 1 名可工作人口值守即可运行。
普通儿童不可作为工作人员。
儿童辅工 / 学徒不能独立运行。
若使用儿童辅工 / 学徒辅助，至少需要成人兜底。
不按人手数量计算效率。

```
建成生效规则：  
```
守炉堂 / 巡查所建成后，若满足值守条件，当日即可视为运行。
不需要 D+1 天等待。
不新增施工后等待日。
不新增修缮期。

```
核心建筑未运行时：  
```
对应路线主动行动不可用。
对应路线终章炉律前置检查不满足。
已签署的永久炉律不失效。
最终条失败轴改写不因此反复开关。

```
   
⸻

## 13. 主动行动冷却  
誓言路线主动行动冷却：  
```
守炉宣誓：3 天
敲响悼亡钟：4 天
共食誓餐：4 天
留城劝诫：3 天

```
铁腕路线主动行动冷却：  
```
巡查：3 天
发布公告：3 天
临时留置：5 天
登记审查：3 天

```
冷却记录方式：  
```
使用 next_available_day。
若第 D 天使用行动，则下一次最早在第 D + 冷却天数 天使用。

```
行动失败规则：  
```
不消耗行动冷却。
不改变状态。
不结算效果。
不结算代价。
不消耗熟食。
不降低旧城派人数。
不改变信任 / 恐慌。

```
   
⸻  
   
## 14. 006C 签署冷却  
006C 炉律签署冷却：  
```
2 天

```
规则：  
```
若玩家在第 D 天签署 006C 炉律，
下一条 006C 炉律最早可在第 D+2 天签署。

```
006A 与 006C 冷却独立：  
```
同一天最多签署 1 条 006A 社会适配炉律。
同一天最多签署 1 条 006C 誓言 / 铁腕炉律。
若两边冷却都可用，同一天可以各签署 1 条。

```
禁止：  
```
同一天连续签署多条 006C 炉律。
用 006A 冷却阻塞 006C。
用 006C 冷却阻塞 006A。

```

## 16. 失败轴改写口径  
终火誓约生效后：  
```
信任降至 0 时，不再直接失败。
信任数值仍存在。
信任仍继续计算、显示并被后续系统读取。
恐慌失败仍然有效。

```
最高戒令生效后：  
```
恐慌达到上限时，不再直接失败。
恐慌数值仍存在。
恐慌仍继续计算、显示并被后续系统读取。
信任失败仍然有效。

```
通用规则：  
```
失败轴改写 ≠ 数值作废。
失败轴改写 ≠ 停止结算。
失败轴改写 ≠ 从系统中删除信任 / 恐慌。

```
信任 / 恐慌仍可用于：  
```
事件触发
路线行动效果判断
旧城派相关判断
居民状态判断
终局评价
结局标签
旁白文案
其他后续 Patch 的系统接口

```

## 17. 旧城派接口  
006C 不包含旧城派完整事件系统。  
006C 只提供两个接口行动：  
```
誓言路线：留城劝诫
铁腕路线：登记审查

```
二者机制对称：  
```
均在旧城派危机激活且旧城派人数 > 0 时可用。
均固定降低旧城派人数。
均不使用动态倍率。
旧城派人数最低降至 0，不可为负数。
具体降低数值由后续数值表确认。

```
006C 不处理：  
```
旧城派低于多少人会解散
旧城派多少人会部分出走
旧城派多少人会大规模出走
出走造成多少信任下降 / 恐慌上升
出走人口如何扣除
是否联动南烬城

```

## 21. 代码窗实现顺序建议  
建议代码窗按以下顺序实现：  
```
1. 新增 006C 页面开放状态。
2. 接入第 30 天 + 006A 8 条开放、第 35 天保底开放。
3. 接入路线选择与互斥锁定。
4. 接入 006C 签署冷却 next_available_day。
5. 接入守炉堂 / 巡查所建造权限解锁。
6. 接入守炉堂 / 巡查所占槽建筑状态。
7. 接入核心建筑运行判断。
8. 接入主动行动可用性判断。
9. 接入主动行动独立冷却。
10. 接入终章前置检查。
11. 接入失败轴改写状态。
12. 接入面板显示与失败反馈。

```
不建议代码窗先做：  
```
具体信任 / 恐慌数值平衡
旧城派完整事件
终局评价权重
建筑成本最终表

```
原因：  
