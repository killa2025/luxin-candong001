# 【炉心残冬·Patch 006B-1 干净封存版】  
主题：工程科技 B1 —— 研究系统与科技层级 状态：干净封存 归属：006B 工程科技  
   
⸻  
   
## 一、合并后的干净版正文  
**1. 定位**  
006B 处理工程科技系统。  
它解决的问题是：  
```
炉城如何通过研究，让供暖、采集、医疗、食物、住房与生产能力逐步增强。

```
006B 不属于社会适配城约。  
006B 不使用：  
```
社会适配城约冷却
精神 / 秩序冷却
后期令次数
签署炉律

```
006B 使用：  
```
research <科技名>
研究 <科技名>

```
作为研究入口。  
说明：  
* 006A 是社会制度线。  
* 006B 是工程科技线。  
* 玩家可以在社会城约冷却期间继续研究科技。  
* 玩家也可以在研究科技期间签署城约，只要城约冷却已结束。  
   
⸻  
   
**2. V1 科技资源边界**  
V1 工程科技不引入蒸汽核心。  
不加入：  
```
蒸汽核心
机器人
自动工厂
义肢工厂
探索队科技
灯塔 / 侦察队科技
外部遗物科技

```
说明：  
蒸汽核心主要服务探索队、机器人、温室高级建造、自动化工业等系统。 这些内容放入 V2.0，不进入 V1。  
V1 科技消耗只使用既有资源。  
科技研究本身主要消耗：  
```
木材
钢铁
研究时间
工程师人手

```
必要时，具体科技或科技效果可以间接涉及：  
```
煤炭
熟食 / 生食
运行人手
建筑维护

```
但 006B-1 不为具体科技补数值。  
   
⸻  
   
**3. 科技层级**  
006B 使用 T0-T5 科技层级。  
```
T0：开局可研究的基础科技
T1：一级科技
T2：二级科技
T3：三级科技
T4：四级科技
T5：终局科技

```
规则：  
```
T0 科技开局可研究。
T0 科技仍然需要研究所运行。
T1-T5 科技需要先完成对应层级门槛科技，开放该 Tier 的研究选项。

```
层级门槛科技只开放下一层科技的研究资格。  
不是：  
```
直接完成下一 Tier 科技
直接获得下一 Tier 科技效果
免费解锁下一 Tier 全部建筑或功能

```
   
⸻  
   
**4. 层级门槛科技**  
层级门槛科技如下：  
```
绘图板：T0 科技，完成后开放 T1 科技的研究选项。
制图仪：T1 科技，完成后开放 T2 科技的研究选项。
机械计算器：T2 科技，完成后开放 T3 科技的研究选项。
差分机：T3 科技，完成后开放 T4 科技的研究选项。
自动成型机：T4 科技，完成后开放 T5 科技的研究选项。

```
   
⸻  
   
**4.1 绘图板**  
```
科技名：绘图板
类型：层级门槛科技
所属层级：T0
效果：开放 T1 科技研究选项
前置：无

```
说明：  
绘图板代表炉城开始系统整理图纸，不再只靠拆废铁和拍脑袋修城。

类型：层级门槛科技

前置：绘图板

```
   
⸻  
   
**4.3 机械计算器**  
```
科技名：机械计算器
类型：层级门槛科技
所属层级：T2
效果：开放 T3 科技研究选项
前置：制图仪

```
   
⸻  
   
**4.4 差分机**  
```
科技名：差分机
类型：层级门槛科技
所属层级：T3
效果：开放 T4 科技研究选项
前置：机械计算器

```
   
⸻  
   
**4.5 自动成型机**  
```
科技名：自动成型机
类型：层级门槛科技
所属层级：T4
效果：开放 T5 科技研究选项
前置：差分机

```
说明：  
自动成型机不是机器人科技。 它只代表炉城拥有更高等级的工程制图、模具、零件标准化能力。  
   
⸻  
   
**5. 研究资格与科技完成分离**  
开放某一 Tier 后，该 Tier 内科技只是变成：  
```
可研究
或
前置未满足但可预览

```
并不会自动完成。  
正确流程：  
```
1. 研究 绘图板
2. 绘图板完成
3. T1 科技列表变为可研究 / 可选择
4. 玩家选择研究 住房保温 I
5. 扣除住房保温 I 的材料
6. 等待住房保温 I 的研究天数完成
7. 住房保温 I 效果生效

```
错误流程禁止：  
```
1. 研究 绘图板
2. 绘图板完成
3. 全部 T1 科技自动完成

```
代码语义：  
```
effect_type = unlock_research_tier
effect_value = T1

```
含义：  
```
允许玩家开始研究 T1 科技。

```
不应写成：  
```
effect_type = complete_tier
effect_value = T1

```
也不应写成：  
```
effect_type = unlock_all_tech_effects
effect_value = T1

```
一句话：  
```
绘图板给你菜单，不给你满汉全席。

```
   
⸻  
   
**6. 科技状态**  
每项科技至少应有以下状态：  
```
locked：所属 Tier 未开放，不能研究。
available：所属 Tier 已开放，前置满足，可以研究。
blocked：所属 Tier 已开放，但具体前置未满足。
researching：正在研究。
completed：研究完成，效果已生效。

```
说明：  
* 层级门槛科技只会把一批科技从 locked 推向 available 或 blocked。  
* 层级门槛科技不会把它们直接变成 completed。  
* 具体科技必须逐项研究完成后，才会生效。  
   
⸻  
   
**7. 科技分类**  
006B 科技按功能分为四大类：  
```
006B-1：研究系统与科技层级
006B-2：供暖与保温科技
006B-3：资源采集与生产科技
006B-4：食物、医疗与住房科技

```
说明：  
006B-1 只定义研究规则与科技层级。 具体科技项目从 006B-2 开始分批封存。  
   
⸻  
   
**8. 研究所规则**  
研究需要研究所与工程师。  
研究所基础口径：  
```
研究所是开局可建基础建筑。
研究所不需要科技解锁。
研究所不需要炉律解锁。
玩家建造研究所后，才能使用 research 命令。
研究所是 V1 科技系统入口。

```
说明：  
这样避免出现：  
```
研究科技需要研究所；
建研究所又需要科技；
科技系统原地死锁。

```
运行规则：  
```
研究所需要工程师运行。
研究所至少需要 1 名工程师在岗。
研究所低温时停工或研究速度下降。
研究所默认不可 heat。
研究所不列入默认 can_heat 建筑。

```
若后续需要研究所可应急加热，必须在建筑表或后续 Patch 中单独添加：  
```
can_heat: true

```
工程学徒规则：  
```
工程学徒可以辅助研究速度。
工程学徒不能替代工程师启动研究所。

```
说明：  
学徒只能提供辅助，不是完整工程师身份。 这与 006A 学徒成人兜底规则保持一致。  
   
⸻  
   
**9. 研究所数量规则**  
研究所数量规则：  
```
第一座研究所：解锁研究功能。
第二座研究所：提高研究速度。
第二座研究所不提供第二研究队列。
同一时间默认只能研究 1 个科技项目。

```
说明：  
V1 不做多研究队列。 避免 AI 玩家同时开三条科技线，把自己和系统一起煮成工程火锅。  
第二研究所规则：  
```
第二研究所提高研究速度。
第二研究所不提供第二研究队列。
同一时间仍然只能研究 1 个科技项目。

```
速度方向：  
```
一座满员研究所：基础研究速度。
两座正常运行研究所：研究速度提高。

```
具体数值留给试玩平衡。  
   
⸻  
   
**10. 研究队列**  
V1 默认只有一个研究队列。  
```
active_research = 当前研究项目

```
规则：  
```
没有 active_research 时，玩家可选择一个科技开始研究。
已有 active_research 时，不能开始第二个科技。
研究完成后，active_research 清空。

```
尝试重复研究时提示：  
```
研究失败：当前已有研究项目进行中。
请等待当前研究完成，或使用 cancel research 取消当前项目。

```
   
⸻  
   
**11. 研究命令**  
**11.1 开始研究**  
命令：  
```
research <科技名>
研究 <科技名>

```
执行检查：  
```
科技是否存在
科技是否已完成
科技层级是否已开放
科技前置是否满足
是否已有研究项目
研究所是否可运行
工程师是否足够
资源是否足够

```
成功后：  
```
扣除研究资源
设置 active_research
显示预计完成时间

```
反馈方向：  
```
研究开始：绘图板

工程师们把冻硬的木桌擦干，重新摊开旧图纸。
预计耗时：X 天。

```

## 1. 补到第 11.1「开始研究」执行检查后  
研究科技前，必须向玩家显示该科技的研究成本与研究耗时。  
显示内容至少包括：  
```
木材消耗
钢铁消耗
研究耗时
前置科技
当前是否满足资源条件

```
若资源不足，系统应提示缺口。  
示例方向：  
```
研究失败：资源不足。

研究“绘图板”需要：
- 木材：20
- 钢铁：10
- 研究耗时：2 天

当前缺少：
- 木材：5

```
具体成本数值由后续科技表填写；006B-1 只规定必须显示成本，不在本节补数值。

**11.2 取消研究**  
命令：  
```

取消研究

```
规则：  
```
取消研究不返还资源。
取消研究清空当前研究进度。
取消后 active_research 置空。

说明：  
防止玩家反复切换研究项目，白嫖进度或试探成本。 只有玩家主动取消研究，才会清空当前研究进度。

**12. 研究进度结算**  
研究进度在每日结算时推进。  
```
end_day 时，若 active_research 存在，则推进研究进度。

```
推进条件：  
```
至少 1 座研究所正常运行。
研究所内有足够工程师。
研究所未因低温停工。
研究所未因事件停摆。

```
研究所停工规则：  
```
研究所停工时，当前研究进度保留。
研究所停工当天，研究进度不推进。
研究所恢复运行后，从原进度继续研究。

```
说明：  
低温、缺人、事件停摆不会清空研究成果。 只有玩家主动取消研究，才会清空进度。  
人手不足规则：  
```
研究所人手不足时，按工程师人数比例推进研究。
无人或低于最低运行要求时，当日不推进研究。

```
具体比例留给后续数值表。  
   
⸻  
   
**13. 研究进度小数**  
研究进度内部允许小数。  
规则：  
```
研究进度内部允许小数。
第二研究所、工程学徒辅助、人手不足等效果可以让研究进度按比例推进。
显示时可保留 1 位小数，或显示预计剩余天数。

```
示例方向：  
```
当前研究进度：2.5 / 4.0 天
预计剩余：1.5 天

```
说明：  
* 小数用于代码结算与面板提示。  
* 玩家不需要手动处理小数。  
* 研究进度小数不代表温度、资源、人口等其他系统也使用小数。  
   
⸻  
   
**14. 科技完成提交时机**  
科技完成只在每日结算阶段统一提交。  
规则：  
```
研究进度在 end_day 中推进。
若研究进度达到或超过所需天数，科技在本次 end_day 结算末尾完成。
科技完成后，再解锁对应建筑、效果或下一层研究选项。

```
禁止：  
```
禁止在日结中途立刻触发科技完成并插入其他系统结算。
禁止其他系统读取尚未提交的科技完成状态。

```
说明：  
科技完成需要有提交阶段，避免中途解锁导致结算顺序混乱。  
   
⸻  
   
**15. 科技完成后效果**  
科技完成后：  
```
加入 completed_techs
应用科技效果
从 active_research 清空
在 tech 面板显示为已完成

```
若科技解锁建筑：  
```
对应建筑加入可建造列表。

```
若科技提供被动效果：  
```
对应结算规则立即生效。

```
若科技增强命令：  
```
对应命令效果更新。

```
科技完成后可以解锁建筑，但不会自动建造建筑。  
规则：  
```
科技解锁建筑后，该建筑加入可建造列表。
玩家仍需使用 build 命令建造该建筑。

```
示例方向：  
```
完成“大型煤矿支架”：
解锁大型煤矿机相关建造资格。

但系统不会自动建造大型煤矿机。
玩家仍需执行对应 build 命令。

```
   
⸻  
   
**16. 科技前置规则**  
每个科技拥有：  
```
tier
category
prerequisites
cost
research_days
effects

```
研究检查顺序：  
```
1. 科技是否存在
2. 科技是否已完成
3. 科技层级是否开放
4. 科技前置是否完成
5. 是否已有研究项目
6. 研究所是否可运行
7. 工程师是否足够
8. 资源是否足够

```
说明：  
层级解锁只是打开该 Tier 的研究资格。 具体科技仍然可以有自己的前置。  
例如：  
```
T3 科技“炉心节煤 II”
需要：
- T3 已开放
- 已完成“炉心节煤 I”

```

**17. T0 科技规则**  
T0 科技开局可研究。  
T0 主要包括方向：  
```
基础采集效率
基础供暖改良
基础狩猎装备
基础医疗工具
基础住房保温

```
规则：  
```
T0 不需要绘图板。
T0 可以作为开局科技选择。
T0 仍然需要研究所运行。

**18. 科技效果类型**  
科技效果分为：  
```
建筑解锁
产出提高
耗煤降低
温度修正
风险降低
容量提高
人手需求降低
命令强化
结算规则修改

```
示例方向：  
```
建筑解锁：
解锁大型煤矿机相关建造资格。

产出提高：
伐木场产出提高。

耗煤降低：
炉心同档位煤耗降低。

温度修正：
医疗站有效温度提高。

风险降低：
外环作业冻伤风险下降。

容量提高：
住房容量提高。

人手需求降低：
某类建筑岗位需求下降。

```
说明：  
示例方向只说明效果类型，不写正式数值。

**19. 科技不应直接改变社会情绪**  
工程科技默认不直接改变信任 / 恐慌。  
规则：  
```
研究完成科技本身不直接提高信任。
研究完成科技本身不直接降低恐慌。
科技通过改善供暖、食物、医疗、住房、生产等系统，间接影响社会情绪。

```
例外：  
```
若后续特殊科技明确写入事件或社会影响，可单独记录。
