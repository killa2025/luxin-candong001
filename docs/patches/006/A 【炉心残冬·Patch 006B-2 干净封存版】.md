## 一、合并后的干净版正文  
**1. 定位**  
006B-2 处理炉城的供暖、保温、过载与外勤防寒科技。  
它服务以下系统：  
```
炉心档位
煤炭消耗
建筑有效温度
应急加热
过载

```
```
外勤作业寒冷风险
第七霜落终局准备

```
006B-2 不处理：  
```
复杂热网
蒸汽枢纽

蒸汽核心
机器人
住宅升级
完整住房科技
医疗效率
食物生产
资源产出

```
说明：  
* 住宅专项保温与住房升级放入 006B-4。  
* 006B-2 的“建筑保温”主要作用于工作建筑、关键建筑与可加热建筑。  
* 006B-2 不直接替代住房科技。  
* 006B-2 不新增蒸汽核心、复杂热网或自动化系统。

**5. 科技生效方式规则**  
每项科技必须标记生效方式。  
```
passive：研究完成后默认持续生效。
unlock_command：研究完成后解锁一个手动命令。
upgrade_command：研究完成后强化已有手动命令。
unlock_building：研究完成后解锁建筑。
unlock_mode：研究完成后解锁可切换模式。

```
说明：  
```
默认持续生效的科技不需要玩家手动开启。
解锁命令的科技不会自动执行，必须玩家输入命令。
强化命令的科技只改变命令效果，不会自动使用命令。

```
006B-2 中：  
```
炉心节煤 I / II、建筑保温 I / II、炉心功率稳定 I、外勤防寒装备、终局炉心稳定：
研究完成后默认持续生效。

```
```

应急加热装置：
强化 heat 命令，不自动对任何建筑使用 heat。

过载调校 / 过载稳定：
解锁或强化 overload 命令，不自动开启过载。

```

**10. C 线：炉心功率 / 过载线**

```
科技名：炉心功率稳定 I
Tier：T1
分类：供暖 / 炉心
前置：无
生效方式：passive

```
效果方向：  
```
提高炉心高档运行稳定性。
降低炉心 3 档运行时的压力或异常风险。

```
说明：  
该科技不降低煤耗。 它解决的是“炉心能不能更稳地烧高档”，不是“烧得更省”。  
   
⸻  
   
**10.2 过载调校**  
```
科技名：过载调校
Tier：T3
分类：供暖 / 过载
前置：炉心功率稳定 I
生效方式：unlock_command / upgrade_command

```
效果方向：  
```
解锁或强化过载 I。
允许玩家在炉心运行时使用或更稳定地使用 overload 1。

```
命令关联：  
```
overload 1

```
限制：  
```
炉心必须运行。
需要额外煤炭。
会增加炉心压力。
不能长期无代价开启。

```
说明：  
过载是临时顶住寒潮的工具，不是免费暖气。 玩家没煤、炉心关闭或压力过高时，不能靠过载凭空发热。  
   
⸻  
   
**10.3 过载稳定**  
```
科技名：过载稳定
Tier：T4
分类：供暖 / 过载
前置：过载调校
生效方式：upgrade_command

```
效果方向：  
```
提高过载运行稳定性。
降低过载带来的炉心压力增长。
解锁或强化 overload 2。

```
命令关联：  
```
overload 1
overload 2

```
限制：  
```
仍然需要额外煤炭。
仍然会增加炉心压力。
不能永久开启。
不能替代正常供暖科技与储煤。

```
说明：  
过载稳定是黑霜前后的高阶科技。 它让玩家有更强的临时应对能力，但不能把炉心改成永动机。

**11. D 线：应急加热**  
**11.1 应急加热装置**  
```
科技名：应急加热装置
Tier：T1
分类：供暖 / 应急
前置：无
生效方式：upgrade_command

```
效果方向：  
```
强化 heat 应急加热命令。

```
命令关联：  
```
heat <建筑名或编号>

```
V1 基础口径：  
```
heat 命令开局可用。
应急加热装置不再解锁 heat 命令。
应急加热装置是 heat 强化科技。

```
强化方向：  
```
降低 heat 成本
提高 heat 效率
提高 heat 稳定性
降低 heat 相关风险
扩大 heat 可用表现

```
具体强化数值待后续数值表确定。  
限制：  
```
只对可加热建筑有效。
目标建筑必须支持应急加热。
住宅不可使用 heat。
研究所默认不可使用 heat。
狩猎小屋不可使用 heat。
墓园、冷藏坑、仓库、火盆集会等特殊建筑不可使用 heat。
应急加热当日生效，日结后自动关闭。
需要额外煤炭或对应成本。
每日使用次数有限。

```
说明：  
* 应急加热是救急按钮。  
* 它不该成为日常无限供暖方案。  
* 建筑 can_heat 代表该建筑支持 heat，不代表研究所等默认不可 heat 建筑自动获得 heat 权限。  
* 若后续需要研究所可 heat，必须在建筑表或后续 Patch 中单独添加 can_heat: true。

**15.1 与天气系统**  
供暖科技会影响有效温度、煤耗或低温风险，但不改变天气本身。  
```
科技不会改变每日基础温度。
科技不会取消寒潮、暴雪、黑霜或第七霜落。

```
   
⸻  
   
**15.2 与建筑温度系统**
