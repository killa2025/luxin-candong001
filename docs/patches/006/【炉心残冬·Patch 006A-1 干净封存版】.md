# 【炉心残冬·Patch 006A-1 干净封存版】  
主题：社会适配城约 A1 —— 儿童 / 人力统筹线 状态：干净封存 归属：006A 社会适配城约  
   
⸻  
   
## 一、合并后的干净版正文  
**1. 定位**  
本节处理儿童相关炉律，包括：  
* 儿童是否参与劳动。  
* 儿童可参与哪些岗位。  
* 儿童保护所与学堂。  
* 医疗学徒 / 工程学徒。  
* 儿童路线对信任、恐慌、风险、岗位与结局评价的影响。  
本节不处理：  
* 医疗法。  
* 食物法。  
* 死亡处理。  
* 娱乐减压。  
* 精神 / 秩序。  
* 工程科技。  
* 后期令。  
   
⸻  
   
**2. 总分支**  
儿童线为主线互斥结构：  
```
儿童辅工线 ↔ 儿童保护线

```
一旦签署其中一条主线第一项炉律，另一条主线永久锁定。  
V1.0 不提供反悔机制。  
   
⸻  
   
**3. 儿童辅工线**  
儿童辅工线偏短期生存与铁律倾向。  
特点：  
* 更快增加可用人手。  
* 降低信任。  
* 提高恐慌。  
* 提高儿童患病、冻伤、事故与死亡风险。  
* 相关选择会进入终局评价。  
   
⸻  
   
**3.1 儿童辅工：低危岗位**  
前置：无 互斥：儿童保护线全部炉律 解锁类型：岗位权限 是否解锁建筑：否  
签署后，儿童可参与部分低危岗位。  
允许方向：  
```
食堂
温室
内环轻工
散落煤炭采集
散落木材采集
散落钢铁采集

```
限制方向：  
```
小型采煤机
小型采钢机
大型煤矿机
大型钢铁矿机
伐木场
暴雪外勤
黑霜外勤
第七霜落期间外勤
工程师岗位
研究主导岗位
高级医疗主治岗位
小酒馆工作
大赌场工作

```
散落采集规则：  
* 儿童可参与散落煤炭 / 散落木材 / 散落钢铁采集。  
* V1 不设置散落食物点。  
* 采集棚不是准入门槛。  
* 有避寒采集棚时，儿童采集风险下降。  
* 无避寒采集棚时，儿童患病 / 冻伤风险更高。  
* 暴雪、黑霜、第七霜落期间禁止或强限制儿童外勤。  
文案方向：  
孩子们将被允许进入食堂、温室与散落资源点。他们不会被送进矿机和伐木场，但炉城已经开始把小手写进每日分工表。采集棚能替他们挡一部分风雪；没有棚，风雪只能他们自己扛。  
效果方向：  
```
儿童可参与低危岗位
可用劳动力增加
信任小幅下降
恐慌小幅上升
儿童患病 / 冻伤 / 轻伤风险上升
儿童保护线永久锁定
终局评价记录儿童辅工路线

```
  
儿童辅工是开局前期的重要抉择，不是后期装饰炉律。  
开局成年人不足以同时覆盖散落煤 / 木 / 钢、研究、医疗、食堂、建造维护、狩猎等岗位。  
不签儿童辅工时，玩家必须取舍。  
签署后儿童提供有限辅助劳动力，但不能变成无限免费人手，也不能消除成人劳动力规划压力。  
儿童辅工收益与风险都进入终局摘要 / 标签。   
⸻  
   
**3.2 儿童辅工：全岗位**  
前置：儿童辅工：低危岗位 互斥：儿童保护线全部炉律 解锁类型：岗位权限 是否解锁建筑：否  
签署后，儿童可参与绝大多数普通劳动力岗位。  
新增允许方向：  
```
小型采煤机
小型采钢机
大型煤矿机
大型钢铁矿机
伐木场
其他普通劳动力岗位

```
仍然禁止：  
```
工程师岗位
研究主导岗位
高级医疗主治岗位
小酒馆工作
大赌场工作
第七霜落期间外勤

```
说明：  
* 儿童辅工：全岗位不代表儿童可以从事所有工作。  
* 儿童仍不能执行工程师主岗位。  
* 儿童仍不能执行研究主导岗位。  
* 儿童仍不能执行高级医疗主治岗位。  
* 儿童仍不能进入小酒馆 / 大赌场工作。  
* 第七霜落期间外勤仍禁止或强限制。  
文案方向：  
孩子们将不再只是在炉边递碗、搬布、照看温室。他们会被送往矿机、伐木场和更冷的工作地，像成年人一样被计入今日产出，也像成年人一样被写进事故名单。  
效果方向：  
```
儿童可参与绝大多数普通工作岗位
可用劳动力进一步增加
信任明显下降
恐慌上升
儿童患病 / 冻伤 / 事故 / 死亡风险显著上升
终局评价记录“全岗位童工”

```
   
⸻  
   
**4. 儿童保护线**  
儿童保护线偏长期稳定与余温倾向。  
特点：  
* 不立刻增加普通劳动力。  
* 降低儿童风险。  
* 提高信任。  
* 降低恐慌。  
* 消耗内环槽位、温度、人手与教育资源。  
* 后续可发展学徒路线。  
   
⸻  
   
**4.1 儿童保护所**  
前置：无 互斥：儿童辅工线全部炉律 解锁类型：正式建筑 解锁建筑：儿童保护所 是否占槽：是  
签署后，玩家可建造儿童保护所。  
附带解锁：  
```
开放学堂建筑建造资格

```
说明：  
* 签署儿童保护所后，玩家可建造儿童保护所。  
* 学堂也从此进入可建造范围。  
* 学徒效果仍需后续医疗学徒 / 工程学徒炉律决定。  
* 儿童保护所不是学堂本体。  
运行规则：  
```
儿童保护所需要工人照护即可运行。
工程师可增强保护效果，但不是强制条件。
需要较高建筑有效温度。
可应急加热。
容量有限。
推荐建在内环。

```
温度规则：  
```
儿童保护所温度要求高于普通住宅，接近医疗建筑。
有效温度过低时，保护效果下降或失效。
低温失效时，儿童患病 / 冻伤 / 恐慌保护不再完整生效。

```
文案方向：  
炉心附近将划出一块更暖的地方，孩子们会被集中安置、照看和保护。他们暂时不能为炉城带回煤炭或食物，但至少不必立刻走入冷冽的寒风。  
效果方向：  
```
解锁儿童保护所
开放学堂建造资格
儿童不参与普通劳动
儿童患病 / 冻伤 / 恐慌风险下降
信任上升
恐慌小幅下降
占用内环槽位
需要人手与较高温度
儿童辅工线永久锁定

```
   
⸻  
   
**4.2 学堂**  
解锁来源：儿童保护所 解锁类型：正式建筑 解锁建筑：学堂 是否占槽：是  
说明：  
* 学堂不是单独炉律本体。  
* 学堂是儿童保护线的后续教育建筑。  
* 没有儿童保护所路线，不开放学堂。  
* 学堂用于承接医疗学徒 / 工程学徒路线。  
运行规则：  
```
学堂必须至少 1 名工程师运行。

```
```
工人不可进入且不能替代工程师。
儿童和医疗/工程学徒也无法代替工程师。
需要较高建筑有效温度。

```
```
可应急加热。
容量有限。

```
温度规则：  
```
学堂属于准关键建筑。
有效温度过低时停课。
停课时不产生医疗学徒 / 工程学徒效果。

```
说明：  
* 学堂不是儿童保护所本体。  
* 没有工程师或温度不足时，儿童仍可被保护所照护。  
* 没有工程师或温度不足时，不能稳定获得学徒收益。  
* 学堂不归入普通工作建筑。  
* 学堂默认不可被加班日指定。  
   
⸻  
   
**5. 学徒路线**  
学徒路线必须建立在儿童保护所与学堂正常运行基础上。  
```
儿童保护所
└─ 学堂
   ├─ 医疗学徒
   └─ 工程学徒

```
医疗学徒与工程学徒互斥。  
V1.0 不设置复杂“辅助岗位”机制。  
学徒按对应建筑的普通岗位人手使用，但只能进入固定名单中的对应类型建筑。  
学徒不能进入：  
```
外环高危岗位
暴雪外勤
黑霜外勤
第七霜落期间外勤
小酒馆工作
大赌场工作

```
学徒不能替代完整工程师身份。  
部分关键工作场所仍必须至少有 1 名成人在岗。  
   
⸻  
   
**5.1 医疗学徒**  
前置：儿童保护所 + 学堂正常运行 互斥：工程学徒 解锁类型：学堂方向 / 岗位权限 是否解锁建筑：否  
可进入：  
```
医疗站
医院

```
不可进入：  
```
小型采煤机
小型采钢机
大型煤矿机
大型钢铁矿机
伐木场
外环高危岗位
暴雪外勤
黑霜外勤
第七霜落期间外勤
工程师岗位
小酒馆工作
大赌场工作

```
效果方向：  
```
医疗岗位可用人手增加
医疗压力下降
信任小幅上升
恐慌小幅下降
工程学徒永久锁定

```
文案方向：  
孩子们将在保护所里学习最基础的照护：递药、烧水、清洗布条、记录病人的名字。他们不会成为真正的医生，但能让医疗站在最拥挤的时候多撑住一点。  
   
⸻  
   
**5.2 工程学徒**  
前置：儿童保护所 + 学堂正常运行 互斥：医疗学徒 解锁类型：学堂方向 / 岗位权限 是否解锁建筑：否  
可进入：  
```
研究所

```
不可进入：  
```
小型采煤机
小型采钢机
大型煤矿机
大型钢铁矿机
伐木场
外环高危岗位
暴雪外勤
黑霜外勤
第七霜落期间外勤
高级工程师主岗位
小酒馆工作
大赌场工作

```
说明：  
* V1 删除修造工坊作为正式建筑。  
* 工程学徒不再进入修造工坊。  
* 其他建筑若要允许工程学徒进入，必须在建筑表中单独标记。  
效果方向：  
```
工程相关岗位可用人手增加
研究压力下降
信任小幅上升
恐慌小幅下降
医疗学徒永久锁定

```
文案方向：  
孩子们将在学堂里学习图纸、工具和炉心结构。他们不会被送进矿机，也不会独自负责炉心，但他们能在研究所里替大人多争回一点明天。  
   
⸻  
   
**5.3 学徒岗位成人兜底规则**  
V1.0 中，医疗学徒 / 工程学徒按对应建筑的普通岗位人手使用，但部分关键工作场所仍必须至少有 1 名成人在岗。  
适用方向：  
```
医疗站
医院
研究所
学堂
其他标记为“必须成人兜底”的关键建筑

```
规则：  
```
若玩家只向这些建筑分配学徒，而没有分配任何成人，该分配无效。
系统提示该建筑至少需要 1 名成人在岗。

```
字段方向：  
```
building.requires_adult_anchor = true
if apprentices_assigned > 0 and adult_assigned == 0:
    reject assignment

```
首次提示文案：  
```
分配失败：至少需要一名成人在岗。

孩子们已经学会递药、抄写图纸、整理工具，但这里不能只交给他们。
孩子的肩膀无法扛起全部重任。

请至少分配 1 名成人后再安排学徒。

```
后续简短提示：  
```
分配失败：该建筑至少需要 1 名成人在岗，学徒不能独立运行。

```
说明：  
* 该规则不引入复杂“辅助岗位”系统。  
* 学徒仍按对应建筑普通人手计入。  
* 只增加“至少 1 名成人兜底”的合法性检查。  
* 首次错误分配显示完整文案，之后只显示短提示。  
   
⸻  
   
**6. 儿童与娱乐建筑规则**  
无论是否签署儿童辅工，以下人口均不可被分配到小酒馆 / 大赌场工作：  
```
儿童
儿童辅工
医疗学徒
工程学徒

```
但小酒馆 / 大赌场的全城娱乐效果默认影响所有居民，包括儿童。  
说明：  
* 儿童不能成为娱乐建筑岗位人手。  
* 儿童可以受到全城娱乐减压效果影响。  
* 这不代表儿童参与小酒馆 / 大赌场工作。  
   
⸻  
   
**7. 儿童风险规则**  
V1.0 不使用每日随机事故概率。  
儿童风险使用风险等级与固定事件处理。  
方向：  
```
儿童低危岗位：风险略高于成人。
儿童无棚散落采集：额外增加患病 / 冻伤风险。
儿童全岗位高危作业：风险显著高于成人。
暴雪 / 黑霜 / 第七霜落：禁止儿童外勤或进入极高风险限制。

```
儿童事故可使用固定触发事件。  
示例：  
```
首次签署“儿童辅工：全岗位”后，
若儿童参与高危岗位，
触发一次“孩子受伤了”事件。

```
事件方向：  
```
让他休息：
- 儿童可用劳动力 -1，持续 1 天
- 信任微微上升
- 恐慌微微下降

继续工作：
- 不损失当日劳动力
- 无正面效果
- 记录冷酷选择

```
代码仍按人口池处理，不追踪具体儿童。  
   
⸻  
   
**8. 防漏洞规则**  
儿童不能被设计成“故意消耗人口降低难度”的漏洞。  
规则：  
```
儿童患病、重症、伤残期间仍消耗食物、住房和医疗容量。
儿童死亡造成的信任下降与恐慌上升同成人死亡。
儿童高危劳作与儿童死亡会记录进终局评价。

```
终局标签方向：  
```
儿童高危劳作
儿童外环死亡
全岗位童工

```
说明：  
儿童死亡不额外放大即时社会惩罚，以继承“成人死亡与儿童死亡对信任 / 恐慌影响同幅度”的规则。 但相关选择会影响结局字幕与路线评价。  
   
⸻  
   
**9. 学徒人数限制**  
不是所有儿童都会自动变成学徒。  
学徒数量受以下条件限制：  
```
儿童保护所容量
学堂容量
学堂是否有工程师运行
学堂温度是否达标
当前儿童人数
医疗学徒 / 工程学徒路线是否已选择

```
学徒不能进入外环高危岗位，也不能替代完整工程师身份。  
一些建筑至少要有一名成人在场。  
   
⸻  
   
**10. 初始 laws 面板显示**  
开局 laws 面板必须显示可签署的初始社会适配炉律。  
006A-1 原初始列表视为示例，不是完整总表。  
初始可选方向包括：  
```
儿童辅工：低危岗位
儿童保护所
加班日
基础医疗法
火盆集会
墓园
冷藏坑

```
显示规则：  
```
同一时间只能签署一条社会适配城约。
签署后进入社会适配城约固定冷却。
冷却结束后才能签署下一条社会适配城约。
未满足前置的炉律可隐藏或显示为“未解锁”。

```
说明：  
* “应急班次”为旧名，统一改为“加班日”。  
* 墓园 / 冷藏坑开局可签署。  
* 墓园 / 冷藏坑属于死亡处理线，但需要出现在开局 laws 面板可签署项中。  
* 其他 006A 小节的可签署项由各自小节和总索引继续补全。  
   
⸻  
   
**11. 跨 Patch 修正点**  
本节对前面已封存内容形成以下继承修正：  
1. “儿童辅工：安全岗位”统一改名为“儿童辅工：低危岗位”。  
2. 儿童可参与散落煤炭 / 散落木材 / 散落钢铁采集。  
3. V1 不设置散落食物点。  
4. 采集棚是风险减免，不是儿童散落采集准入门槛。  
5. 儿童保护所温度要求提高，低温时保护效果下降或失效。  
6. 学堂温度要求提高，低温时停课，学徒效果暂停。  
7. 儿童保护所工人即可运行，工程师可增强但不强制。  
8. 学堂必须至少 1 名工程师运行。  
9. 学徒不使用复杂辅助岗位机制，按对应建筑普通岗位人手处理。  
10. 学徒岗位使用固定名单。  
11. 医疗学徒可进入医疗站 / 医院。

**12. 本节结论**  
儿童线形成明确取舍：  
```
儿童辅工线：
短期补人手，操作简单，但提高儿童风险，降低信任，提高恐慌，并影响终局评价。

儿童保护线：
短期不补普通劳动力，但保护儿童，提高信任，降低恐慌；代价是内环槽位、温度、人手、学堂与工程师资源。

```
最终分支树：  
```
儿童辅工线
├─ 儿童辅工：低危岗位
└─ 儿童辅工：全岗位

儿童保护线
└─ 儿童保护所
   └─ 学堂
      ├─ 医疗学徒
      └─ 工程学徒

```
核心原则：  
儿童可以成为炉城短期劳动力，但不能成为系统漏洞。 儿童保护可以换来长期稳定，但不会凭空解决人手压力。 学徒能帮忙，但不能独自扛起成人岗位。 玩家必须知道：把孩子写进分工表，不只是多几个劳动力，而是在结局里留下无法抹掉的一笔。
