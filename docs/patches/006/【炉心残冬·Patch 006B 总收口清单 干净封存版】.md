# 【炉心残冬·Patch 006B 总收口清单 干净封存版】  
主题：006B 工程科技总收口 状态：干净封存 归属：Patch 006B  
   
⸻  
   
## 一、006B 定位  
006B 处理：  
```
工程科技系统

```
它负责炉城通过研究逐步提升：  
```
供暖能力
保温能力
资源采集能力
资源生产能力
食物生产能力
医疗能力
住房能力
建筑 / 升级资格
终局工程容错

```
006B 不属于社会适配城约。  
006B 不使用：  
```
社会适配城约冷却
精神 / 秩序冷却
后期令次数
签署炉律

```
006B 使用：  
```
research <科技名>
研究 <科技名>

```
作为研究入口。  
说明：  
```
006A 是社会制度线。
006B 是工程科技线。
006B 可以在社会城约冷却期间继续推进。

```
   
⸻  
   
## 二、006B 文件结构  
006B 分为四个子 Patch：  
```
006B-1：研究系统与科技层级
006B-2：供暖与保温科技
006B-3：资源采集与生产科技
006B-4：食物、医疗与住房科技

```
后续实现时，以本总收口为 006B 总索引。  
   
⸻  
   
## 三、研究系统总口径  
**3.1 研究所**  
研究所是科技系统入口。  
```
研究所开局可建。
研究所不需要科技解锁。
研究所不需要炉律解锁。
玩家建造研究所后，才能使用 research 命令。

```
研究所默认不可 heat。  
```
研究所可受建筑保温影响。
研究所默认不可使用 heat。
建筑保温不等于 heat 权限。
应急加热装置不会让研究所自动获得 heat。

```
若后续需要研究所可 heat，必须单独在建筑表写：  
```
can_heat: true

```
   
⸻  
   
**3.2 科技层级**  
006B 使用 T0-T5 科技层级。  
```
T0：开局可研究的基础科技
T1：一级科技
T2：二级科技
T3：三级科技
T4：四级科技
T5：终局科技

```
层级门槛科技：  
```
绘图板：T0，开放 T1 研究选项。
制图仪：T1，开放 T2 研究选项。
机械计算器：T2，开放 T3 研究选项。
差分机：T3，开放 T4 研究选项。
自动成型机：T4，开放 T5 研究选项。

```
层级门槛科技只开放研究菜单，不自动完成整层科技。  
禁止：  
```
完成绘图板后自动完成全部 T1 科技。
完成制图仪后自动完成全部 T2 科技。
层级科技免费解锁整层效果。

```
   
⸻  
   
**3.3 研究队列**  
V1 默认只有 1 个研究队列。  
```
同一时间只能研究 1 个科技项目。
第二研究所只提高研究速度。
第二研究所不提供第二研究队列。

```
   
⸻  
   
**3.4 研究完成时机**  
科技研究进度在 end_day 中推进。  
```
科技完成只在 end_day 结算末尾提交。
科技完成后才应用效果、解锁建筑、开放升级或开放下一层研究。

```
禁止：  
```
日结中途完成科技并插入其他系统结算。
其他系统读取尚未提交的科技完成状态。

```
   
⸻  
   
**3.5 取消研究**  
```
cancel research / 取消研究

```
规则：  
```
取消研究不返还资源。
取消研究清空当前研究进度。
取消后 active_research 置空。

```
   
⸻

**5.1 heat 与应急加热装置**  
最终口径：  
```
heat 开局可用。
应急加热装置是 heat 强化科技。
应急加热装置不解锁 heat 命令。
应急加热装置研究完成后不会自动加热所有建筑。
应急加热装置研究完成后不会让所有建筑获得 heat。

```
研究所默认不可 heat，不被应急加热装置覆盖。  
   
⸻  
   
**5.2 外勤防寒装备**  
旧名作废：  
```
外环防寒装备

```
统一改为：  
```
外勤防寒装备

```
效果方向：  
```
降低外勤 / 外环 / 边缘区作业寒冷相关风险。

```
不改变：  
```
暴雪停工规则
黑霜停工规则
第七霜落外勤限制
儿童外勤禁令

```
   
⸻  
   
**5.3 建筑保温**  
建筑保温不等于 heat 权限。  
```
建筑保温是被动抗寒科技。
heat 是手动命令。
建筑获得保温，不代表该建筑可以 heat。

```
研究所可受建筑保温影响，但默认不可 heat。  
   
⸻  
   
**5.4 医疗建筑保温与养护所**  
医疗建筑保温作用于：  
```
医疗站
医院
养护所

```
养护所只获得：  
```
保温收益
护理风险降低

```
不获得：  
```
治疗容量增加
治疗速度提高
分级救治资格
急救治疗能力
医疗站 / 医院类功能

```
   
⸻

```
狩猎装备

温室改良

```
   
⸻  
   
**7.1 食物线**  
狩猎装备：  
```
提高狩猎小屋产出或普通天气下狩猎稳定性。
不增加猎区数量。
不刷新猎区。
不取消暴雪、黑霜、第七霜落停工规则。

```
食堂流程改良：  
```
只提高生食加工为熟食的效率。
不凭空生产熟食。
食堂停工时无法发挥作用。
不解锁配给模式。

```
温室栽培：  
```
解锁温室建筑。
不再强制前置狩猎装备。
科技完成后不会自动建造温室。

```
温室改良：  
```
生效方式：unlock_upgrade。
只解锁“温室 → 改良温室”的逐座升级权限。
不是 passive。
不会自动强化所有温室。
每座温室仍需玩家手动 upgrade。
温室升级耗材待确认。

```
温室改良不改变：  
```
温室数量上限
温室低温停产规则
无人 / 停工 / 有效温度不足时不能生产的规则
第七霜落终局压力

```
   
⸻  
   
**7.2 医疗线**  
医疗站由 006A 的基础医疗法解锁，不由 006B-4 解锁。  
医疗工具改良：  
```
提高医疗站治疗效率方向。
不替代医疗处置炉律。
不改变分级救治。
不改变额外医疗配给。

```
医院标准化：  
```
解锁医院工程建造能力。
建造医院需要已签署基础医疗法。
建造医院需要已完成医院标准化。
二者缺一不可。

```
医院标准化不会：  
```
绕过基础医疗法
自动建造医院
改变分级救治规则

```
   
⸻  
   
**7.3 住房线**  
住宅三档：  
```
基础住宅：10 人，1 槽，不可 heat。
改良住宅：15 人，1 槽，不可 heat。
高级住宅：20 人，1 槽，不可 heat。

```
住房保温 I：  
```
提高基础住宅抗寒能力。
降低住宅低温风险 / 提高居住安全。
不增加住房容量。
不让住宅 heat。

```
改良住宅标准：  
```
Tier：T2。
前置：住房保温 I。
生效方式：unlock_upgrade。
解锁“基础住宅 → 改良住宅”的逐座升级权限。
升级耗材待确认。

```
高级住宅标准：  
```
正式加入 006B-4。
Tier：T3。
前置：改良住宅标准。
生效方式：unlock_upgrade。
只解锁“改良住宅 → 高级住宅”的容量升级。
不提供住房保温。
不允许住宅 heat。
不替代住房保温科技。
升级耗材待确认。

```
住房保温与住宅升级分离：  
```
住房保温管温度安全。
住宅升级管单座容量。
二者不能互相替代。

```

## 八、建筑与升级统一口径  
科技只提供资格，不自动完成操作。  
```
unlock_building：只解锁建造资格。
unlock_upgrade：只解锁升级资格。
passive：研究完成后自动参与结算。

```
建筑与升级：  
```
拍下后立即可运行 / 立即生效。
不新增施工期。
不新增修缮期。
不新增升级中等待。

```
玩家仍需主动执行：  
```
build <建筑>
upgrade <建筑编号> <升级目标>

```
升级材料如果未封存，统一写：  
```
升级耗材：待确认

```
代码窗不得自行猜测升级耗材。
