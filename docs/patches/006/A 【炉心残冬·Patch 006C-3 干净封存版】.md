# A 【炉心残冬·Patch 006C-3 干净封存版】  
主题：铁腕路线 归属：006C 誓言与铁腕炉约 状态：干净封存版  
   
⸻  
   
## 1. 路线定位  
铁腕路线是“誓言与铁腕炉约”的其中一条中后期路线。  
它不是单纯的暴政路线，也不是无成本稳定按钮。  
它代表的是：  
```
巡查
登记
公告
留置
纪律
以秩序和强制管控承接城市崩裂

```
铁腕路线的核心问题是：  
```
当居民不再相信守炉人、不再愿意忍耐、不再愿意留下时，
城市是否还能靠纪律、登记与强制管控维持运转？

```
铁腕路线主要用于压制恐慌、管控旧城派、阻断失控出走与维持中后期秩序。  
它能让城市继续服从，不代表居民重新信任守炉人。  
   
⸻  
   
## 2. 路线互斥  
签署第一条铁腕炉律后，誓言路线永久关闭。  
规则：  
```
签署“炉城巡令”
→ 锁定铁腕路线
→ 永久关闭誓言路线

```
签署前必须二次确认：  
```
你即将选择铁腕路线。
签署后，誓言路线将永久关闭。

confirm sign 炉城巡令

```
禁止：  
```
先签铁腕路线，再切换誓言路线。
同时拥有铁腕路线与誓言路线。
同时解锁最高戒令与终火誓约。
签署铁腕路线后再建造守炉堂。

```
   
⸻  
   
## 3. 路线结构  
铁腕路线共 6 条炉律：  
```
1. 炉城巡令
2. 晨钟点名
3. 统一公告
4. 临时留置
5. 户籍清查
6. 最高戒令

```
路线结构：  
```
炉城巡令
├─ 晨钟点名 → 统一公告
└─ 临时留置 → 户籍清查
      └─ 最高戒令

```
最高戒令是铁腕路线最终炉律。  
最高戒令不在路线中段提前开放。  
开放条件：  
```
必须已签署铁腕路线前五条炉律。

```
即必须完成：  
```
炉城巡令
晨钟点名
统一公告
临时留置
户籍清查

```
全部完成后，最高戒令才可进入可签署状态。  
说明：  
```
铁腕路线总条目不多。
最高戒令作为最终条，应要求玩家完整走完铁腕路线，而不是只完成其中一条分支后提前开放。

```
006C-4 只负责处理：  
```
006C 页面开放时间
006C 签署冷却
主动行动冷却
最终条理论最早可签时间
路线推进节奏

```
006C-4 不改变本条结构前置：  
```
最高戒令必须在铁腕路线前五条全部签署后才开放。

```

## 4. 路线代价规则  
铁腕路线的代价以信任下降、终局标签、社会压迫感与长期后果为主。  
规则：  
```
被动炉律只在签署当日结算一次性效果或代价。
签署后默认不每日持续降低信任。
签署后默认不每日持续降低恐慌。
主动行动释放时，再结算对应效果与代价。

```
代价方向：  
```
铁腕可以压制恐慌、阻断混乱、管控旧城派。
但它不能真正解决死亡、饥饿、寒冷、医疗与住房问题。
居民会逐渐依赖纪律与强制秩序维持城市。
终局评价会记录城市依靠最高戒令承接恐慌崩塌。

```
说明：  
```
铁腕路线不是每日自动减恐慌的路线。
它的收益与代价应当在签署炉律或释放行动时清楚结算。

```
   
⸻  
   
## 5. 核心建筑：巡查所  
巡查所是铁腕路线核心建筑。  
巡查所是占槽建筑。  
解锁来源：  
```
炉城巡令

```
规则：  
```
巡查所占普通区域槽位。
巡查所不使用特殊核心槽。
巡查所需要玩家手动 build。
巡查所不会在签署炉城巡令后自动生成。
巡查所不会自动占槽。
巡查所上限为 1 座。

```
建筑口径：  
```
巡查所：占 1 槽，上限 1，推荐内环 / 中环。

```
建造区域口径：  
```
推荐内环 / 中环不是硬性限制。
除非建筑表明确写“只能建造于某区域 / 某资源点”，否则推荐区域只代表建议位置。

```
巡查所可建区域按源文保留：  
```
内环
中环
外环

```
代码化方向：  
```
巡查所：
allowed_zones = ["inner", "middle", "outer"]
recommended_zone = ["inner", "middle"]
slot_cost = 1
max_count = 1

```
签署“炉城巡令”后：  
```
铁腕路线被锁定。
誓言路线永久关闭。
解锁巡查所建造权限。
不自动建成巡查所。
不自动占用建筑槽位。

```
巡查所不做区域范围效果。  
```
巡查所建成并运行后，作用于全城铁腕路线。
不存在“建在内环 / 中环 / 外环”导致路线范围不同的问题。
巡查所所在区域只影响普通槽位占用。

```
可用命令方向：  
```
build 巡查所 <区域>

```
说明：  
```
巡查所是路线核心建筑，不是炉旁行动效果。
巡查所占槽。
巡查所需要手动建造。
巡查所未建成或未运行时，对应主动行动不可用。

```
   
⸻  
   
## 6. 巡查所运行规则  
巡查所是非生产型路线核心建筑，不按人手数量计算效率。  
运行条件：  
```
巡查所已建成。
巡查所有人值守。
巡查所处于运行状态。

```
值守条件：  
```
至少 1 名可工作人口值守即可运行。
普通儿童不可作为工作人员。
儿童辅工 / 学徒不能独立运行。
若使用儿童辅工 / 学徒辅助，至少需要成人兜底。

```
核心判断：  
```
是否已建成。
是否有人值守。
是否满足成人兜底规则。
是否处于运行状态。

```
巡查所运行特殊口径按原文保留：  
```
巡查所不受有效温度影响。
第七霜落期间仍可运行。
炉心关闭不额外影响巡查所。
只有无人分配 / 无人值守时，巡查所停工或未运行。

```
巡查所未建成、无人值守、停工或未运行时（或被后续特殊事件明确禁用时）：  
```
巡查不可用。
发布公告不可用。
临时留置不可用。
登记审查不可用。
最高戒令前置检查不满足。

```
```
已签署的永久炉律不失效。

```
永久炉律规则：  
```
已签署的永久炉律不会因为巡查所未建成、停工或无人值守而失效。
巡查所运行状态只影响主动行动与最高戒令前置检查，不回滚已经签署的炉律。

```
说明：  
```
巡查所运行状态主要影响主动行动可用性。
巡查所未运行不撤销已经签署的永久炉律。

```

这些内容不是普通建筑，不占槽。  
设施 / 承接物列表：  
```
公告墙
炉心登记牌
执勤岗
巡查铃

```
规则：  
```
由对应炉律启用或记录。
不使用 build 命令。
不占建筑槽位。
不需要选择建造区域。
不进入普通建筑列表。
可在 laws、city 或 006C 附属信息中显示为“炉旁设施 / 行动承接物”。

```
显示示例：  
```
炉旁设施：
- 公告墙：已启用
- 炉心登记牌：已启用
- 执勤岗：未启用
- 巡查铃：未启用

```
说明：  
```
巡查所是占槽核心建筑。
公告墙、炉心登记牌、执勤岗、巡查铃不是占槽建筑。
不占槽不等于隐藏。
由炉律启用不等于没有状态。
炉旁设施必须能在面板或附属信息中被玩家看到。

```

## 12. 炉律五：户籍清查  
前置：  
```
临时留置

```
类型：  
```
旧城派接口炉律

```
解锁：  
```
炉旁设施 / 行动承接物：炉心登记牌
主动行动：登记审查

```
效果方向：  
```
允许通过登记与审查处理旧城派危机。

```
主动行动：登记审查  
```
需要巡查所已建成并处于运行状态。
旧城派危机已激活时可用。
旧城派人数 > 0 时可用。
使用后固定降低旧城派人数。
旧城派人数最低降至 0，不可为负数。

```
机制对称规则：  
```
登记审查与誓言路线的留城劝诫机制对称。
二者均固定降低旧城派人数。
二者降低数值相同。
二者均不使用动态倍率。
具体降低数值由后续数值表统一确定。

```
边界：  
```
不一次性清空旧城派。
不改变旧城派危机的触发。
不改变旧城派每日增长规则。
旧城派最终出走后果由旧城派事件系统单独处理。
炉心登记牌不占建筑槽位。
登记审查作为主动行动不占建筑槽位。

```
说明：  
```
铁腕路线处理旧城派的方式，是让他们不能轻易组织和离开。
不是让他们重新相信炉城。

```

## 13. 炉律六：最高戒令  
类型：  
```
铁腕路线最终炉律

```
最高戒令是终章炉律，不是建筑，不占槽。  
前置：  
```
必须已签署炉城巡令。
必须已签署晨钟点名。
必须已签署统一公告。
必须已签署临时留置。
必须已签署户籍清查。
必须已建成巡查所。
巡查所必须处于运行状态。
必须满足 006C-4 中规定的路线签署冷却与推进节奏。

```
核心效果：  
```
恐慌达到上限时，不再直接失败。
因恐慌爆满或过高触发的失败倒计时不再启动。
若签署前已存在对应恐慌危机倒计时，则终止或转为非失败警告。

```
签署成功后：  
```
最高戒令的恐慌失败轴改写成为永久炉律效果。

```
```
之后即使巡查所无人值守、未运行或被后续特殊事件明确禁用，也不会取消最高戒令。

```
仍然有效的失败条件：  
```
存活人口 = 0 仍然失败。
炉心崩毁仍然失败。
信任 = 0 且未被终火誓约承接，仍然失败。
资源崩溃造成的死亡链仍然有效。

```
  
说明：    
  
最高戒令只改写恐慌失败轴。  
最高戒令不免疫信任、死亡、饥荒、寒冷、炉心崩毁。  
第七霜落终局验收不作为 006C 新增硬失败条件；低综合分、多系统崩坏、低于城市连续性参考线只进入终局状态评级。  
  
status 显示方向：  
```
恐慌已失控，最高戒令正在压住其后果。
信任危机仍然有效。

```
边界：  
```
不免疫信任失败。
不免疫死亡。
不免疫饥荒。
不免疫寒冷。
不免疫旧城派全部后果。
不等于无条件胜利。
不占建筑槽位。
不要求新建额外终章建筑。

```
说明：  
```
最高戒令不是让居民不再恐惧。
它是让城市在“恐惧已经压不住”之后，仍然被强制秩序锁住。

```
   
⸻  
   
## 14. 最高戒令后的恐慌运算规则  
最高戒令只改写恐慌失败轴的“直接失败判定”，不删除恐慌数值本身。  
最高戒令生效后：  
```
恐慌仍然继续计算。
恐慌仍然继续显示。
恐慌仍然可以继续上升、维持、下降。
恐慌仍然可以被其他系统读取。

```
规则：  
```
恐慌达到上限时，不立刻触发恐慌爆满失败。
但恐慌值仍然保留。
恐慌仍参与事件、行动、旧城派判断、终局评价和后续系统接口。

```
说明：  
```
失败轴改写 ≠ 恐慌数值作废。
失败轴改写 ≠ 停止恐慌结算。
失败轴改写 ≠ 从系统中删除恐慌状态。

```
恐慌后续仍可用于：  
```
事件触发
路线行动效果判断
旧城派相关判断
居民状态判断
终局评价
结局标签
旁白文案
其他后续 Patch 的系统接口

```
结论：  
```
最高戒令只改变“恐慌是否因此直接导致游戏失败”。
不改变恐慌作为全局城市状态继续运算、显示和被读取的事实。

```
   
⸻  
   
## 15. 被动炉律结算规则  
铁腕路线被动炉律只在签署时结算一次性效果或代价。  
```
炉城巡令：签署时开启铁腕路线，永久关闭誓言路线，解锁巡查所建造权限，启用基础巡查承接物。
晨钟点名：签署时解锁或强化日常秩序方向。
统一公告：签署时解锁公告墙与发布公告行动。
临时留置：签署时解锁临时留置行动。
户籍清查：签署时解锁炉心登记牌与登记审查行动。

## 16. 主动行动清单  
铁腕路线主动行动共 4 个：  
```

临时留置
登记审查

通用规则：  
```

使用后进入主动行动冷却。

```
主动行动释放时结算固定效果，不使用复杂动态倍率。  
主动行动效果方向：  
```

临时留置：固定降低恐慌，固定降低信任。
登记审查：旧城派危机激活且旧城派人数 > 0 时可用，固定降低旧城派人数。

```
说明：  
```
铁腕路线的主动行动不根据当前信任、恐慌、旧城派人数进行复杂倍率计算。
具体固定数值由后续数值表统一确定。

```

## 17. 主动行动边界  
主动行动不会因为炉城巡令已签署而自动执行。  
主动行动仍需满足：  
```
铁腕路线已选择。
对应行动已由炉律解锁。
巡查所已建成。
巡查所处于运行状态。
行动不在冷却中。
满足行动目标条件。

```
炉城巡令已签署 ≠ 巡查所已建成。 巡查所已建成 ≠ 主动行动自动执行。 巡查所未运行时，铁腕主动行动不可用。  
玩家仍需手动使用主动行动。
