# A 【炉心残冬·Patch 006A-3 干净封存版】  
主题：社会适配城约 A3 —— 死亡处理线 状态：干净封存 归属：006A 社会适配城约  
   
⸻  
   
## 一、合并后的干净版正文  
**1. 定位**  
本节处理死者安置与遗体处理规则。  
它解决的问题不是“尸体放哪”，而是：  
```
死者在炉城里，是否仍被当作人。

```
本节包括：  
* 墓园线。  
* 冷藏坑线。  
* 未处理遗体。  
* 追思行动。  
* 遗体利用分支。  
* 死亡处理对信任、恐慌、医疗、食物与结局评价的影响。  
本节不处理：  
* 独立火化场。  
* 完整固定事件文本。  
* 具体信任 / 恐慌数值。  
* 具体遗体处理容量。  
* 具体医疗 / 温室加成数值。  
   
⸻  
   
**2. 分支结构**  
死亡处理线为主线互斥结构：  
```
墓园线 ↔ 冷藏坑线

```
一旦签署其中一条主线第一项炉律，另一条主线永久锁定。  
V1.0 不提供反悔机制。  
完整分支：  
```
墓园线
└─ 追思仪式

冷藏坑线
├─ 器官利用
└─ 灰烬肥料

```
冷藏坑下层分支互斥：  
```
器官利用 ↔ 灰烬肥料

```
   
⸻  
   
**3. 火化场旧口径作废**  
V1 不保留独立火化场建筑。  
旧“火化 / 快速清理”路线不再作为独立路线存在。  
V1 死亡处理主线统一为：  
```
墓园线 ↔ 冷藏坑线

```
旧“火化 / 快速清理”功能并入冷藏坑路线的后续炉律或文案中，不保留独立火化场建筑。  
作废内容：  
```
火化场作为独立建筑
火化 / 快速清理路线作为独立于冷藏坑的路线
墓园线 ↔ 火化场线
火化场占用外环 / 边缘区槽位
火化场进入 V1 正式建筑表

```
   
⸻  
   
**4. 开局显示与首次死亡提示**  
墓园与冷藏坑开局即可在 laws 面板中预览和签署。  
006A-1 的初始 laws 列表视为示例，不是完整总表。 V1 开局 laws 面板可签署项应包含：  
```
墓园
冷藏坑

```
设计目的：  
* 玩家能提前知道死亡处理路线存在。  
* 玩家可以选择提前签署，避免首次死亡后措手不及。  
* 玩家可以选择保留社会城约冷却，应对死亡风险。  
首次死亡发生时，若玩家尚未签署墓园或冷藏坑，系统弹窗强提醒。  
首次死亡提示文案方向：  
```
第一具遗体被抬到了炉心外。

城市还没有决定如何安放死者。
你可以继续临时停放，但未处理遗体会让恐慌上升、信任下降。

可签署炉律：
- 墓园
- 冷藏坑

```
   
⸻  
   
**5. 默认状态：临时停尸**  
在玩家尚未选择死亡处理路线前，死亡不会卡死系统。  
规则：  
```
死亡发生后，遗体进入“未处理遗体”池。
若尚未签署墓园或冷藏坑，遗体临时停放。
未处理遗体越多，恐慌越高，信任越低。

```
未处理遗体属于社会风险源，不是资源。  
未处理遗体不会阻止游戏继续运行，但会逐渐形成社会压力。  
   
⸻  
   
**6. 墓园线**  
墓园线偏余温、人性与社会稳定。  
**6.1 墓园**  
前置：无 互斥：冷藏坑线全部炉律 解锁类型：正式死亡处理建筑 是否占槽：是  
建筑规则：  
```
墓园：1 槽
推荐区域：外环（边缘区为显示别名，不作为独立代码区域）

```
```
上限：1 座
不需要大量常驻人手
用于正式安葬死者
降低死亡社会冲击

```
签署后：  
```
解锁墓园建筑。
玩家可建造墓园。
墓园建成前，遗体仍可能处于临时停尸 / 未处理状态。

```
效果方向：  
```
允许正式安葬死者
降低死亡造成的信任下降
降低死亡造成的恐慌上升
大量死亡时社会崩溃速度减缓
占用槽位
处理效率较低
不可进行遗体利用
提升余温倾向结局评价

```
签署瞬间影响方向：  
```
信任：微微上升
恐慌：无变化

```
文案方向：  
```
炉城将在风雪中划出一片安静的地方。
死者不会再被堆在临时棚下，他们会有名字、有位置，也有最后一块不被炉心账本吞掉的土地。

```
说明：  
墓园必须提供明确社会收益，不能只是“好看但没产出”。 它的价值是降低死亡冲击、稳定信任与恐慌。  
   
⸻  
   
**6.2 追思仪式**  
前置：墓园 类型：社会安抚行动 位置：社会行动面板 / laws 附属行动面板 是否解锁建筑：否  
追思仪式不是纯被动效果，而是带冷却的可点击行动。  
命令方向：  
```
举行追思

```
可兼容命令：  
```
ritual memorial

```
使用条件：  
```
已选择墓园路线。
墓园已建成并正常运行。
追思行动不在冷却中。

```
效果方向：  
```
信任小幅上升
恐慌小幅下降
进入冷却

```
签署瞬间影响方向：  
```
信任：微微上升
恐慌：无变化

```
显示规则：  
```
追思仪式冷却中时，在社会行动面板显示剩余冷却。
冷却结束后，可再次举行追思。
墓园未运行时不可使用。

```
失败提示方向：  
```
追思失败：墓园尚未正常运行，死者还没有一个能被念出名字的地方。

```
文案方向：  
```
人们会在炉火熄暗前念出死者的名字。
这不会让他们回来，但至少能提醒活人：他们不是被悄悄从账本上擦掉的数字。

```
说明：  
追思仪式效果必须较小，避免墓园线变成无限刷信任工具。 它的定位是稳定社会情绪，不是强力翻盘按钮。  
   
⸻  
   
**7. 冷藏坑线**  
冷藏坑线偏效率、铁律与遗体利用。  
**7.1 冷藏坑**  
前置：无 互斥：墓园线全部炉律 解锁类型：正式死亡处理建筑 是否占槽：是  
建筑规则：  
```
冷藏坑：1 槽
推荐区域：外环（边缘区为显示别名，不作为独立代码区域）

```
```
上限：1 座
不需要大量常驻人手
用于快速保存遗体
解锁遗体利用分支

```
签署后：  
```
解锁冷藏坑建筑。
玩家可建造冷藏坑。
冷藏坑建成前，遗体仍可能处于临时停尸 / 未处理状态。

```
效果方向：  
```
快速集中保存遗体
占地少
处理效率高
信任下降
恐慌上升
解锁遗体利用分支
终局评价更冷

```
签署瞬间影响方向：  
```
信任：小幅下降
恐慌：小幅上升

```
文案方向：  
```
风雪不会让遗体腐坏，只会让他们更安静。
炉城将在边缘开凿冷藏坑，把死者集中保存。这里没有墓碑，只有编号和下一步用途。

```
说明：  
冷藏坑不能是纯收益路线。 它省地、省人手、效率高，但必须带来信任与恐慌代价。  
   
⸻  
   
**7.2 器官利用**  
前置：冷藏坑 互斥：灰烬肥料 解锁类型：遗体利用功能 / 医疗增益 是否解锁建筑：否  
前置条件：  
```
需要冷藏坑路线。
需要冷藏坑遗体储备或遗体处理能力。

```
效果方向：  
```
医疗建筑治疗效率提高
重症死亡压力下降
可能降低伤残恶化压力
需要冷藏坑遗体储备或遗体处理能力
终局评价记录“遗体医疗利用”

```
签署瞬间影响方向：  
```
信任：无变化
恐慌：无变化

```
文案方向：  
```
有些死者还能救活人。
这句话太正确了，正确到没有人愿意在饭前听见。

```
说明：  
器官利用不使用复杂概率。 它作为医疗系统增益存在，具体数值留给后续平衡表。 签署瞬间不直接改变信任 / 恐慌，但终局评价和后续事件可记录该路线选择。  
   
⸻  
   
**7.3 灰烬肥料**  
前置：冷藏坑 互斥：器官利用 解锁类型：遗体利用功能 / 温室增益 是否解锁建筑：否  
前置条件：  
```
需要冷藏坑路线。
需要冷藏坑遗体储备或遗体处理能力。
需要温室才能发挥主要效果。

```
效果方向：  
```
温室产出提高
食物压力下降
信任下降
恐慌上升
需要冷藏坑遗体储备或遗体处理能力
终局评价记录“遗体农业利用”

```
签署瞬间影响方向：  
```
信任：大幅下降
恐慌：小幅上升

```
文案方向：  
```
温室需要养分。
炉城没有肥沃土地，也没有多余牲畜。只有一些已经不再喊冷的人，还能让活人多吃一口。

```
说明：  
灰烬肥料是食物压力缓解路线。 它不能与器官利用同时签署，避免冷藏坑线同时获得医疗与食物双收益。  
旧火化场不作为独立建筑保留。 灰烬肥料属于冷藏坑路线下的遗体利用功能，不代表恢复独立火化场路线。  
   
⸻  
   
**8. 防漏洞规则**  
死亡处理线必须防止玩家把死亡变成正收益生产线。  
规则：  
```
任何死亡仍正常造成信任下降与恐慌上升。
墓园只能降低死亡社会冲击，不能消除死亡代价。
冷藏坑与遗体利用不能抵消死亡本身的社会冲击。
器官利用 / 灰烬肥料收益必须小于主动制造死亡带来的综合损失。
儿童死亡、外环高危死亡、过劳死亡等仍记录终局标签。

```
说明：  
死者可以被安葬，也可以被利用。 但死亡不能成为资源生产策略。  
   
⸻  
   
**9. 代码记录方向**  
V1.0 不追踪每具遗体身份。  
建议记录：  
```
death_path = none / graveyard / cold_pit
death_sub_law = none / memorial / organ_use / ash_fertilizer
unhandled_dead = 整数
buried_dead = 整数
stored_dead = 整数
memorial_cooldown = 整数

```
日结方向：  
```
有人死亡 → 未处理遗体增加
有墓园 → 按墓园处理能力安葬
有冷藏坑 → 按冷藏坑容量保存
未处理遗体过多 → 恐慌上升、信任下降
追思仪式使用后 → 信任小幅上升、恐慌小幅下降，并进入冷却

```
说明：  
* 墓园线记录已安葬遗体。  
* 冷藏坑线记录冷藏遗体。  
* 未处理遗体是社会风险源。  
* 遗体利用不应变成鼓励制造死亡的资源循环。  
   
⸻

**11. 与其他系统的关系**  
**11.1 与死亡状态链**  
人口死亡仍按人口状态链与死亡规则处理。  
死亡处理线只处理遗体与死亡后的社会后果，不改变死亡是否发生。  
**11.2 与信任 / 恐慌**  
死亡本身正常造成信任下降与恐慌上升。  
墓园线降低死亡冲击。 冷藏坑线提高处理效率，但更冷酷。 追思仪式是小幅社会安抚行动。 器官利用与灰烬肥料进入终局记录或后续事件，不得变成无代价收益。  
**11.3 与温室 / 医疗**  
器官利用是医疗增益方向。 灰烬肥料是温室增益方向。  
具体数值留到医疗表、温室表或平衡表处理。  
**11.4 与终局评价**  
以下内容应进入终局评价记录：  
```
墓园安葬路线
冷藏坑路线
遗体医疗利用
遗体农业利用
未处理遗体堆积
儿童死亡
外环高危死亡
过劳死亡

```
   
⸻  
   
**12. 本节结论**  
死亡处理线形成明确取舍：  
```
墓园线：
占地、慢、没有资源收益，但降低死亡冲击，稳定信任与恐慌，并解锁追思仪式。

冷藏坑线：
省地、快、可利用遗体，但降低信任、提高恐慌，并让终局评价更冷。

```
最终分支树：  
```
墓园线
└─ 追思仪式

冷藏坑线
├─ 器官利用
└─ 灰烬肥料

```
核心原则：  
炉城可以选择如何处理死者。 但无论选择墓园还是冷藏坑，死亡都不能被洗成免费资源。 墓园保存尊严，冷藏坑保存效率；两条路都在告诉玩家，这座城不只是如何活下去，也是在决定死者还剩下什么。
