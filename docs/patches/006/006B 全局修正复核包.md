## 全局修正 46：heat 开局可用，应急加热装置为 heat 强化科技  
**修正内容**  
V1 中，heat 命令开局可用。  
前提仍然是：  
```
目标建筑支持应急加热。
目标建筑当前满足可加热条件。
玩家支付对应燃料或资源成本。

```
006B-2 中的“应急加热装置”不再解锁 heat 命令。  
应急加热装置改为 heat 强化科技。  
强化方向包括但不限于：  
```
降低 heat 成本
提高 heat 效率
提高 heat 稳定性
降低 heat 相关风险
扩大 heat 可用表现

```
具体强化数值待后续数值表确定。  
**作废旧口径**  
```
heat 命令需要研究“应急加热装置”后才可使用
应急加热装置解锁 heat 命令
未研究应急加热装置前所有建筑都不能 heat
应急加热装置研究完成后自动加热所有建筑
应急加热装置研究完成后让所有建筑获得 heat

```

## 全局修正 47：研究所开局可建，但默认不可 heat  
**修正内容**  
研究所是开局可建基础建筑。  
```
研究所不需要科技解锁。
研究所不需要炉律解锁。
玩家建造研究所后，才能使用 research 命令。
研究所是 V1 科技系统入口。

```
研究所默认不可 heat。  
```
研究所默认不支持应急加热。
研究所不列入默认 can_heat 建筑。

```
研究所可以受建筑保温科技影响，但：  
```
建筑保温不等于 heat 权限。
应急加热装置不会让研究所自动获得 heat。

```
若后续需要研究所可 heat，必须在建筑表或后续 Patch 中单独添加：  
```
can_heat: true

```
**作废旧口径**  
```
研究所需要科技解锁
研究所需要炉律解锁
未解锁研究所前不能建造研究所
研究所默认可 heat
研究所天然属于可应急加热建筑
研究所获得建筑保温后自动获得 heat 权限
应急加热装置让研究所自动可 heat

```

## 全局修正 48：医疗建筑保温作用范围包含养护所  
**修正内容**  
医疗建筑保温作用于：  
```
医疗站
医院
养护所

```
其中：  
```
医疗站：获得医疗建筑保温收益。
医院：获得医疗建筑保温收益。
养护所：只获得保温与护理风险降低收益。

```
养护所不因此获得治疗功能增强。  
医疗建筑保温作用于养护所时，不增加：  
```
治疗容量
治疗速度
分级救治资格
急救治疗能力
医疗站 / 医院类功能

```
养护所仍然是伤残人口安置建筑，不是急救治疗建筑。  
**作废旧口径**  
```
医疗建筑保温只作用于医疗站 / 医院，完全不影响养护所
养护所获得医疗建筑保温后增加治疗容量
养护所获得医疗建筑保温后提高治疗效率
养护所获得医疗建筑保温后可执行分级救治
养护所被视为急救治疗建筑

```

## 全局修正 49：006B 数值可信度分级与未封存数值处理  
**修正内容**  
后续从 008 开始，凡由新施工窗、新补表、后续施工窗写出的数值，必须标记可信度状态。  
数值状态分为四级：  
```
未封存：施工窗写的，不能进代码。
待复核：看起来能用，但还没跑流程。
已审核：无明显冲突，但数值可能还需微调。
已封存：可以交给代码窗。

```
规则：  
```
只有“已封存”数值可以进入代码窗。
“未封存”“待复核”“已审核”都不能直接交给代码窗作为最终实现数值。

```
以原文与用户明确确认口径为准。  
```
施工窗补表中未能逐条追溯来源的内容，全部降级为【未封存】或【待确认】，不进入定稿。
原文没写、用户没确认的数值，不得脑补，不得交给代码窗猜。

```
**006B 当前已封存数值 / 机制口径**  
```
基础住宅容量：10
改良住宅容量：15
高级住宅容量：20
三者均占 1 槽
住宅不可 heat
温室改良机制：不是全局 passive，走逐座 upgrade 口径
高级住宅标准：正式加入 006B-4，T3，前置改良住宅标准，只解锁容量升级

```
**006B 当前未封存数值**  
```
006B-1～006B-4 科技木材成本
006B-1～006B-4 科技钢铁成本
006B-1～006B-4 科技研究天数
基础住宅 → 改良住宅升级耗材
改良住宅 → 高级住宅升级耗材
温室 → 改良温室升级耗材
施工窗补表中未能追溯来源的所有数值
施工窗死局自审中引用的所有新增数值

```
这些数值不进入代码窗。

## 全局修正 51：温室改良改为逐座升级科技  
**修正内容**  
006B-4 中，温室改良的生效方式为：  
```
unlock_upgrade

```
最终口径：  
```
温室栽培 = 解锁温室建筑。
温室改良 = 解锁“温室 → 改良温室”的逐座升级权限。

```
温室改良不是全局被动科技。  
```
研究完成后，不会自动强化所有温室。
玩家需要逐座执行 upgrade 温室。
每座温室升级需要材料。
温室升级耗材当前待确认。

```
温室升级后，该座温室效果方向：  
```
提高该座温室产出。
提高该座温室抗寒能力。

```
不改变：  
```
温室数量上限
温室人手需求
温室低温停产规则
无人 / 停工 / 有效温度不足时不能生产的规则
第七霜落终局压力

```
**作废旧口径**  
```
温室改良是 passive
温室改良研究后自动强化所有温室
温室改良提高温室数量上限
温室改良取消低温停产阈值
温室改良让无人 / 停工 / 有效温度不足的温室继续生产

```

## 全局修正 52：住宅三档、住宅升级与住房保温分离  
**修正内容**  
V1 住宅三档：  
```
基础住宅：10 人，1 槽，不可 heat。
改良住宅：15 人，1 槽，不可 heat。
高级住宅：20 人，1 槽，不可 heat。

```
住房保温与住宅升级分离：  
```
住房保温 = 降低住宅低温风险 / 提高居住安全。
住宅升级 = 提高单座容量。

```
二者不能互相替代。  
改良住宅标准：  
```
Tier：T2。
前置：住房保温 I。
生效方式：unlock_upgrade。
解锁“基础住宅 → 改良住宅”的逐座升级权限。
升级耗材待确认。

```
高级住宅标准：  
```
正式加入 006B-4。
Tier：T3。
前置：改良住宅标准。
生效方式：unlock_upgrade。
只解锁“改良住宅 → 高级住宅”的容量升级。
不提供住房保温。
不允许住宅 heat。
不替代住房保温科技。
升级耗材待确认。

```
住宅升级不需要修造工坊。  
**作废旧口径**  
```
住宅升级提供住房保温
住房保温增加住宅容量
改良住宅标准自动升级所有住宅
高级住宅标准提供住房保温
住宅升级让住宅可 heat
住宅升级改变占槽
住宅升级需要修造工坊

```

## 全局修正 53：小型采矿不能升级为大型矿机  
**修正内容**  
006B-3 中，小型采矿建筑与大型矿机不是同一条 upgrade 链。  
```
小型采煤机不能升级为大型煤矿机。
小型采钢机不能升级为大型钢铁矿机。

```
大型矿机是独立建筑：  
```
大型煤矿机必须绑定大型煤矿点。
大型钢铁矿机必须绑定大型钢铁矿点。

```
小型采煤机 / 小型采钢机是中期资源生产建筑。 大型煤矿机 / 大型钢铁矿机是独立后期矿点建筑。  
**作废旧口径**  
```
小型采煤机升级为大型煤矿机
小型采钢机升级为大型钢铁矿机
大型矿机不需要大型矿点
大型矿机可随意安放

```
   
⸻  
   
## 全局修正 54：建筑与升级拍下后立即生效，不新增施工期  
**修正内容**  
V1 中，建筑与升级按已封口径：  
```
拍下后立即可运行 / 立即生效。

```
不得新增：  
```
施工期
修缮期
升级中等待
临时施工状态
升级期间停用机制
升级期间居民迁出机制

```
科技只提供资格：  
```
unlock_building：只解锁建造资格。
unlock_upgrade：只解锁升级资格。
passive：研究完成后自动参与结算。

```
玩家仍需主动执行：  
```
build <建筑>
upgrade <建筑编号> <升级目标>

```
升级材料如果未封存，统一写：  
```
升级耗材：待确认

```
代码窗不得自行猜测升级耗材。  
**作废旧口径**  
```
科技完成后自动建造建筑
科技完成后自动升级建筑
建筑升级新增施工期
建筑升级新增等待天数
建筑升级期间自动迁出居民
建筑升级耗材由代码窗自行猜测

```

## 全局修正 55：006B-3 不封存地图模板数量与补偿参数  
**修正内容**  
006B-3 只处理资源科技与资源点使用规则，不最终锁定地图模板平衡。  
006B-3 只确认：  
```
地图中可以存在大型煤矿点。
地图中可以存在大型钢铁矿点。
地图中可以存在伐木场点。
大型煤矿机必须绑定大型煤矿点。
大型钢铁矿机必须绑定大型钢铁矿点。
伐木场必须绑定伐木场点。

```
以下内容不在 006B-3 中封存：  
```
各地图模板的大型煤矿点数量
各地图模板的大型钢铁矿点数量
各地图模板的伐木场点数量
三张地图资源总量是否相同
地图是否存在补偿
地图资源距离、风险、储量补偿
地图模板名称与最终结构

```
若后续 008 地图系统已经确定三张地图资源相同、无补偿，则以 008 地图系统为准。  
**作废旧口径**  
```
006B-3 封存地图模板数量
006B-3 封存地图补偿原则
006B-3 确定三地图资源差异
006B-3 确定三地图资源补偿

```
