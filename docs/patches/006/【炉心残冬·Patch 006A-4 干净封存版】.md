# 【炉心残冬·Patch 006A-4 干净封存版】  
主题：社会适配城约 A4 —— 食物配给线 状态：干净封存 归属：006A 社会适配城约  
   
⸻  
   
## 一、合并后的干净版正文  
**1. 定位**  
本节处理食物如何被分配、稀释与应急使用。  
它只处理：  
* 普通饭。  
* 粗粮汤。  
* 白米稀粥。  
* 应急口粮。  
* 配给模式切换。  
* 食物消耗、信任、恐慌与健康风险方向。  
本节不处理：  
* 温室解锁。  
* 温室产量科技。  
* 狩猎科技。  
* 食物储藏科技。  
* 仓库扩容。  
* 社会情绪日上限。  
* 食物生产科技。  
* 散落食物点。  
说明：  
* 温室属于后续工程科技 / 建筑科技，不属于普通社会适配城约。  
* V1 不设置散落食物点。  
* 食物配给线不新增建筑，不进入建筑占槽表。  
   
⸻  
   
**2. 核心规则**  
食物配给线采用：  
```
签署炉律 = 解锁配给模式
ration / 配给 命令 = 切换当前配给模式

```
签署炉律后，不会自动永久执行该配给。 玩家必须主动使用配给命令切换。  
例如：  
```
签署“粗粮汤”：
只代表粗粮汤模式已解锁。

配给 粗粮汤：
才代表当前配给切换为粗粮汤。

```
签署瞬间影响、模式使用影响、每日持续影响分层结算。  
   
⸻  
   
**3. 分支结构**  
默认模式：  
```
普通饭

```
主分支互斥：  
```
粗粮汤 ↔ 白米稀粥

```
后置炉律：  
```
应急口粮

```
完整结构：  
```
普通饭（默认）
├─ 粗粮汤
│  └─ 应急口粮
└─ 白米稀粥
   └─ 应急口粮

```
说明：  
* 普通饭开局默认可用。  
* 普通饭永远可切回。  
* 粗粮汤与白米稀粥互斥。  
* 应急口粮前置为粗粮汤或白米稀粥任意一项。  
* 应急口粮不与粗粮汤 / 白米稀粥互斥。  
* 应急口粮是当日救急配给行动 / 模式，不是日常持续配给制度。  
   
⸻  
   
**4. 配给命令**  
标准命令方向：  
```
ration normal

ration rice_porridge
ration emergency

```
中文兼容方向：  
```
配给 普通饭
配给 粗粮汤
配给 白米稀粥
配给 应急口粮

```
status 面板应显示当前配给模式。  
   
⸻  
   
**5. 普通饭**  
状态：默认可用 前置：无 解锁类型：默认配给模式 是否解锁建筑：否  
定位：  
普通饭是标准熟食配给。 它不节粮，也不额外伤害健康或社会情绪。  
命令：  
```
配给 普通饭
ration normal

```
规则：  
```
普通饭开局默认可用。
普通饭永远可切回。
无额外节粮效果。
无额外健康或情绪代价。

```
效果方向：  
```
正常食物消耗
无额外患病风险
无额外信任 / 恐慌代价

```
说明：  
* 普通饭始终可切回。  
* 玩家签署粗粮汤或白米稀粥后，普通饭不会被锁定。  
* 从粗粮汤 / 白米稀粥切回普通饭后，不会补偿此前已造成的社会或健康后果。  
   
⸻  
   
**6. 粗粮汤**  
前置：无 互斥：白米稀粥 解锁类型：配给模式 是否解锁建筑：否  
定位：  
粗粮汤是高强度节粮路线。 它最省食物，但最伤人心与健康。  
签署后解锁：  
```
配给 粗粮汤
ration coarse_soup

```
规则：  
```
粗粮汤与白米稀粥互斥。
签署后不会自动切换。
开启后持续，直到玩家切换其他配给。
普通饭仍可切回。

```
签署瞬间影响方向：  
```
信任：小幅下降
恐慌：小幅上升

```
模式效果方向：  
```
食物消耗明显下降
信任下降
恐慌上升
患病风险上升

```
文案方向：  
```
准备签署：粗粮汤

食堂将被允许把粗粮、碎壳与能煮开的边角料一起熬成汤。
它会让锅看起来更满，也会让每个人更清楚地意识到：炉城已经开始把“活着”和“吃饱”分成两件事。

效果：
- 解锁“配给 粗粮汤”。
- 粗粮汤模式下，食物消耗明显下降。
- 信任下降。
- 恐慌上升。
- 患病风险上升。
- 白米稀粥路线永久锁定。

```
执行提示方向：  
```
配给已切换：粗粮汤

食堂的锅变得更满了。
碗却变得更轻。

```
   
⸻  
   
**7. 白米稀粥**  
前置：无 互斥：粗粮汤 解锁类型：配给模式 是否解锁建筑：否  
定位：  
白米稀粥是温和节粮路线。 它不如粗粮汤节省，但代价较轻。  
签署后解锁：  
```
配给 白米稀粥
ration rice_porridge

```
规则：  
```
白米稀粥与粗粮汤互斥。
签署后不会自动切换。
开启后持续，直到玩家切换其他配给。
普通饭仍可切回。

```
签署瞬间影响方向：  
```
信任：微微下降
恐慌：微微上升

```
模式效果方向：  
```
食物消耗中度下降
信任小幅下降
恐慌小幅上升
健康代价较低

```
文案方向：  
```
准备签署：白米稀粥

食堂将把白米熬得更稀，让每一袋粮食多撑几天。
没人会把它叫作丰盛，但至少它仍是热的、干净的，也还能被称作一碗饭。

效果：
- 解锁“配给 白米稀粥”。
- 白米稀粥模式下，食物消耗中度下降。
- 信任小幅下降。
- 恐慌小幅上升。
- 健康代价较低。
- 粗粮汤路线永久锁定。

```
执行提示方向：  
```
配给已切换：白米稀粥

米被熬得更稀了。
人们低头看着碗，至少还能看见一点热气。

```
   
⸻  
   
**8. 应急口粮**  
前置：粗粮汤或白米稀粥任意一项 类型：当日救急配给行动 / 模式 互斥：无 解锁类型：当日救急配给行动 / 模式 是否解锁建筑：否 是否需要确认：是  
定位：  
应急口粮不是日常食物制度。 它是断粮前的最后救急手段。  
应急口粮不新增“虫子资源”、蠕虫资源或其他库存。 机制上只作为当日极限节粮模式处理。  
内容方向：  
```
仓底碎粮
冻硬旧饼
厨房边角料
可食用苔藓
城角缝隙里的小虫与蠕虫
一切还能被称作“入口”的东西

```
说明：  
* 上述内容是文案表现，不是新资源系统。  
* 不新增虫子、蠕虫或其他库存资源。  
* 不产生新食物。  
* 不额外提高患病风险。  
* 它的代价主要来自尊严、恐慌与信任受损，而不是健康惩罚。  
签署后解锁：  
```
配给 应急口粮
ration emergency

```
规则：  
```
应急口粮当日生效。
日结后自动恢复此前配给模式。
有冷却。
不能连续使用。
不新增虫子资源或其他库存。
不额外提高患病风险。
使用前必须 confirm。

```
签署瞬间影响方向：  
```
信任：微微下降
恐慌：微微上升

```
说明：  
签署“应急口粮”时，居民会意识到炉城已经准备把所有还能入口的东西都端上桌。 但它只是开放救急权限，不代表今天立刻执行。  
执行效果方向：  
```
当日食物消耗大幅下降
信任小幅下降
恐慌小幅上升
不额外提高患病风险
日结后自动恢复此前配给模式
进入冷却

```
确认提示方向：  
```
准备启用：应急口粮

食堂将刮下锅底、碾碎旧饼，并把一切还能入口的东西分进碗里。
这会显著降低今日食物消耗，但会伤害信任，并让恐慌上升。

确认请输入：
confirm 配给 应急口粮

```
执行成功提示方向：  
```
配给已切换：应急口粮

食堂开始刮下锅底、碾碎旧饼，筛出那些平时不会被放进碗里的东西。

```
```
有人盯着碗里微微动了一下的黑点，也许是一部分虫子，他们沉默了很久，最后还是咽了下去。

```
日结恢复提示方向：  
```
应急口粮已结束。

食堂停止使用最低限度存活配给。
当前配给已恢复为此前模式。

```

**9. 配给持续时间规则**  
配给模式分为持续配给与当日救急配给。  
持续到玩家手动切换的配给：  
```
普通饭
粗粮汤
白米稀粥

```
日结后自动结束的配给：  
```
应急口粮

```
规则：  
```
普通饭、粗粮汤、白米稀粥会持续到玩家手动切换。
应急口粮只在当日生效，日结后恢复此前配给模式。
切换配给不会回滚此前已经结算的社会、健康或食物后果。

```
   
⸻  
   
**10. 食堂与停工规则**  
食物配给模式主要通过食堂执行。  
规则方向：  
```
若食堂正常运行，按当前配给模式加工与分配熟食。
若食堂停工或温度不足，配给模式无法正常执行。
若食堂不存在，居民只能依靠默认可食资源与临时结算规则。

```
应急口粮仍需要基本分配能力。  
```
若食堂完全无法运行，应急口粮不能作为完整配给模式执行。
此时需按断粮 / 生食 / 临时食物规则结算。

```
说明：  
* 食堂停工不会自动取消已解锁配给炉律。  
* 食堂停工会暂停或破坏配给模式的正常执行。  
* 食堂恢复运行后，持续配给模式仍按当前已选择模式执行。  
* 应急口粮若因食堂无法运行而未能执行，具体是否消耗冷却留到实现阶段确认。  
   
⸻  
   
**11. 防漏洞规则**  
```
粗粮汤与白米稀粥互斥，不能同时拥有两种节粮路线。
应急口粮不能连续使用。
应急口粮日结后自动恢复此前配给模式，避免玩家忘记关闭。
应急口粮不产生新资源，只降低当日食物消耗。
应急口粮不新增虫子、蠕虫或其他库存资源。
普通饭永远可切回。
签署配给炉律不会自动执行该配给模式。
切换回普通饭不会回滚此前配给造成的信任、恐慌、患病或事件后果。

```
食物配给线不能变成无代价节粮工具。

**13. 与其他系统的关系**  
**13.1 与 Patch 004 命令系统**  
配给线接入 Patch 004 的 ration 命令入口。  
```
ration <mode>
配给 <模式>

```
**13.2 与日结顺序**  
食物配给在日结中接入：  
```
食物加工与消耗

```
并在疾病 / 情绪 / 事件前形成对应影响方向。  
**13.3 与信任 / 恐慌**  
信任 / 恐慌分层结算：  
```
签署炉律的一次性影响
切换或执行配给行动的影响
配给模式每日持续影响
事件影响

```
具体数值留给签署影响表、社会情绪 Patch 或平衡表。  
**13.4 与食堂温度**  
食堂低温停工时，配给无法正常执行。 食堂是否可应急加热按建筑温度规则处理，不在本节重复定义。  
**13.5 与断粮 / 生食**  
若食堂不存在、食堂停工或熟食加工失败，居民可能转入断粮 / 生食 / 临时食物规则。 本节不重写断粮和生食系统。  
   
⸻  
   
**14. 本节结论**  
食物配给线形成四层选择：  
```
普通饭：
稳定，无额外代价。

粗粮汤：
强节粮，高代价，持续到玩家切换。

白米稀粥：
中度节粮，低代价，持续到玩家切换。

应急口粮：

```
```
当日救急，极限节粮，有代价，必须 confirm，日结自动恢复，有冷却。

```
最终分支树：  
```
普通饭（默认）
├─ 粗粮汤
│  └─ 应急口粮
└─ 白米稀粥
   └─ 应急口粮

```
核心原则：  
炉城可以把饭熬稀，可以把锅底刮干净，也可以把最后一点能入口的东西端上桌。 但签署炉律不是自动执行，节粮不是凭空产粮。 每一次把碗变轻，都应该在信任、恐慌、健康风险和结局记账里留下痕迹。
