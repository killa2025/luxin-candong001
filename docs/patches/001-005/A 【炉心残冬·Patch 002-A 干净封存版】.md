# A 【炉心残冬·Patch 002-A 干净封存版】  
版本定位：建筑分类与温度规则封存 用途：供后续建筑表、源码结构、AI 玩家面板、人类说明文档继承  
   
⸻  
   
## 一、合并后的干净版正文  
**1. 本 Patch 核心目标**  
Patch 002-A 用于确定《炉心残冬》第一版中的建筑分类、推荐区域、温度规则、应急加热规则，以及哪些建筑需要人手、哪些建筑不参与温度系统。  
本 Patch 不负责最终槽位数值、人口曲线、具体建筑造价和产量。 这些内容留到后续 Patch 处理。  
   
⸻  
   
**2. 区域结构**  
地图采用功能分区，不采用复杂二维地图。  
区域分为：  
1. 内环：生活核心区  
2. 中环：技术与轻工业区  
3. 外环（边缘区为显示别名，不作为独立代码区域）：大型资源、制度设施、危险作业区  
4. 外缘储放区：仓储与无人工后勤建筑区  
重要原则：  
* 推荐区域不等于强制区域。  
* AI 玩家可以把建筑建在非推荐区域，但系统应提示风险或代价。  
* 外环不等于全部危险区域。  
* 外环中存在边缘设施区和大型资源区。  
* 外缘储放区专门用于仓库等不需要人手、不需要保温的建筑。  
   
⸻  
   
**3. 情绪系统命名**  
城市情绪字段统一为：  
**信任**  
代表居民对执政官、炉城秩序、分配制度和生存路线的认可。  
信任降低，说明居民开始怀疑当前领导是否还能带领他们活下去。  
**恐慌**  
代表灾难压力下的失控、猜疑、求生恐惧和群体崩溃风险。  
恐慌升高，说明居民越来越难以保持秩序。  
死亡影响规则：  
* 成人死亡和儿童死亡对信任、恐慌的影响幅度相同。  
* 不额外设置“儿童死亡更大幅降低信任 / 增加恐慌”。  
* 儿童可以在疾病链中略微更脆弱，但死亡后的社会数值冲击不额外放大。  
   
⸻  
   
**4. 温度系统总原则**  
1.0 版本不做复杂效率倍率。  
温度不足不再普遍造成“效率下降百分比”。  
建筑后果主要分为三类：  
1. 关键建筑温度不足：直接停工、停业或停课。  
2. 普通工作场所温度不足：增加患病、冻伤、事故或研究风险。  
3. 特殊建筑：不参与温度系统，不允许或不需要应急加热。  
这样可以减少代码复杂度，也让 AI 玩家更容易判断局势。  
   
⸻  
   
**5. 应急加热总原则**  
应急加热是开局可用的单点救急能力。    
  
指令方向：    
  
```
heat <建筑名>

```
  
规则：    
* 对单个建筑当日生效。    
* 消耗煤炭。    
* 日结后自动关闭。    
* 如果目标建筑温度已经足够，系统提示无需加热，不消耗煤炭。    
* 同一建筑同日不可重复 heat。    
* heat 不按建筑大小、占槽、区域、类型变化。    
* heat 不重复计入今日完整日耗。    
  
heat 使用时必须检查：    
* 目标建筑是否允许 heat。    
* 目标建筑当前是否需要 heat。    
* 玩家是否有足够煤炭支付 heat 成本。    
* 当日该建筑是否已经使用过 heat。    
* 建筑是否处于可被加热状态。    
  
应急加热装置口径：    
* 应急加热装置不是 heat 解锁科技。    
* 应急加热装置是 heat 强化科技。    
* heat 开局可用。    
* 应急加热装置只影响后续强化方向，例如降低成本、提高效率、提高稳定性。    
  
说明：    
* 住宅不能靠一键加热解决。    
* 研究所 / 工程研究站当前不列入默认可 heat 建筑。    
* 守炉堂 / 巡查所不可 heat。    
* 修造工坊 / 工坊不进入 V1 正式建筑表，因此不作为 heat 目标建筑。（代码窗无需写工坊）   
⸻  
   
**6. A 类建筑：必须维持最低温度，否则停工 / 停业**  
这些建筑温度不足时，不做效率下降，直接停止核心功能。  
**医疗站**  
推荐区域：内环 需要人手：是 是否可应急加热：是 温度不足后果：停止治疗，病人恶化与死亡风险上升  
定位：早期急救建筑。容量小，治疗效率正常，但怕冷。  
**医院**  
推荐区域：中环，也可建内环 需要人手：是 是否可应急加热：是 温度不足后果：停止治疗，病人恶化与死亡风险上升  
定位：中后期医疗建筑。容量是医疗站的 2 倍或更多，治疗效率约为医疗站的 50%。保温效果明显好于医疗站，可建在中环，不可建在外环作业区。  
**温室**  
推荐区域：中环 需要人手：是 是否可应急加热：是 温度不足后果：当日停产  
定位：稳定食物来源之一。温度不足时不减产，直接停产。  
**儿童保护所**  
推荐区域：内环 需要人手：是 是否可应急加热：是 温度不足后果：保护功能下降或失效，儿童患病风险上升  
定位：儿童保护路线核心建筑。温度要求高于普通住宅，等于医疗建筑。建筑有效温度过低时，保护效果下降或失效。  
**食堂 / 厨房**  
推荐区域：内环 需要人手：是 是否可应急加热：是 温度不足后果：停止做饭，居民可能只能消耗生食或低质量食物，影响健康与信任 / 恐慌  
定位：食物加工与配餐建筑。寒冷时必须优先保住。  
**酒馆**  
推荐区域：内环 需要人手：是 是否可应急加热：是 温度不足后果：停业  
说明： 酒馆属于安抚与消遣建筑，用于后续调节信任、恐慌或触发相关事件。具体数值效果留到城约 / 法令 / 民心系统中再定。  
**赌场**  
推荐区域：内环 需要人手：是 是否可应急加热：是 温度不足后果：停业  
说明： 赌场属于安抚与秩序风险并存的消遣建筑，用于后续调节信任、恐慌或触发相关事件。具体数值效果留到城约 / 法令 / 民心系统中再定。  
   
⸻  
   
**7. 准关键建筑：低温停课**  
**学堂**  
推荐区域：中环 需要人手：是 是否可应急加热：是 温度不足后果：停课；停课时不能产生医疗学徒 / 工程学徒效果  
说明：  
* 学堂属于准关键建筑。  
* 温度要求提高。  
* 建筑有效温度过低时，停课。  
* 停课时不能产生医疗学徒 / 工程学徒效果。  
* 儿童患病后若长期未治愈，可略微更容易死亡。  
* 儿童死亡与成人死亡对信任、恐慌的社会数值影响幅度相同。  
   
⸻  
   
**8. B 类建筑：温度不足不降效率，但增加患病 / 风险**  
这些建筑冷了仍可工作，但工作人员更容易生病、冻伤或发生事故。  
**研究所 / 工程研究站**  
推荐区域：中环 需要人手：是，主要消耗工程师 是否可应急加热：否，当前不列入默认可 heat 建筑 温度不足后果：工程师患病、冻伤、事故或研究风险上升  
说明：  
* 研究所 / 工程研究站属于普通工作场所。  
* 低温主要增加工程师患病、冻伤、事故或研究风险。  
* 不直接作为关键停工建筑处理。  
* 工程师稀缺，工程师生病后不能工作，会自然导致研究变慢或中断。  
* 若后续需要，可在 008-D 或建筑表中单独加“可 heat”标记。  
  
说明：  
* 小型采煤机 / 小型采钢机属于中期资源生产建筑。  
* 不放中环。  
* 不绑定大型矿点。  
* 不消耗地表小型煤堆 / 小型钢铁堆储量。  
* 地表小型煤堆 / 木材堆 / 钢铁堆是前期采集点，不是建筑。  
**大型煤矿机**  
推荐区域：外环（边缘区为显示别名，不作为独立代码区域） 需要人手：是 是否可应急加热：是 温度不足后果：工人患病、冻伤或事故风险上升  
说明：  
* 大型煤矿点本身是资源锚点，不占普通区域槽位。  
* 绑定大型煤矿点的大型煤矿机占用外环普通槽位。  
* 产量较大，但受区域、天气、事故和人力限制。  
**大型钢铁矿机**  
推荐区域：外环（边缘区为显示别名，不作为独立代码区域） 需要人手：是 是否可应急加热：是 温度不足后果：工人患病、冻伤或事故风险上升  
说明：  
* 大型钢铁矿点本身是资源锚点，不占普通区域槽位。  
* 绑定大型钢铁矿点的大型钢铁矿机占用外环普通槽位。  
* 产量较大，但受区域、天气、事故和人力限制。  
**木材区 / 伐木场**  
推荐区域：外环（边缘区为显示别名，不作为独立代码区域） 需要人手：是 是否可应急加热：是 温度不足后果：工人患病、冻伤风险上升  
说明：  
* 森林区本身不占普通区域槽位。  
* 伐木场作为建筑占用外环槽位。  
* 木材既是建筑材料，也是紧急燃料。  
* 木材不可稳定再生。  
   
⸻  
   
**9. 前期采集点与避寒采集棚**  
**地表小型煤堆 / 木材堆 / 钢铁堆**  
类型：前期采集点，不是建筑 推荐区域：外环 / 资源点 需要人手：是 是否可应急加热：否 温度不足后果：采集人员患病、冻伤风险上升  
说明：  
* 地表小型煤堆 / 木材堆 / 钢铁堆是前期采集点。  
* 采集点不是建筑，不按普通建筑占槽。  
* 采集点不等于小型采煤机 / 小型采钢机。  
* 可通过避寒采集棚降低采集风险。  
**避寒采集棚**  
可建位置：散落资源点 需要人手：否 是否可应急加热：否 温度不足后果：无 是否占常规建筑槽位：否  
作用：  
* 建在散落资源点上。  
* 提供基础挡风和短暂维持温度。  
* 降低采集人员患病、冻伤风险。  
* 不显著增加效率，或只小幅增加效率。  
* 资源点总产量固定，不因采集棚增加。  
* 资源点采完后可拆除，回收少量木材和钢材。  
说明： 避寒采集棚本身不是生产建筑，而是资源点安全附件。  
   
⸻  
   
**10. C 类建筑：特殊天气规则**  
**狩猎小屋**  
推荐区域：外环（边缘区为显示别名，不作为独立代码区域） 需要人手：是 是否可应急加热：否 温度不足后果：普通寒冷下猎人有小概率患病；极寒、暴雪、第七霜落阶段停工  
说明：  
* 狩猎小屋不允许一键加热。  
* 猎人主要风险来自外出狩猎，而不是小屋本身温度。  
* 狩猎小屋受猎区上限限制，不能无限建造、无限产食物。  
* 猎区本身不占普通槽位，狩猎小屋作为建筑占用外环槽位。  
   
⸻  
   
**11. D 类建筑：不参与温度系统**  
这些建筑不需要保温，不允许或不需要应急加热。  
**住宅 / 居民住房**  
推荐区域：内环 需要人手：否 是否可应急加热：否 温度不足后果：居民患病风险上升，严重低温时可能冻伤  
说明：  
* 住宅不能靠一键加热解决。  
* 住宅保温依靠炉心档位、住房等级、整体保温科技解决。  
**仓库**  
推荐区域：外缘储放区 需要人手：否 是否可应急加热：否 温度不足后果：无  
说明：  
* 仓库是容量建筑，不是岗位建筑。  
* 仓库可建在外缘储放区，不占用核心保温区。  
**小仓库 / 临时储备棚**  
推荐区域：外缘储放区 需要人手：否 是否可应急加热：否 温度不足后果：无  
定位：前期低成本、小容量，用于防止爆仓。  
**大仓库 / 深层储藏库**  
推荐区域：外缘储放区 需要人手：否 是否可应急加热：否 温度不足后果：无  
定位：后期高成本、大容量，可能需要研究解锁。  
**墓园**  
推荐区域：外环（边缘区为显示别名，不作为独立代码区域） 需要人手：否 是否可应急加热：否 温度不足后果：无  
说明： 墓园是死亡处理路线建筑，不占日常人手。  
**火化场**  
推荐区域：外环（边缘区为显示别名，不作为独立代码区域） 需要人手：否 是否可应急加热：否 温度不足后果：无  
说明： 火化场不需要常驻人手，不参与温度系统。  
**精神 / 秩序类建筑**  
推荐区域：外环（边缘区为显示别名，不作为独立代码区域） 需要人手：是 是否可应急加热：否 温度不足后果：无  
说明：  
* 精神 / 秩序类建筑需要派人维持效果，但不需要保温，也不因低温停工。  
* 它们代表守钟、巡逻、宣告、集会、看守、纪念、秩序维护等制度性岗位，而不是依赖恒温的生产建筑。  
示例：  
* 纪念碑  
* 誓约碑  
* 守炉钟  
* 集会旗场  
* 秩序岗台  
* 追思墙  
   
⸻  
   
**12. 医疗与死亡状态链**  
人口状态链暂定为：  
```
健康 → 患病 → 重症 → 死亡 / 伤残

```
规则方向：  
* 患病人口不能工作。  
* 重症人口更容易死亡。  
* 医疗站或医院停工时，病人恶化与死亡风险上升。  
* 冻伤可导致伤残。  
* 伤残人口不能常规劳动，但仍消耗食物、住房和热量。  
* 儿童可以在疾病中略微更脆弱，但儿童死亡和成人死亡对信任、恐慌的影响幅度相同。  
   
⸻  
   
**13. 儿童学徒简化规则**  
儿童成为学徒后，1.0 暂定按成人普通劳动力处理，不设置 0.5～0.7 劳动力系数。  
学徒是否能承担工程师岗位，暂不在 002-A 中钉死，留到后续平衡阶段或科技树阶段再定。  
设计原因：  
减少代码复杂度，避免过早引入细碎效率倍率。若后续测试发现学徒过强，再通过法令、科技、岗位限制或风险事件调整。  
   
⸻  
   
**14. 本 Patch 明确删除 / 暂不采用的复杂项**  
1.0 版本暂不采用：  
* 复杂效率百分比下降。  
* 每个建筑单独效率倍率。  
* 复杂热网 / 导热泵 / 管线覆盖。  
* 住宅一键加热。  
* 狩猎小屋一键加热。  
* 墓园 / 火化场常驻人手。  
* 仓库人手管理。  
* 前哨站与探索小队。  
   
⸻  
   
**15. Patch 002-A 总结**  
本 Patch 确定了《炉心残冬》第一版建筑温度系统的核心方向：  
* 推荐区域不是强制区域。  
* 外环被拆分为大型资源 / 边缘设施区与外缘储放区。  
* 仓库可放在外缘储放区，不需要人手和保温。  
* 精神 / 秩序类建筑需要派人，但不需要保温。  
* 墓园、火化场不需要人手，不参与温度系统。  
* 温室、医疗站、医院、儿童保护所、食堂、酒馆、赌场温度不足直接停工或停业。  
* 学堂属于准关键建筑，温度不足时停课，不能产生医疗学徒 / 工程学徒效果。  
* 研究所 / 工程研究站属于普通工作场所，当前不列入默认可 heat 建筑。  
* * 小型采煤机 / 小型采钢机、大型矿机、伐木场等不做统一效率下降，只增加患病、冻伤、事故或相关工作风险。  
* 小型采煤机 / 小型采钢机统一放在外环（边缘区为显示别名，不作为独立代码区域），不放中环，不绑定大型矿点。  
* 地表小型煤堆 / 木材堆 / 钢铁堆是前期采集点，不是建筑。  
* 狩猎小屋不允许应急加热，极寒或暴雪时停工。  
* 情绪系统统一为信任 / 恐慌。  
核心原则：  
玩家不需要管理复杂效率表。 玩家只需要判断：哪些建筑必须保温，哪些岗位可以冒冷硬干，哪些人一旦病倒会让城市立刻失去关键能力。
