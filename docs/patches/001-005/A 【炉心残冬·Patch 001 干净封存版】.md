# A 【炉心残冬·Patch 001 干净封存版】  
版本定位：基础机制方向封存 用途：供后续开发总纲、源码设计、人类说明文档继续继承  
   
⸻  
   
## 一、合并后的干净版正文  
**1. 游戏基础定位**  
游戏名暂定：《炉心残冬》。  
游戏类型：给 AI 玩家盲玩的文字策略生存游戏，同时保证人类旁观者能看懂进度、抉择和局势变化。  
游戏长度对玩家显示为：  
**55 个寒冬日夜**  
后台可以按阶段推进降温、回暖、寒潮和终局灾害，但对人类展示时优先使用直观天数。  
终局灾害暂名：  
**第七霜落**  
玩家身份暂定：  
**执政官**  
V1 文字版采用每日回合制 / 日计划制。  
规则：  
```
玩家在白天提交今日计划。
系统在 end_day 后统一结算。
玩家不能在日结中途追加操作。

```
因此：  
```
不模拟小时级煤耗。
不模拟日内实时温度。
不模拟上午 / 下午 / 夜晚分段操作。

```
说明：  
文字版不能照搬实时生存城市游戏。玩家按下 end_day 后，当日就进入结算，不再允许中途反应。  
   
⸻  
   
**2. 核心能源与资源定位**  
炉心井提供基础热源。  
煤炭用于推动热流，是主要燃料。  
木材可以在紧急情况下代替煤炭燃烧，但燃烧更快、效率更低，且不可稳定再生。大量烧木材会挤压建筑、采集棚、修缮、保温等需求。  
钢铁用于建造、研究、升级与工程设施。部分科技研究需要消耗木材和钢铁，代表试制设备、实验材料和结构改造。  
资源定位总结：  
* 煤炭：正经燃料。  
* 木材：建筑材料，也是救急燃料。  
* 钢铁：发展材料，用于高级建筑、科技、工程设施。  
V1 日结资源顺序采用：  
```
燃料 / 供暖 → 生产入库 → 食物加工与消耗 → 疾病 / 情绪 / 事件

```
煤炭生产与炉心消耗的结算口径：  
```
今日炉心供暖消耗读取日结前煤仓。
今日采集 / 生产的煤炭在本日日结后入库。
新入库煤炭主要用于下一日供暖。

```
今日生产不能自动支付当日炉心燃料缺口。  
示例口径：  
```
当前煤炭：20
炉心 1 档日耗：45
今日预计采煤：45

```
此时不能显示为“今日炉心安全”。  
应显示为：  
```
今日炉心 1 档燃料不足：缺口 25 煤。
今日预计采煤 45 将在日结后入库，主要保障明日。
若强行 end_day，将按供暖不足日结算。

```

**3. 信息面板原则**  
游戏内只保留三大主要面板，避免 AI 玩家和人类旁观者被信息淹没。  
**status**  
生死总览。显示天气、天数、炉心状态、煤炭、食物、人口、病人、重症、伤残、信任、恐慌、疲劳、主要警告。  
status 还应承载 end_day 前的关键风险预览。  
煤炭面板不再把小数天数作为主显示。  
内部可以计算：  
```
当前煤炭 ÷ 当前炉心日耗

```
但玩家主面板显示为：  
```
可完整供暖 X 日
日结后预计剩余 Y 煤
若无新增煤，第 N 日缺口 Z 煤

```
煤炭预览固定拆成三行：  
```
今日供暖支付：是否足够
今日生产入库：预计新增多少
日结后库存：预计剩余多少

```
**city**  
城市与建筑面板。显示三环区域、建筑、资源点、人力分配、温度状态、停工建筑、低温风险。  
**laws**  
城约与炉律面板。显示已签署法律、互斥路线、普通冷却、紧急令次数、路线倾向、可签署项目。  
**面板设计原则**  
* AI 能用面板做决策。  
* 人类旁观者能看懂 AI 为什么这样布局。  
* 面板不重复长篇说明。  
* 法律影响在签署前详细提示一次。  
* 签署后只显示短标签、互斥项、路线倾向。  
* 完整法律分支与影响另写人类可看文档。  
   
⸻  
   
**4. end_day 前风险预警原则**  
由于玩家不能在日内中途修正，所以系统必须在 end_day 前给出风险预览。  
end_day 前必须检查：  
```
今日炉心燃料是否足够。
今日食物是否明显不足。
今日医疗容量是否严重不足。
今日建筑是否会因有效温度过低停工。
今日是否存在大量无住所人口。
今日是否存在过载压力红线。
今日是否存在可能触发危机倒计时的信任 / 恐慌风险。

```
凡是会在日结中造成严重后果、且玩家日内无法补救的项目，都应在 end_day 前预警。  
包括：  
```
食物不足
医疗容量不足
关键建筑停工
大量无住所人口
极端寒冷暴露
信任 / 恐慌危机
过载压力红线

```
原则：  
```
可以让玩家犯错。
不能让玩家在无提示情况下被日结偷袭。

```
**end_day 预警分级**  
**A. 信息提示**  
不阻塞 end_day。  
用于：  
```
明日可能缺煤
仓储接近上限
食物储备偏低但今日够吃
轻度无住所风险
普通低温风险

```
显示即可，不要求 confirm。  
**B. 强警告**  
需要 confirm end_day。  
用于：  
```
今日炉心燃料不足
今日食物明显不足
关键建筑今日会停工
医疗容量严重不足
大量无住所人口处于寒冷风险
过载压力可能达到红线
信任 / 恐慌可能进入危机区

```
必须显示：  
```
风险原因
预计后果
可修正方向

```
**C. 硬阻塞**  
不能直接 confirm end_day，必须先处理或二次确认特殊命令。  
用于：  
```
炉心压力达到首次红线警告
furnace off 等极高风险操作尚未确认
拆除关键建筑等高风险操作尚未确认
命令状态不完整或存在非法分配

```
说明：  
硬阻塞不是为了保护玩家不犯错，而是防止玩家在没意识到后果时误按结算。  
**预览显示精度**  
日结预览可以显示风险等级和大致方向，但不要在所有系统上承诺精确人数。  
允许：  
```
预计患病风险：高
预计冻伤风险：中
预计恐慌：上升
预计信任：下降

```
谨慎使用：  
```
预计新增患病 7 人
预计死亡 3 人

```
推荐显示口径：  
```
低 / 中 / 高 / 极高
不足 / 勉强 / 稳定 / 充足
上升 / 下降 / 基本稳定

```

5. 地图结构  
  
不做复杂二维地图，不做方格覆盖计算。    
地图采用功能分区结构。    
  
正式区域名为：    
* 内环    
* 中环    
* 外环    
* 外缘储放区    
  
**内环**    
生活与高温需求建筑区。    
适合建筑：    
* 住宅    
* 医疗站    
* 食堂    
* 儿童保护所    
* 酒馆    
* 赌场    
  
**中环**    
技术、轻工业与部分中后期建筑区。    
适合建筑：    
* 研究所    
* 温室    
* 医院    
* 部分轻工业或公共建筑    
  
**外环**    
高风险资源、外勤、制度设施与危险作业区。    
适合内容：    
* 精神 / 城约相关建筑    
* 远处散落资源    
* 狩猎区    
* 深层资源点    
* 小型采煤机    
* 小型采钢机    
* 大型煤矿机    
* 大型钢铁矿机    
* 伐木场    
* 死亡处理建筑    
  
**外缘储放区**    
仓储与无人工后勤建筑区。    
适合内容：    
* 仓库    
* 大仓库    
* 储备设施    
* 其他不需要常驻人手的后勤建筑    
  
说明：    
* “边缘区”只作为外环显示别名，不作为独立代码区域。    
* 炭矿 / 铁矿 / 工坊不作为 V1 正式建筑名。    
* 修造工坊 / 工坊不进入 V1 正式建筑表。    
* 小型采煤机 / 小型采钢机属于中期生产建筑，放外环，不绑定大型矿点。    
* 大型煤矿点 / 大型钢铁矿点是资源锚点，不是直接产出建筑。    
* 绑定大型矿点的大型煤矿机 / 大型钢铁矿机占用外环建筑槽。    
  
1.0 版本不加入前哨站，不加入探索小队，不开启第二张探索地图。    
前哨站与探索系统留到 2.0。    
   
**6. 炉心与供暖系统**  
不使用复杂热网、导热泵、供热名额系统。  
供暖系统采用：  
**炉心三档 + 研究解锁过载 + 单点应急加热**  
**炉心三档**  
炉心普通档位开局自带 1～3 档。  
炉心档位影响整圈温度，不按建筑数量切割供热名额。  
furnace 档位一旦设置，会持续到玩家再次修改，或被系统规则强制关闭。  
示例：  
```
furnace 2

```
表示炉心维持 2 档，直到玩家输入新的 furnace 命令，或燃料不足导致系统按规则降档 / 关闭。  
**1档：节煤运行**  
* 煤耗最低。  
* 内环勉强可住。  
* 中环寒冷。  
* 外环危险。  
**2档：标准运行**  
* 煤耗增加。  
* 内环稳定。  
* 中环偏冷但可短时作业。  
* 外环危险。  
**3档：高功率运行**  
* 煤耗显著增加。  
* 内环温暖。  
* 中环基本可工作。  
* 外环仍危险。  
**过载**  
过载不是普通 4档 / 5档。  
过载需要科技研究后解锁。  
过载在当前炉心档位基础上临时强化炉心：  
* 提高温度。  
* 扩大有效抗寒范围。  
* 大幅增加煤耗。  
* 增加炉心压力。  
* 压力过高可能导致炉心事故、停炉或强制冷却。  
overload 状态持续到玩家手动关闭、炉心关闭、煤不足或压力规则强制关闭。  
过载用于寒潮、医疗崩溃、中环关键建筑失温、终局灾害等紧急情况。  
过载压力同样需要日结前预警。  
若当前过载设置可能导致炉心压力达到红线：  
```
end_day 前必须显示炉心压力风险。

```
示例：  
```
炉心压力：82
今日过载后预计压力：104
风险：将触发炉心红线警告
可修正方向：关闭过载或降低炉心档位

```
压力达到 100 的旧规则保留：  
```
第一次触发强制红线警告。
玩家有最后一次关闭过载机会。
若继续维持过载并再次爆表，触发炉心崩毁失败。

```
**木材燃烧状态**  
woodfuel 状态持续到玩家手动关闭，或木材不足 / 权限变化导致无法继续。  
涉及 woodfuel 时，核心命令不由事件按钮自动修改。  
规则：  
```
事件只显示可执行命令。
玩家必须手动输入命令。

```
示例：  
```
woodfuel on：开启木材燃烧，但需要已解锁木材燃烧权限。
woodfuel off：关闭木材燃烧。

```
woodfuel on 后持续到玩家手动关闭，或木材不足导致无法继续。  
煤炭恢复后，woodfuel 不会自动关闭。  
玩家必须手动输入 woodfuel off，才会停止木材燃烧。

7. 应急加热  
应急加热是单点救急能力。    
heat 命令开局可用，不需要先研究科技解锁。    
  
指令方向：    
  
```
heat <建筑名>

```
  
效果：    
* 对单个建筑当日生效。    
* 消耗煤炭。    
* 降低目标建筑当日低温惩罚。    
* 可在当日救回部分关键建筑运行。    
* 日结后自动关闭。    
* 如果目标建筑温度已经足够，系统提示无需加热，不消耗煤炭。    
* 每日使用次数有限，避免变成繁琐操作。    
  
heat 属于当日一次性效果。    
  
heat 使用时必须检查：    
* 目标建筑是否允许 heat。    
* 目标建筑是否已经满足运行温度。    
* 当日是否已对该建筑使用过 heat。    
* 今日剩余煤炭是否足够。    
* 当前建筑是否处于可被加热状态。    
  
默认可 heat 方向：    
* 医疗站    
* 医院    
* 食堂    
* 温室    
* 儿童保护所    
* 酒馆    
* 大赌场    
* 其他建筑表明确标记 can_heat: true 的关键建筑    
  
默认不可 heat：    
* 住宅    
* 仓库    
* 墓园    
* 冷藏坑    
* 炉旁设施    
* 普通精神 / 秩序路线建筑    
* 避寒采集棚    
* 狩猎小屋    
* 守炉堂    
* 巡查所    
* 研究所    
  
说明：    
* 应急加热装置不是 heat 解锁科技。    
* 应急加热装置是 heat 强化科技。    
* 应急加热装置可用于后续降低 heat 成本、提高 heat 效率或改善 heat 稳定性。    
* 具体强化数值以后续科技表封存为准。    
* 修造工坊 / 工坊不进入 V1 正式建筑表，因此不作为 heat 目标建筑。

**8. 建筑温度需求**  
同一环内基础温度一致，但不同建筑有不同温度要求。  
**住宅**  
温度要求较低。温度不足会增加居民患病风险。  
外勤 / 资源生产建筑  
温度不足时不统一做效率下降，而是提高工作人员患病、冻伤或事故风险。    
小型采煤机、小型采钢机、大型煤矿机、大型钢铁矿机、伐木场等具体规则以后续建筑表与温度表为准。    
修造工坊 / 工坊不进入 V1 正式建筑表。    
**温室**  
温度不足时直接停产。  
可通过应急加热恢复当日产出。  
**医疗站**  
早期医疗建筑。  
特点：  
* 成本低。  
* 容量较小。  
* 治疗效率正常。  
* 温度要求高。  
* 温度不足时停止治疗。  
* 可用应急加热恢复治疗。  
定位：早期急救点，救得快，但怕冷。  
**医院**  
中后期医疗建筑，需要研究解锁。  
特点：  
* 成本高。  
* 容量是医疗站的 2 倍或更多。  
* 治疗效率约为医疗站的 50%。  
* 保温效果明显好于医疗站。  
* 可建于内环或中环。  
* 不可建于外环。  
* 温度严重不足时仍会停止治疗。  
* 可用应急加热恢复治疗。  
定位：大容量收治建筑，治疗慢，但更抗寒，适合应对大量病人。  
   
⸻  
   
**9. 避寒采集棚**  
类型：临时资源设施。  
可建位置：散落资源点。  
作用：  
* 建在散落资源点上。  
* 提供基础挡风和短暂维持温度。  
* 降低采集人员患病、冻伤风险。  
* 不显著增加效率，或只小幅增加效率。  
* 资源点总产量固定，不因采集棚增加。  
* 资源点采完后可拆除，回收少量木材和钢材。  
* 不占常规建筑槽位。  
* 不需要人员管理。  
设计定位：  
避寒采集棚不是产能建筑，而是安全建筑。它让玩家用少量前期资源换取更低的采集伤病风险，尤其适合距离炉心较远、暴露在寒风中的散落资源点。

**10. 城约 / 炉律系统**  
法律系统统称为城约，单条法律可称为炉律。  
普通城约有统一冷却。  
紧急令一局次数有限，使用后也有冷却，代价递增。  
终极城约不算紧急令，只在终局阶段且前置完成后解锁。  
终极城约是路线终章，不是普通万能按钮。  
城约签署、炉律解锁、路线选择、终极城约属于永久解锁 / 永久路线效果。  
规则：  
```
永久解锁效果不会因冷却结束而消失。
不会因主动行动未使用而消失。
不会因一天结束自动取消。

```
但：  
```
建筑无人值守时，可能暂停主动行动使用。
建筑停工不取消已签署炉律。

```
冷却只限制再次使用，不代表效果持续。  
规则：  
```
冷却期间，不能再次使用该行动。
冷却结束后，可再次使用。
冷却不会每天重复结算上一次行动效果。

```
法律展示规则：  
* 签署前详细提示一次效果、代价、互斥项。  
* 签署后面板只显示短标签。  
* 已签署法律旁边显示对应放弃的互斥路线。  
* 完整法律影响写入人类可看文档。  
   
⸻  
   
**11. 状态效果持续规则**  
所有操作、事件、炉律、科技效果统一分为五类：  
```
1. 持续到玩家手动修改
2. 当日一次性效果
3. 明确持续 N 天的临时效果
4. 永久解锁 / 永久路线效果
5. 冷却 / 承诺 / 倒计时效果

```
若旧 Patch 已明确持续时间，则按旧 Patch 执行。  
若旧 Patch 未明确持续时间，则按本规则的默认口径执行。  
若后续具体事件另行指定持续时间，则以后续具体事件为准。  
**持续到玩家手动修改**  
以下状态一旦设置，会持续到玩家再次修改，或被系统规则强制关闭：  
```
furnace 档位
overload 状态
woodfuel 状态
ration 配给模式
长班制开关
建筑人手分配
工作岗位分配

```
这些不是“点一次只管一天”的按钮。  
**炉心相关状态不由事件按钮自动修改**  
涉及以下核心命令时，事件不得自动执行：  
```
furnace
overload
woodfuel

```
规则：  
```
事件只显示可执行命令。
玩家必须手动输入命令。

```
**当日一次性效果**  
若事件、主动行动或选项没有明确写“持续 N 天”，默认视为当日一次性结算。  
包括：  
```
信任变化
恐慌变化
资源消耗
旧城派人数变化
死亡余波降低
当日安抚
当日公告
当日悼念
当日食物优先分配

```
一次性效果只在触发时结算一次，不会每天持续生效。  
**明确持续 N 天的临时效果**  
若某效果会限制分配、暂停制度、改变某类风险，必须写明持续到哪一天。  
推荐字段：  
```
effect_until_day: int

```
effect_until_day 表示最后一个仍然生效的日期。  
规则：  
```
current_day <= effect_until_day 时，效果生效。
current_day > effect_until_day 时，效果失效。

```
示例：  
```
当前第 10 天触发效果。
effect_until_day = 11。
则第 10 天和第 11 天效果生效。
第 12 天开始失效。

```
所有临时效果到期后自动清除。  
失效后：  
```
系统恢复原规则。
不会补偿资源。
不会自动恢复信任 / 恐慌。
不会自动重新分配岗位。

```

**12. 儿童路线**  
儿童路线存在互斥选择。  
**儿童辅工**  
保前中期。  
特点：  
* 提供低效率劳动力。  
* 儿童工作效率低于成人。  
* 儿童更容易生病、冻伤、事故。  
* 低温和高危岗位风险更高。  
* 偏效率 / 铁律路线。  
**儿童保护**  
保中后期。  
特点：  
* 儿童不参与常规劳动。  
* 需要儿童保护所。  
* 儿童保护所占内环槽位。  
* 儿童保护所温度要求高于普通住宅，接近医疗建筑。  
* 后续可解锁学堂与学徒体系。  
* 偏余温 / 人性路线。  
学堂不能太早开放。  
学徒效率低于成人，只能做辅助岗位，不能从事外环、高危采矿等危险工作。  
   
⸻  
   
**13. 死亡与伤残**  
冻伤可导致伤残人口。  
伤残人口不能常规劳动，但仍消耗食物、住房和热量。  
完整义肢工厂线暂不放进初版核心，作为后续扩展模块。  
死亡处理路线可分为：  
**墓园 / 安葬路线**  
偏人性。  
特点：  
* 占地。  
* 需要维护。  
* 死亡带来的信任 / 恐慌冲击较低。  
* 更容易发展伤残照护、追思、康复等后续机制。  
**火化 / 快速清理路线**  
偏效率 / 铁律。  
特点：  
* 省地。  
* 省管理。  
* 处理效率高。  
* 更冷酷，可能带来更高信任 / 恐慌冲击或路线倾向变化。  
   
⸻  
   
**14. 仓库**  
仓库不需要人员管理。  
仓库是容量建筑，不是岗位建筑。  
仓库分为：  
**小仓库 / 临时储备棚**  
前期低成本、小容量，用于防止爆仓。  
**大仓库 / 深层储藏库**  
后期高成本、大容量，需要研究或更高建设条件。  
   
⸻  
   
**15. 狩猎小屋**  
狩猎小屋不能无条件永续强产食物。  
限制方向：  
* 受温度和天气影响。  
* 极寒 / 暴雪时减产或停工。  
* 受猎区上限限制数量。  
* 不因无限建造而无限产出。  
   
⸻

**17. Patch 001 总结**  
本 Patch 确定了《炉心残冬》第一版的核心方向：  
* 55 天固定生存挑战。  
* 每日回合制 / 日计划制。  
* end_day 后统一结算，不做日内实时补救。  
* 三环文字地图。  
* 炉心三档供暖。  
* furnace / overload / woodfuel 等核心状态持续到玩家手动修改或被规则强制关闭。  
* 过载作为研究后赌命能力。  
* 应急加热作为单点救急能力，heat 当日生效，日结后自动关闭。  
* 今日炉心供暖消耗读取日结前煤仓。  
* 今日采集 / 生产的煤炭在本日日结后入库，主要用于下一日供暖。  
* 煤炭面板主显示使用“可完整供暖 X 日 + 日结后剩余 + 次日缺口”。  
* 高风险后果必须在 end_day 前预警。  
* 医疗站、医院、温室具有明确温度需求。  
* 避寒采集棚解决远处资源采集风险。  
* 前哨站与探索系统暂缓到 2.0。  
* 游戏信息必须同时服务 AI 决策和人类旁观理解。  
* 城市情绪字段统一使用信任 / 恐慌。  
核心原则：  
玩家不需要管理复杂工程表。  
玩家只需要判断：今天烧多少煤、救哪个建筑、派谁工作、签什么城约，以及这座城要用什么方式活过 55 个寒冬日夜。  
**温度计算规则**  
V1.0 所有温度均使用整数。  
基础温度、炉心修正、过载修正、建筑保温修正、科技修正均为整数。  
有效温度 = 基础温度 + 区域供暖修正 + 过载修正 + 建筑 / 住房 / 科技修正。  
若后续出现小数，统一四舍五入为整数后再结算。
