# A 【炉心残冬·Patch 002-B 干净封存版】  
版本定位：人口、住房容量与固定增员规则封存 用途：供后续槽位设计、事件系统、源码结构、AI 玩家面板继承  
   
⸻  
   
## 一、合并后的干净版正文  
**1. 本 Patch 核心目标**  
Patch 002-B 用于确定《炉心残冬》第一版中的人口统计方式、住房容量、劳动力规则、固定增员事件、失败条件与人口面板字段。  
本 Patch 不追踪单个居民姓名，不追踪“某人住在哪栋房子”。 游戏只统计人口池、住房容量、可工作人口、无法工作人口和医疗压力。  
   
⸻  
   
**2. 情绪系统**  
正式采用：  
* 信任  
* 恐慌  
不再使用“希望 / 压迫 / 不满”。  
**信任**  
代表居民对执政官、炉城秩序、分配制度和生存路线的认可。  
**恐慌**  
代表灾难压力下的失控、猜疑、求生恐惧和群体崩溃风险。  
后续城约 / 炉律系统中：  
* 有些法令可能提高信任、降低恐慌。  
* 有些法令可能降低信任、提高恐慌。  
* 有些法令可能短期稳定局势，但长期留下路线代价。  
   
⸻  
   
**3. 人口分类**  
人口按群体池统计，不按个人追踪。  
**工人**  
数量最多，是主要劳动力。  
可从事：  
* 采煤  
* 采铁  
* 伐木  
* 狩猎  
* 食堂  
* 建造  
* 普通采集  
* 其他普通工作岗位  
工人生病后不能工作。  
**工程师**  
数量稀少，是关键技术人口。  
可从事：  
* 研究所  
* 医疗站  
* 医院  
* 高级工程岗位  
* 工程相关建筑  
工程师也可以从事普通工人岗位。 但工人不能执行工程师岗位。  
工程师生病后不能工作，会直接影响研究、医疗和高级工程能力。  
**儿童**  
儿童默认不能参与常规劳动。  
儿童是否能工作，取决于儿童路线法令。  
**儿童辅工**  
签署儿童辅工相关法令后，儿童可以参与普通工作。  
儿童辅工可以参加普通工作岗位，但更容易：  
* 生病  
* 冻伤  
* 发生事故  
儿童不能执行工程师岗位。  
**学徒**  
学徒属于儿童保护路线后续发展。  
1.0 版本暂定：  
* 学徒按对应建筑普通人手处理。  
* 不设置 0.5～0.7 劳动力系数。  
* 学徒不能进入外环高危岗位。  
* 医疗学徒可去医疗站 / 医院。  
* 工程学徒可去研究所 。  
设计原因：  
减少代码复杂度。若后续测试发现学徒过强，再通过岗位限制、科技条件、法令代价或风险事件调整。  
   
⸻  
   
**4. 疾病与工作规则**  
所有人生病后都不能工作。  
不采用“患病人口降低工作时长”的复杂设计。  
正式规则：  
* 健康人口可以工作。  
* 患病人口不能工作。  
* 重症人口不能工作。  
* 伤残人口不能常规工作。  
* 生病人口会从可工作人口中移除。  
* 低温、饥饿、过劳、危险岗位会增加患病、冻伤或事故风险。  
这样可以减少半工时、来回治疗、部分工作效率等复杂计算。  
   
⸻  
   
**5. 人口状态链**  
人口状态链暂定为：  
```
健康 → 患病 → 重症 → 死亡 / 伤残

```
规则方向：  
* 患病人口若长期得不到治疗，可能恶化为重症。  
* 重症人口死亡风险更高。  
* 冻伤、事故、疾病恶化可导致伤残。  
* 伤残人口不能常规劳动，但仍消耗食物、住房和热量。  
* 医疗站或医院停工时，患病与重症人口恶化、死亡风险上升。  
* 成人死亡和儿童死亡对信任、恐慌的影响幅度相同。  
* 儿童可以在疾病链中略微更脆弱，但死亡后的社会数值冲击不额外放大。  
   
⸻  
   
**6. 住房系统**  
住房系统只统计容量，不追踪每个人住在哪。  
需要统计：  
* 当前总人口  
* 当前住房容量  
* 无住所人口  
**住宅容量暂定**  
* 基础住宅：10 人  
* 改良住宅：15 人  
* 高级住宅：20 人  
具体数值后续可根据人口曲线和平衡测试调整。  
**无住所人口**  
若总人口超过住房容量，则产生无住所人口。  
无住所人口会带来：  
* 患病风险上升  
* 冻伤风险上升  
* 信任下降  
* 恐慌上升  
寒潮、极寒、第七霜落期间，无住所惩罚加重。  
**死亡后的住房处理**  
居民死亡后，住房容量自动空出来。  
玩家不需要手动重新分配住房。  
若人口减少导致住宅空置，玩家可以选择拆除空房屋，回收少量材料并释放建筑空间。 拆除不是强制操作。  
   
⸻  
   
**7. 住宅与保温**  
住宅不能使用应急加热。  
住宅保温依靠：  
* 炉心档位  
* 住房等级  
* 整体保温科技  
* 所在区域温度  
住宅温度不足不会让住宅“停工”，而是让居民患病、冻伤风险上升。  
   
⸻  
   
**8. 固定增员事件**  
1.0 不加入前哨站、探索小队和第二张探索地图。  
因此采用固定增员事件，让城市人口在 55 个寒冬日夜中发生变化。  
固定增员事件用于：  
* 补充劳动力  
* 提供稀缺工程师  
* 增加儿童与照护压力  
* 增加患病、重症、伤残人口  
* 制造住房、食物、医疗压力  
**事件数量暂定**  
1.0 设置 3 次固定增员事件：  
1. 早期：小批求生者  
2. 中期：工程残队  
3. 后期：大批难民  
具体日期与人数留到事件 Patch 再定。  
   
⸻  
   
**9. 增员事件内容方向**  
**早期：小批求生者**  
目的：让玩家第一次面对新增人口压力。  
可能加入：  
* 少量工人  
* 少量儿童  
* 少量患病者  
压力：  
* 住房压力上升  
* 食物消耗增加  
* 医疗开始吃紧  
**中期：工程残队**  
目的：补充稀缺工程师，同时制造医疗或照护压力。  
可能加入：  
* 少量工程师  
* 患病工程师  
* 少量伤残者  
压力：  
* 接纳后增强研究和医疗潜力。  
* 短期会占用医疗床位、食物和住房。  
**后期：大批难民**  
目的：在终局前制造食物、住房、医疗三重压力。  
可能加入：  
* 较多工人  
* 儿童  
* 患病者  
* 重症者  
* 伤残者  
压力：  
* 食物消耗暴涨  
* 医疗系统容易崩溃  
* 住房不足会放大低温伤病  
* 终局前资源调度压力上升  
   
⸻  
   
**10. 接纳与拒绝规则**  
增员事件可以设计为：  
* 接纳全部  
* 接纳部分  
* 拒绝  
但接纳或拒绝不直接改变信任和恐慌。  
它只影响后续发展条件，例如：  
* 人口结构  
* 劳动力数量  
* 工程师数量  
* 医疗压力  
* 食物压力  
* 住房压力  
* 伤残人口压力  
* 后续事件条件  
* 路线记录  
示例：  
接纳患病者不会立刻增加恐慌。 但如果医疗系统被压垮，后续出现大量死亡，就会间接降低信任、增加恐慌。  
拒绝工程残队不会立刻降低信任。 但后续工程师不足，可能导致研究落后、炉心升级困难、终局更难撑过。  
   
⸻  
   
11. 失败条件：  
  
1.0 当前硬失败条件统一引用全局口径。    
  
V1 当前硬失败条件为：    
1. 存活人口 = 0。    
2. 炉心崩毁。    
3. 信任 = 0 且未被终火誓约承接。    
4. 恐慌 = 100 且未被最高戒令压制。    
  
说明：    
* 信任归零不再无条件直接失败，需检查是否已被终火誓约承接。    
* 恐慌顶满不再无条件直接失败，需检查是否已被最高戒令压制。    
* 第七霜落未撑过去不作为 002-B 独立硬失败条件。    
* 低综合分、多系统崩坏、低于城市连续性参考线不作为硬失败。    
* 危机倒计时只作为预警 / 事件链 / 连锁升级机制，不新增独立硬失败轴。     
⸻  
   
12. 危机倒计时机制：  
高风险状态不直接失败，而是触发危机事件、预警或连锁升级。    
  
信任危机：  
信任过低时，触发信任危机。    
玩家需要在限定天数内改善信任，避免事件升级。    
若未完成，可能触发起义、流放执政官、炉城秩序崩溃等事件后果。    
但信任危机倒计时本身不新增独立硬失败轴。    
最终是否失败，仍以全局硬失败口径为准：信任 = 0 且未被终火誓约承接。    
  
恐慌危机：  
恐慌过高时，触发恐慌危机。    
玩家需要在限定天数内降低恐慌，避免事件升级。    
若未完成，可能触发暴乱、逃亡、抢粮、秩序崩溃等事件后果。    
但恐慌危机倒计时本身不新增独立硬失败轴。    
最终是否失败，仍以全局硬失败口径为准：恐慌 = 100 且未被最高戒令压制。    
  
可用解决方向：  
解决危机的方式可以包括：    
* 改善保温    
* 颁布城约    
* 建设安抚建筑    
* 调整食堂配给    
* 提供更好的食物    
* 启用酒馆、赌场等消遣建筑    
* 处理医疗、住房、饥饿、寒冷等根源问题    
  
具体数值与事件文本留到后续危机系统 Patch 处理。    
   
⸻  
   
**13. 安抚与消遣方向记录**  
后续建筑与法令系统需加入安抚类内容。  
可能包括：  
* 酒馆  
* 赌场  
* 热汤厨房  
* 稀粥配给  
* 米饭 / 标准餐  
* 节庆餐 / 安抚餐  
这些内容可能影响信任、恐慌、食物消耗或秩序风险。  
本 Patch 只记录方向，不在 002-B 中设定具体数值。  
   
⸻  
   
**14. status 面板人口字段**  
status 面板必须显示足够清楚的人口信息，避免 AI 和人类旁观者自行计算。  
必须显示：  
* 总人口  
* 工人  
* 工程师  
* 儿童  
* 患病人口  
* 重症人口  
* 伤残人口  
* 可工作人口  
* 已分配工作  
* 无工作人数  
* 无法工作人口  
* 住房容量  
* 无住所人口  
* 医疗容量  
* 信任  
* 恐慌  
**字段解释**  
可工作人口：当前健康且具备工作资格的人口。  
已分配工作：已经被安排到岗位的人口。  
无工作人数：健康、可工作，但当前没有被分配岗位的人口。  
无法工作人口：患病、重症、伤残、未被允许工作的儿童等无法常规工作的人口。  
面板示例：  
```
总人口：68
工人：42｜工程师：8｜儿童：10｜伤残：8
患病：12｜重症：3
可工作人口：43
已分配工作：38
无工作人数：5
无法工作人口：25
住房：60/68，8人无住所
医疗：10/12
信任：54｜恐慌：48

```
   
⸻

Patch 002-B 暂不处理：  
* 最终建筑槽位数  
* 每次增员事件具体日期  
* 每次增员事件具体人数  
* 增员事件完整文本  
* 接纳 / 拒绝的具体事件分支  
* 城约对增员事件的影响

* 酒馆、赌场、配给改善的具体效果  
* 义肢工厂与伤残恢复劳动  
* 单个居民姓名与个人经历  
* 复杂家庭系统  
   
⸻  
   
**16. Patch 002-B 总结**  
本 Patch 确定了《炉心残冬》第一版人口系统的核心方向：  
* 人口按群体池统计，不追踪个人。  
* 住房按容量统计，不追踪住户分配。  
* 工程师可以做工人工作，工人不能做工程师工作。  
* 儿童默认不能工作，儿童辅工法令后可参与普通工作，但风险更高。  
* 学徒按对应建筑普通人手处理，不设置 0.5～0.7 劳动力系数。  
* 学徒不能进入外环高危岗位。  
* 医疗学徒可去医疗站 / 医院。  
* 工程学徒可去研究所 。  
* 所有患病、重症、伤残人口不能常规工作。  
* 不采用患病降低工作时长。  
* 1.0 设置 3 次固定增员事件。  
* 接纳或拒绝不直接影响信任、恐慌，只影响后续发展。  
* 当前硬失败条件统一引用全局口径：存活人口 = 0、炉心崩毁、信任 = 0 且未被终火誓约承接、恐慌 = 100 且未被最高戒令压制。  
* 信任过低或恐慌过高会先触发危机倒计时；危机倒计时只作为预警 / 事件链 / 连锁升级机制，不新增独立硬失败轴。  
* status 面板必须显示无工作人数和无法工作人口，方便 AI 决策与人类旁观。  
核心原则：  
玩家不需要管理每个居民。 玩家只需要判断：还有多少人能工作，多少人在病床上，多少人没有房子，接纳新人会不会压垮医疗和粮食，以及这座炉城还能不能撑过下一个寒冬日夜。
