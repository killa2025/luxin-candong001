# A 【炉心残冬·Patch 003 干净封存版】  
版本定位：资源系统与每日结算封存 用途：供后续伪源码、资源表、日结算函数、status / city 面板继承  
⸻  
## 一、合并后的干净版正文  
**1. 本 Patch 只处理什么** Patch 003 只处理：  
* 核心资源种类  
* 食物与饥饿  
* 燃料与炉心消耗  
* 生食 / 熟食逻辑  
* 人手不足与产出比例  
* 仓储容量  
* 爆仓规则  
* 资源与生存日结局部顺序  
    * 资源面板显示方向  
本 Patch 不重复处理：  
* 建筑占槽  
* 建筑数量上限  
* 人口分类  
* 医疗建筑分类  
* 城约 / 炉律具体效果  
* 科技树具体解锁  
* 随机事件文本  
* 承诺到期结算  
* 事件优先级队列  
* 终局状态评级  
* 终局结算  
* 硬失败总流程  
说明： 003 只提供资源与生存结算的局部顺序。 最终全局日结顺序以后续 007 / 008 / 009 及全局总控为准。  
⸻  
**2. 核心资源种类** 1.0 版本资源暂定为：  
* 煤炭  
* 木材  
* 钢铁  
* 生食  
* 熟食  
* 仓储容量  
**煤炭** 煤炭是主要燃料。  
主要用途：  
* 炉心运行  
* 炉心高档运行  
* 炉心过载  
* 应急加热  
* 部分后续建筑或科技消耗  
煤炭不足会导致炉心供暖不足、降档或关闭，并在 end_day 前给出明确警告。  
**木材** 木材是基础建材，也是紧急燃料。  
主要用途：  
* 建造基础建筑  
* 建造避寒采集棚  
* 建造仓库  
* 部分科技试制  
* 煤炭不足时作为应急燃料  
木材可作为燃料，但不会自动燃烧。 必须由玩家解锁相关权限后手动开启。  
说明： 修造工坊 / 工坊不进入 V1 正式建筑表，不作为木材的正式资源用途接口。  
**钢铁** 钢铁是发展材料。  
主要用途：  
* 高级建筑  
* 医院  
* 温室  
* 研究所  
* 炉心改造  
* 科技试制材料  
* 工程设施建设  
钢铁不作为燃料。  
说明： 修造工坊 / 工坊不进入 V1 正式建筑表，不作为钢铁的正式资源用途接口。  
**生食** 生食是未经加工的食物。  
来源包括：  
* 狩猎小屋  
* 温室  
* 固定事件  
* 开局储备  
居民开局即可直接食用生食。  
**熟食** 熟食是更稳定、更被接受的日常食物。  
来源：  
* 食堂加工生食  
食堂加工比例采用全局口径：  
* 1 生食 = 2 熟食  
食堂建成并正常运行后，居民优先消耗熟食。  
⸻  
**3. 生食与熟食规则** 居民开局即可食用生食，不需要签署城约。  
设计原因： 前期可能尚未建造食堂。 若居民不能直接吃生食，开局阶段容易因为食堂缺失而不合理饿死。  
**生食后果** 直接食用生食不会立刻造成灾难，但每次食用会带来轻微代价：  
* 信任小幅下降  
* 恐慌小幅上升  
* 患病风险小幅上升  
前期人口较少，短期吃几天生食可以接受。 但长期依赖生食会逐渐伤害城市稳定。  
**熟食优先** 若有熟食，居民优先消耗熟食。 熟食不足时，才消耗生食。 若熟食和生食都不足，则进入饥饿结算。  
**食物显示口径** 食物面板应区分：  
* 熟食  
* 生食  
* 可加工熟食等价  
说明： 总食物只能作为库存理解辅助，不应只显示一个总食物数字。 生食只有在食堂可运行并完成加工时，才能转化为熟食。 食堂停工时，生食不能自动视为熟食。  
⸻  
**4. 食堂加工** 食堂负责将生食加工为熟食。  
加工比例采用全局口径：  
* 1 生食 = 2 熟食  
当天产出的生食可在同一天被食堂加工。 当天加工出的熟食可在同一天被居民食用。  
食堂最多 1 座。 如果加工速度不足，后续通过以下方式平衡：  
* 食堂升级  
* 厨房设备改良  
* 延长供餐工时  
* 配给法令  
* 调整单日加工量  
不通过无限建造食堂解决。  
**食堂停工** 若食堂停工，当日不能加工熟食。  
可能导致：  
* 居民改吃生食  
* 熟食不足  
* 信任下降  
* 恐慌上升  
* 患病风险上升  
说明： 食堂加工不是复制食物，而是按照全局比例将生食转为熟食。 若食堂无法运行，生食不会自动折算为熟食。  
⸻  
**5. 饥饿规则** 1.0 不追踪单个居民饥饿天数。 饥饿按人口池结算。  
饥饿状态分为：  
* 轻度饥饿人口  
* 重度饥饿人口  
* 濒饿人口  
**饥饿推进** 每日统计未吃饱人口。  
示例： 第 1 天：60 人中有 10 人没饭吃。 这 10 人进入轻度饥饿。  
若后续仍然食物不足，这批饥饿人口会逐步进入：  
```
轻度饥饿 → 重度饥饿 → 濒饿 → 死亡风险

```
新的食物缺口会产生新的轻度饥饿人口。 旧的饥饿人口若继续得不到食物，会逐层恶化。  
**饥饿后果** 饥饿不会直接减少工作人口。  
饥饿主要造成：  
* 信任小幅下降  
* 恐慌小幅上升  
* 患病风险小幅上升  
若长期饥饿导致死亡，则死亡事件再造成：  
* 信任明显下降  
* 恐慌明显上升  
**饥饿死亡** 饥饿不会第一天立刻死人。 只有濒饿人口继续得不到食物时，才开始出现死亡。  
具体恶化天数和死亡比例留到平衡阶段。  
⸻  
**6. 燃料与炉心关闭** 炉心优先使用煤炭。  
煤炭不足时：  
* 炉心不会自动燃烧木材。  
* 系统在 end_day 前给出明确警告。  
* 玩家确认后，按供暖不足日结算。  
* 炉心可能因燃料不足降档或关闭。  
* 不模拟具体第几小时熄火。  
* 不做隐藏半日熄火。  
木材应急燃烧必须满足：  
* 已解锁相关科技或城约权限。  
* 玩家手动开启木材应急燃烧。  
签署城约或解锁科技只代表获得权限。 不会自动判定燃烧木材。  
设计原因： 避免 AI 或玩家无意识把建材烧空。  
⸻  
**7. 炉心关闭与温度风险** 炉心可以关闭。 在较高温度阶段，关闭炉心可以节省煤炭，但会承担健康风险。  
暂定温度段方向：  
* -10 到 -20：不开炉心，少量患病风险。  
* -30 到 -40：不开炉心，大量患病风险，少量冻伤风险。  
* -50 到 -60：不开炉心，直接出现冻伤和死亡风险。  
具体温度数值后续由天气 Patch 和平衡表确定。  
设计目的： 允许玩家在前期较暖天气中冒险屯煤。 但天气越冷，不开炉心的代价越高。  
⸻  
**8. 炉心档位区域效果** 炉心普通档位为 1～3 档。  
**1 档：节煤运行** 效果方向：  
* 内环勉强可住。  
* 中环寒冷。  
* 外环危险。  
适合短期节煤，不适合长期稳定运行。  
**2 档：标准运行** 效果方向：  
* 内环稳定。  
* 中环偏冷，可作业，但小幅增加患病风险。  
* 外环中到大幅增加患病、冻伤或事故风险。  
适合常规运行。  
**3 档：高功率运行** 效果方向：  
* 内环温暖。  
* 中环基本可工作。  
* 外环仍有小到中幅风险。  
适合寒潮、病潮或关键生产阶段。  
⸻  
**9. 过载规则** 过载不是普通 4 档 / 5 档。 过载是在当前炉心档位基础上的短期强化。  
过载需要研究解锁。  
**过载 I** 效果方向：  
* 内环更稳定，但不提供额外 buff。  
* 中环风险明显下降。  
* 外环风险下降一级。  
* 炉心压力上升。  
* 煤炭消耗增加。  
**过载 II** 效果方向：  
* 内环更稳定，但不提供额外 buff。  
* 中环接近安全。  
* 外环可短时作业，但仍不适合长期作业。  
* 炉心压力大幅上升。  
* 煤炭消耗大幅增加。  
内环不会因为过载而获得“过热加成”。 也暂不设置过热惩罚。  
设计原因： 避免额外引入“热量 buff / 过热 debuff”系统。 过载的核心代价是煤耗和炉心压力。  
⸻  
**10. 应急加热** 应急加热对单个建筑当日生效。  
规则：  
* 消耗煤炭。  
* 每日次数有限。  
* 目标建筑温度足够时，系统提示无需加热，不消耗煤炭。  
* 当日结束后，应急加热自动关闭。  
* 次日若仍然低温，需要玩家重新选择是否加热。  
如果次日天气回暖到安全范围，建筑不需要再次加热。  
说明： heat 命令开局可用。 应急加热装置是 heat 强化科技，不是 heat 解锁科技。  
⸻  
**11. 资源产出来源**  
**煤炭来源**  
* 小型采煤机  
* 大型煤矿机  
* 散落煤炭资源点  
* 固定事件奖励  
**钢铁来源**  
* 小型采钢机  
* 大型钢铁矿机  
* 散落钢铁资源点  
* 固定事件奖励  
**木材来源**  
* 森林区 / 伐木场  
* 散落木材资源点  
* 建筑拆除返还  
* 固定事件奖励  
**生食来源**  
* 狩猎小屋  
* 温室  
* 固定事件奖励  
**熟食来源**  
* 食堂加工生食  
说明： V1 不设置散落食物点。 修造工坊 / 工坊不进入 V1 正式建筑表，不作为资源产出或资源用途接口。  
⸻  
**12. 工作分配与产出比例** 所有工作人口都按整数统计。 不会出现 0.3 个人、86.2 个居民这类小数人口。  
每个工作建筑 / 工作区域只要至少分配 1 人，就可以开工。 人手不足时，不直接停工，而是按比例降低产出。  
示例：  
```
大型煤矿标准人手：10 人
实际分配：8 人
当日产出：基础产出 × 80%

```
日结产出结果按整数处理。 具体使用四舍五入或向下取整，留到源码阶段统一决定。  
设计原因：  
* 避免少量缺人导致建筑完全停摆。  
* 避免死亡潮后全城生产链卡死。  
* 保持人口和产出显示直观。  
⸻  
**13. 低温工作风险** 1.0 不做复杂效率倍率。 低温对工作场所的主要影响是风险，而不是复杂减产。  
方向：  
* 建筑能运行：正常按人手比例产出。  
* 低温作业：增加患病、冻伤、事故风险。  
* 极寒 / 暴雪 / 第七霜落：部分外环作业可能停工。  
* 关键建筑温度不足：按前置规则停工。  
具体哪些建筑停工，由建筑温度分类与天气 Patch 共同决定。  
⸻  
**14. 仓储系统** 1.0 不做分类仓库。 所有资源共用总仓储容量。  
计入仓储的资源包括：  
* 煤炭  
* 木材  
* 钢铁  
* 生食  
* 熟食  
仓库只提供仓储上限，不指定资源类型。  
设计原因： 分类仓库会增加建造选择、仓库改类型、单项爆仓、AI 调度等复杂度。 1.0 优先保证代码简单、AI 好理解、玩家好旁观。  
分类仓库可作为 2.0 或困难模式扩展。  
⸻  
**15. 爆仓规则** 当资源总量超过仓储容量时，进入临时超仓状态。  
**临时超仓** 事件奖励、固定增员携带物资等情况，可以造成临时超仓。 临时超仓资源不会立刻消失。  
**超仓后果** 当总资源高于仓储容量时：  
* 日常采集和生产无法继续入库。  
* 玩家需要消耗资源、建造仓库或升级仓储。  
* 资源总量降回仓储容量以内后，日常生产恢复。  
**食堂加工特例** 食堂加工采用全局口径：1 生食 = 2 熟食。 因此“食堂加工不增加资源总量”的旧表述作废。  
爆仓状态下是否允许食堂继续加工，需按仓储规则判断：  
* 若熟食入库会导致仓储占用继续超过上限，则按爆仓限制处理。  
* 若后续平衡表另有食物仓储特例，以后续封存口径为准。  
  
**16. 资源与生存日结局部顺序** 本节只提供资源、供暖、食物、疾病、信任 / 恐慌变化的局部结算顺序。 最终全局日结顺序以后续 007 / 008 / 009 及全局总控为准。 003 不单独决定事件、承诺、终局与硬失败总流程。  
**第一步：读取当天状态** 读取：  
* 天数  
* 天气  
* 炉心档位  
* 是否过载  
* 是否开启木材应急燃烧  
* 是否使用应急加热  
* 建筑状态  
* 人口状态  
* 当前资源  
* 仓储状态  
**第二步：结算燃料与供暖支付** 结算：  
* 炉心档位煤耗  
* 过载额外煤耗  
* 应急加热煤耗  
* 特殊建筑运行煤耗  
规则：  
* 今日炉心供暖消耗读取日结前煤仓。  
* 今日采集 / 生产的煤炭在本日日结后入库。  
* 今日生产不能自动支付今日炉心燃料缺口。  
若煤炭不足：  
* 检查是否手动开启木材应急燃烧。  
* 若未开启或木材不足，则按供暖不足日结算。  
* 炉心可能因燃料不足降档或关闭。  
* 不模拟具体第几小时熄火。  
**第三步：判定供暖结果** 根据燃料是否足够、炉心档位、天气、过载、应急加热，判定：  
* 各区域温度状态  
* 关键建筑是否停工  
* 住宅健康风险  
* 工作场所低温风险  
* 外环作业风险  
**第四步：结算生产与食堂加工** 根据建筑运行状态与分配人手，结算：  
* 煤炭产出  
* 木材产出  
* 钢铁产出  
* 生食产出  
* 熟食加工  
人手不足按比例产出。 停工建筑不产出。  
食堂加工比例采用全局口径：  
* 1 生食 = 2 熟食  
当天产出的生食可在同一天被食堂加工。 当天加工出的熟食可在同一天被居民食用。  
今日生产的煤炭在本步骤结算入库后，主要用于下一日供暖，不回填今日炉心燃料缺口。  
**第五步：结算仓储** 检查资源总量与仓储容量。  
若总资源超过仓储容量：  
* 标记临时超仓。  
* 日常新增资源无法继续入库。  
* 事件获得资源可造成临时超仓。  
食堂加工若会导致熟食入库继续超过仓储上限，则按爆仓限制处理。 若后续平衡表另有食物仓储特例，以后续封存口径为准。  
**第六步：结算食物消耗** 居民优先消耗熟食。 若熟食不足，则消耗生食。 若熟食和生食都不足，产生饥饿人口。  
食用生食会小幅降低信任、提高恐慌，并小幅增加患病风险。 饥饿会推进饥饿层数。  
说明： 食物面板应区分熟食、生食、可加工熟食等价。 总食物不应作为唯一显示数字。  
**第七步：结算疾病、冻伤与伤亡** 根据当天风险结算：  
* 新增患病  
* 饥饿导致的患病风险  
* 低温导致的患病风险  
* 冻伤  
* 伤残  
* 重症恶化  
* 死亡  
医疗停工时，恶化与死亡风险上升。  
**第八步：结算信任与恐慌** 根据当天情况调整：  
* 生食食用  
* 饥饿  
* 死亡  
* 无住所  
* 医疗崩溃  
* 寒冷  
* 法令效果  
* 危机事件  
具体数值留到平衡阶段。

**20. Patch 003 总结** 本 Patch 确定了《炉心残冬》第一版资源与每日结算方向：  
* 核心资源为煤炭、木材、钢铁、生食、熟食、仓储容量。  
* 居民开局即可食用生食。  
* 生食会小幅降低信任、提高恐慌，并增加患病风险。  
* 熟食由食堂加工，食堂加工比例采用全局口径：1 生食 = 2 熟食；居民优先食用熟食。  
* 食物面板应区分熟食、生食、可加工熟食等价，不应只显示一个总食物数字。  
* 饥饿采用人口池层数，不第一天立刻死亡。  
* 长期饥饿会进入濒饿状态，并产生死亡风险。  
* 煤炭是主要燃料。  
* 今日炉心供暖消耗读取日结前煤仓。  
* 今日采集 / 生产的煤炭在本日日结后入库，主要用于下一日供暖。  
* 今日生产不能自动支付今日炉心燃料缺口。  
* 木材可作为应急燃料，但不会自动燃烧。  
* 木材应急燃烧需要解锁权限后手动开启。  
* 煤炭不足时不做隐藏半日熄火。  
* 炉心可以关闭，但低温下会造成患病、冻伤和死亡风险。  
* 炉心 1～3 档影响内环、中环、外环的安全程度。  
* 过载提高区域抗寒能力，但增加煤耗与炉心压力。  
* 应急加热当日生效，次日自动关闭。  
* 工作区至少分配 1 人即可开工。  
* 人手不足按比例降低产出。  
* 人口与岗位均按整数统计。  
* 1.0 使用统一仓储容量，不做分类仓库。  
* 临时超仓不会销毁资源，但会阻止日常生产继续入库。  
* 爆仓状态下食堂加工需按仓储规则判断，不再使用“食堂加工不增加资源总量”的旧表述。  
* 003 只提供资源与生存日结局部顺序；最终全局日结顺序以后续 007 / 008 / 009 及全局总控为准。  
* 硬失败统一引用全局口径：存活人口 = 0、炉心崩毁、信任 = 0 且未被终火誓约承接、恐慌 = 100 且未被最高戒令压制。  
* 危机倒计时只作为预警 / 事件链 / 连锁升级机制，不新增独立硬失败轴。  
* 第七霜落未撑过、低综合分、多系统崩坏、低于城市连续性参考线不作为 003 独立硬失败条件。  
* 狩猎小屋受当前可用猎区数量限制；第 2 猎区开启方式未封存，不进入代码窗。  
* status 面板中煤炭显示使用“今日供暖支付 / 今日生产入库 / 日结后库存 / 可完整供暖天数 / 缺口预警”，不使用单一小数天数作为主显示。  
核心原则： 玩家不用手算每一块煤、每一口饭。 系统负责结算清楚，面板负责提示清楚。 玩家只需要判断：今天要不要烧炉、要不要冒冷、要不要吃生食、要不要扩仓、要不要牺牲未来换今天活下去。
