# A 【炉心残冬·Patch 005 干净封存版】  
版本定位：天气系统、固定 55 天曲线、有效温度、停工阈值与寒冷暴露封存 用途：供后续天气表、日结算函数、建筑停工判定、终局阶段继承  
   
⸻  
   
## 一、合并后的干净版正文  
**1. 本 Patch 只处理什么**  
Patch 005 只处理：  
* V1.0 固定 55 天天气曲线  
* 每日基础温度  
* 天气温度显示口径  
* 每日有效温度  
* 天气类型  
* 天气预报  
* 炉心档位对区域温度的影响  
* 建筑直接停工阈值  
* 寒冷暴露等级  
* 连续暴露递增伤害  
* 黑霜整备日  
* 第七霜落天气结构  
本 Patch 不重复处理：  
* 建筑占槽  
* 建筑建造上限  
* 资源日结算  
* 人口分类  
* 城约具体效果  
* 科技具体效果  
* 随机事件文本  
* 结局评分  
   
⸻  
   
**2. 天气系统核心原则**  
《炉心残冬》的天气不是背景板，而是主要敌人。  
V1.0 天气系统采用：  
```
固定 55 天天气表 + 有效温度判定 + 明确停工阈值 + 连续寒冷暴露递增伤害

```
V1.0 暂不采用全随机天气。  
设计原因：  
* 方便 AI 玩家学习流程。  
* 方便人类旁观者复盘。  
* 避免普通版本因为随机坏天气直接卡死。  
* 方便后续调平衡。  
* 降低代码复杂度。  
V2.0 可再加入阶段天气池、权重随机、天气种子和困难模式随机天气。  
   
⸻  
   
**3. 天气温度显示口径**  
V1.0 保留后台具体整数天气表。  
后台计算仍使用具体整数温度，例如：  
```
-12
-38
-56
-65
-76

```
这些数值用于：  
```
建筑有效温度计算
供暖风险计算
寒冷暴露判断
建筑停工判断
第七霜落终局验收

```
玩家主面板显示：  
```
真实基础温度 + 整十级天气标签

```
示例：  
```
基础温度：-38（-40级寒潮）
基础温度：-56（-60级深冬）
基础温度：-65（-70级黑霜回压）
基础温度：-76（-80级第七霜落）

```
说明：  
```
真实数值用于计算。
整十级标签用于玩家理解。

```
**整十级标签规则**  
温度标签按接近的整十级显示：  
```
-10级：残暖 / 初寒
-20级：寒风
-30级：降温
-40级：寒潮
-50级：深冬
-60级：深冬 / 黑霜
-70级：第七霜落
-80级：第七霜落极限

```
标签不是新的计算值。  
```
不要用整十级标签替代后台真实温度。
不要让建筑停工读取整十级标签。
不要让煤耗读取整十级标签。

```
天气表不需要全部改成整十数值。  
保留具体整数波动，用于表现天气变化感：  
```
-38 可以显示为 -40级寒潮。
-56 可以显示为 -60级深冬。
-65 可以显示为 -70级黑霜回压。

```
后续若微调天气表，应避免过多无意义碎数。  
推荐原则：  
```
大阶段按整十级推进。
小波动用具体整数制造天气感。
玩家主面板用整十级标签帮助理解。

```

**4. 显示方式**  
游戏对玩家显示：  
* 当前第几天  
* 当前天气  
* 当日基础温度  
* 整十级天气标签  
* 各区域有效温度或风险状态  
* 明日预报  
* 三日趋势  
* 主要天气预警  
示例：  
```
第 18 天 / 55 天

天气：寒潮
基础温度：-32（-30级降温 / 寒潮边缘）
明日预报：暴雪，继续降温
三日趋势：外环生产风险升高
预警：中环作业患病风险上升，外环作业可能冻伤。

```
说明：  
基础温度是当天天气温度。 有效温度是经过炉心档位、过载、应急加热、区域位置和建筑保温修正后的结算温度。  
建筑是否停工、居民是否受冻，按有效温度判断，不只看基础温度。  
   
⸻  
   
**5. 不做昼夜双温度**  
V1.0 不单独计算白天温度和夜晚温度。  
每日只使用一个“当日基础温度”，再通过炉心、区域、建筑状态生成有效温度。  
设计原因：  
* 减少代码复杂度。  
* 减少 AI 玩家计算负担。  
* 避免每天拆成白天 / 夜晚两套结算。  
* 夜晚的危险通过 end_day 结算与寒冷暴露体现。  
   
⸻  
   
**6. 天气类型**  
V1.0 暂定天气类型如下。  
**残暖**  
前期少见回暖天气。  
方向：  
* 温度相对较高。  
* 玩家可低档运行炉心，甚至短期关闭炉心。  
* 适合采集、建造和囤煤。  
* 仍有少量患病风险。  
**寒风**  
常规寒冷天气。  
方向：  
* 炉心需要保持运行。  
* 中环开始有轻度风险。  
* 外环作业风险高于内环与中环。  
**降温**  
过渡天气。  
方向：  
* 温度下降。  
* 提示寒潮或暴雪可能接近。  
* 主要用于预警，不用于突然处决玩家。  
**寒潮**  
中高压天气。  
方向：  
* 炉心低档风险明显。  
* 中环作业患病风险上升。  
* 外环作业可能冻伤或事故。  
* 关键建筑更可能需要应急加热。  
**暴雪**  
特殊天气灾害。  
方向：  
* 外环作业大幅受限。  
* 狩猎小屋停工。  
* 散落资源采集可能停工。  
* 大型资源区风险提高。  
* V1.0 暴雪不直接摧毁建筑。  
**极寒**  
后期高压天气。  
方向：  
* 低档炉心难以支撑城市。  
* 中环不再天然安全。  
* 外环高度危险。  
* 连续供暖不足会导致冻伤、伤残和死亡递增。  
**黑霜**  
第七霜落前兆天气。  
方向：  
* 温度极低。  
* 外环生产明显不稳定。  
* 恐慌更容易上升。  
* 用于提醒玩家终局逼近。  
**假性回温**  
黑霜期中的短暂整备窗口。  
方向：  
* 温度暂时回升，但不代表安全。  
* 给玩家最后补仓、补住房、补医疗、调整人手的机会。  
* 用于制造“还能抢救”的窗口。  
**第七霜落**  
终局天气阶段。  
方向：  
* 第 49～55 天持续压迫。  
* 外环作业基本不可依赖。  
* 炉心、燃料、住房、医疗、食物、信任、恐慌全部接受最终检验。  
* 低档炉心或燃料不足会造成严重后果。

**8. 温度计算规则**  
V1.0 所有温度均使用整数。  
包括：  
* 当日基础温度  
* 炉心档位修正  
* 过载修正  
* 应急加热修正  
* 建筑保温修正  
* 住房保温修正  
* 科技修正  
* 最终有效温度  
V1.0 不使用小数温度，不使用百分比温度修正。  
有效温度计算方向：  
```
有效温度 = 基础温度 + 区域供暖修正 + 过载修正 + 建筑 / 住房 / 科技修正

```
若后续平衡阶段意外出现小数，统一四舍五入为整数后再进行停工、患病、冻伤和死亡结算。  
设计原因：  
* 降低代码复杂度。  
* 避免浮点误差。  
* 方便 AI 玩家理解。  
* 方便人类旁观者复盘。  
* 避免出现 -37.5℃、86.2 个居民、今日死亡 2.4 人这类不适合文字生存游戏的显示。  
   
⸻  
   
**9. 炉心档位与区域有效温度**  
有效温度用于判断：  
* 建筑是否停工  
* 居民是否患病  
* 是否冻伤  
* 是否进入寒冷暴露  
* 是否产生死亡风险  
**炉心关闭**  
```
内环：无炉心加成
中环：无炉心加成
外环：无炉心加成

```
**炉心 1 档**  
```
内环：明显改善
中环：小幅改善
外环：几乎无改善

```
**炉心 2 档**  
```
内环：稳定改善
中环：中等改善
外环：小幅改善

```
**炉心 3 档**  
```
内环：强改善
中环：明显改善
外环：有限改善

```
**过载 I**  
```
内环：更稳定，但不提供额外 buff
中环：风险明显下降
外环：风险下降一级
炉心压力上升
煤耗增加

```
**过载 II**  
```
内环：更稳定，但不提供额外 buff
中环：接近安全
外环：可短时作业，但仍不适合长期
炉心压力大幅上升
煤耗大幅增加

```
说明：  
内环不会因为过载获得“过热加成”。 V1.0 不加入过热惩罚系统。

**10. 建筑直接停工阈值**  
V1.0 不使用“停工概率”。  
建筑是否停工按有效温度阈值判定。  
达到阈值以下，直接停工。 未达到停工线，则可以运行，但可能增加患病、冻伤或事故风险。  
**关键建筑阈值方向**  
```
医疗站：建筑有效温度低于 -35，停治。
医院：建筑有效温度低于 -45，停治。
食堂：建筑有效温度低于 -35，停工。
温室：建筑有效温度低于 -35，停产。
儿童保护所：建筑有效温度低于 -30，保护功能失效或大幅下降。
酒馆：建筑有效温度低于 -35，停业。
赌场：建筑有效温度低于 -35，停业。

```
**准关键建筑**  
学堂属于准关键建筑。  
规则方向：  
* 建筑有效温度过低时，停课。  
* 停课时不能产生医疗学徒 / 工程学徒效果。  
* 学堂不归入普通工作建筑。  
* 学堂具体停课阈值待后续建筑表 / 温度表确认。  
**普通工作建筑**  
研究所、工程研究站、小型采煤机 / 小型采钢机等普通工作建筑，一般不因低温概率停工。  
  
说明：修造工坊 / 工坊不进入 V1 正式建筑表，不作为普通工作建筑。  
  
规则方向：  
* 未达极端阈值时：继续运行，但增加患病 / 冻伤 / 事故 / 研究风险。  
* 极寒或终局特殊天气下：可由天气规则强制停工或高风险提示。  
**外环与资源点建筑**  
```
狩猎小屋：暴雪 / 黑霜 / 第七霜落期间停工。
散落煤炭 / 散落木材 / 散落钢铁采集：暴雪或外环有效温度过低时停工。
大型煤矿机：极寒下可运行但高风险；第七霜落期间不建议依赖。
大型钢铁矿机：同大型煤矿机。
伐木场：极寒下高风险；暴雪和第七霜落期间可能停工。

```
说明：  
* 地表小型煤堆 / 木材堆 / 钢铁堆是前期采集点，不是建筑。  
* V1 不设置散落食物点。  
* 猎区不因狩猎次数或累计产出而枯竭。  
* 狩猎小屋仍受当前可用猎区数量限制；第 2 猎区开启方式未封存，不进入代码窗。  
设计原因：  
* 停工规则清楚。  
* AI 不需要猜概率。  
* 玩家可以通过炉心档位、过载、应急加热或提前准备避免停工。

**11. 停工预警**  
直接停工必须配合明确预警。  
系统应在 status 或 city 面板中提前提示。  
示例：  
```
明日预警：
- 明日基础温度将降至 -40。
- 温室#1 若不使用应急加热，将停产。
- 医疗站#1 可能低于运行温度。
- 外环采集存在冻伤风险。

```
设计原则：  
玩家可以被惩罚，但不应被无提示处决。 AI 玩家必须能根据预警做出操作。  
end_day 风险预警按全局规则分为：  
```
A 信息提示：不阻塞
B 强警告：需要 confirm
C 硬阻塞：必须处理或二次确认特殊命令

```

**12. 寒冷暴露机制**  
低温不会直接清空人口。  
寒冷通过暴露等级推动人口状态链恶化：  
```
健康 → 患病 → 重症 → 死亡 / 伤残
健康 / 患病 → 冻伤
冻伤严重者 → 伤残或死亡

```
这继承 Patch 002-B 的人口状态链，不推翻前置规则。  
**何时进入寒冷暴露**  
当城市或部分人口处于以下状态时，进入寒冷暴露结算：  
* 炉心关闭  
* 燃料不足导致炉心降档或关闭  
* 住房不足  
* 住宅有效温度过低  
* 外环作业暴露  
* 暴雪中外出作业  
* 医疗停工导致病人无法恢复  
* 第七霜落期间保护不足  
   
⸻  
   
**13. 寒冷暴露等级**  
V1.0 可使用 0～5 级寒冷暴露等级。  
**等级 0：安全**  
方向：  
* 不产生额外寒冷伤害。  
* 正常结算疾病与食物。  
**等级 1：轻度暴露**  
方向：  
* 少量新增患病。  
* 不产生冻伤。  
* 不直接死亡。  
**等级 2：中度暴露**  
方向：  
* 新增患病明显增加。  
* 少量患病人口恶化。  
* 极少量冻伤。  
**等级 3：重度暴露**  
方向：  
* 大量新增患病。  
* 患病加速恶化为重症。  
* 中量冻伤。  
* 重症人口开始明显死亡。  
**等级 4：灾害暴露**  
方向：  
* 大量冻伤。  
* 少量普通人口死亡。  
* 重症死亡加快。  
* 伤残人口增加。  
* 信任明显下降。  
* 恐慌明显上升。  
**等级 5：终局暴露**  
方向：  
* 死亡明显增加。  
* 冻伤和伤残显著增加。  
* 医疗压力迅速失控。  
* 信任大幅下降。  
* 恐慌大幅上升。  
* 仍不直接无条件全灭。  
   
⸻  
   
**14. 连续暴露递增制**  
寒冷死亡采用连续暴露递增制。  
系统记录“连续严重寒冷暴露天数”。  
当城市连续处于严重供暖不足、极寒、无住所、外环暴露、医疗停工等状态时，暴露计数增加。  
连续暴露天数越多，患病、冻伤、重症恶化和死亡越严重。  
**连续暴露方向**  
```
连续暴露第 1 天：
大量新增患病，少量冻伤，不直接死亡或极少死亡。

连续暴露第 2 天：
患病快速恶化，冻伤增加，重症开始死亡。

连续暴露第 3 天：
大量冻伤，少量普通人口死亡，重症死亡明显增加。

连续暴露第 4 天及以后：
每日死亡递增，伤残增加，信任大幅下降，恐慌大幅上升。

```
**暴露中断**  
若玩家恢复供暖、补足住房、减少外环暴露或恢复医疗，则连续暴露计数降低或清零。  
但已经产生的后果不会自动消失：  
* 患病人口仍需治疗。  
* 重症人口仍占用医疗容量。  
* 冻伤人口可能转为伤残。  
* 死亡已经发生，无法回滚。  
设计目的：  
玩家有容错。 一天失误不会立刻全城死亡。 但连续失误会滚雪球，直到名单每天变短。  
   
⸻  
   
**15. 单日死亡上限**  
普通难度下，寒冷死亡应设置单日上限。  
方向：  
* 避免一天直接清空城市。  
* 允许玩家从一次失误中尝试补救。  
* 第七霜落期间可提高上限，但仍不无条件全灭。  
示例方向：  
```
普通阶段：单日寒冷直接死亡不超过总人口一定比例。
黑霜阶段：单日上限提高。
第七霜落阶段：单日上限进一步提高。

```
具体比例留到平衡阶段。  
说明：  
单日死亡上限不阻止总崩盘。 如果玩家连续无煤、无食、无医疗、无住房，城市仍会逐日崩溃。  
   
⸻  
   
**16. 死亡不是单一温度触发**  
V1.0 不采用：  
```
温度低于某个值，人口直接全死。

```
改为：  
```
温度越低，暴露等级越高；
连续暴露越久，死亡越重。

```
直接死亡只在以下高风险条件下出现：  
* 极端低温  
* 炉心关闭  
* 燃料不足  
* 无住所  
* 外环暴露  
* 医疗崩溃  
* 第七霜落期间保护不足  
这样既保留绝望感，也保留普通难度的容错。  
   
⸻  
   
**17. 天气预报机制**  
V1.0 天气固定，但玩家仍通过预报获得信息。  
**明日预报**  
明日预报较准确。  
示例：  
```
明日预报：暴雪，温度可能降至 -40 以下。

```
**三日趋势**  
三日趋势较模糊，但能提示方向。  
示例：  
```
三日趋势：寒潮正在逼近，外环作业风险将持续升高。

```
**终局预警**  
第七霜落前必须多次预警。  
示例：  
```
黑霜正在外墙上蔓延。第七霜落已不再只是传闻。

```
设计目的：  
玩家可以被逼入绝境，但必须知道绝境正在靠近。  
   
⸻  
   
**18. 天气对外环生产的影响**  
外环是最危险区域。  
天气对外环影响方向：  
* 常规寒风：可作业，但风险高于内环与中环。  
* 寒潮：患病、冻伤、事故风险明显上升。  
* 暴雪：狩猎与散落采集停工。  
* 极寒：外环作业高度危险。  
* 黑霜：外环生产明显不稳定。  
* 第七霜落：外环作业基本不可依赖。  
设计目的：  
玩家不能长期把生存建立在外环稳定产出上。 前期可以靠外环抢资源，后期必须靠储备、科技、炉心和路线选择活下去。  
   
⸻  
   
**19. 黑霜整备日**  
黑霜期中间设置短暂回温。  
暂定：  
```
第 39～43 天：黑霜初现，压力拉高。
第 44～45 天：假性回温，最后整备。
第 46～48 天：黑霜回压，终局警告。
第 49～55 天：第七霜落。

```
假性回温的作用：  
* 给玩家补仓机会。  
* 给玩家调整人手机会。  
* 给玩家最后补住房、医疗、食堂、仓储的机会。  
* 让终局前有一次喘息和心理落差。  
说明：  
假性回温不是安全。 它只是寒冬在合拢前，给玩家最后一次看清缺口的机会。  
   
⸻  
   
**20. 第七霜落规则方向**  
第七霜落是终局天气阶段，不是单日随机事件。  
第七霜落期间：  
* 温度维持极低区间。  
* 外环作业基本不可依赖。  
* 炉心燃料消耗极高。  
* 低档炉心会造成严重后果。  
* 医疗容量不足会导致死亡扩大。  
* 食物不足会推动恐慌与死亡。  
* 住房不足会被放大惩罚。  
* 信任与恐慌进入最终压力测试。  
**第七霜落不是翻盘阶段**  
进入第七霜落后，玩家不应主要依赖：  
* 新建大量建筑  
* 外环采集补资源  
* 临时狩猎补食物  
* 长期低档省煤  
玩家应依赖：  
* 之前储备的煤炭  
* 之前准备的食物  
* 住房容量与保温  
* 医疗容量  
* 城约路线  
* 科技成果  
* 炉心过载  
* 应急加热  
设计目的：  
第七霜落是总考试，不是新手教程。  
   
⸻  
   
**21. 天气与 AI 玩家**  
AI 玩家不需要自己推导复杂公式。  
系统应直接提示：  
* 是否需要提高炉心档位  
* 是否适合外环作业  
* 是否应该使用应急加热  
* 是否有建筑即将停工  
* 是否应该停止危险岗位  
* 是否应该提前储煤  
* 是否应该优先建医疗或住房  
* 是否进入终局准备阶段  
AI 玩家根据预警做决策。 系统负责把天气风险翻译成可执行信息。  
   
⸻

**24. Patch 005 总结**  
本 Patch 确定《炉心残冬》第一版天气系统方向：  
* V1.0 使用固定 55 天天气曲线。  
* V1.0 不做全随机天气。  
* V2.0 再扩展随机天气池和天气种子。  
* 每天使用一个基础温度，不拆白天 / 夜晚温度。  
* 后台温度使用具体整数。  
* 玩家面板显示真实基础温度 + 整十级天气标签。  
* 整十级天气标签不参与建筑停工、供暖、暴露或终局验收计算。  
* 建筑停工按有效温度阈值直接判定，不做停工概率。  
* 建筑是否停工看有效温度，不只看基础天气温度。  
* 关键建筑低于阈值直接停工。  
* 学堂属于准关键建筑，低温停课，具体阈值待后续建筑表 / 温度表确认。  
* 普通工作建筑低温时主要增加患病、冻伤、事故或研究风险。  
* 暴雪主要用于打断外环生产。  
* 黑霜用于提示第七霜落逼近。  
* 黑霜期中间加入第 44～45 天假性回温整备窗口。  
* 第七霜落固定为第 49～55 天。  
* 寒冷死亡不按单一温度直接全灭。  
* 寒冷会推动健康、患病、重症、冻伤、伤残和死亡链条。  
* 严重寒冷采用连续暴露递增制。  
* 玩家恢复供暖、住房或医疗后，可以中断连续暴露递增。  
* 普通难度设置单日寒冷死亡上限，避免一天清空城市。  
* 第七霜落提高压力，但仍不无条件全灭。  
* AI 玩家通过预警做决策，不需要自行计算隐藏概率。  
核心原则：  
冬天不是突然杀死炉城。 它先让玩家觉得还能撑一撑，再让每一次“撑一撑”都变成债。 炉心熄灭后，死亡不会立刻清空城市；它会每天拿走一些人，让玩家一天天明白，自己已经不是在规划未来，而是在决定谁还能看见明天。  
   
⸻
