## 建筑与岗位基础规则  
## 2. 建筑成本设计原则  
普通难度建筑成本应满足：  
```
玩家第一天能建几个关键建筑。
玩家第一天不能把所有问题一次性建完。
住房、食物、医疗、研究之间必须有取舍。
建筑成本尽量使用整数。
基础建筑主要消耗木材。
中高级建筑逐渐增加钢铁消耗。

```
开局木材基准：  
```
木材：100

```
因此第一天建设必须形成选择：  
```
补住房
建狩猎小屋
建食堂
建研究所
建医疗站
建采集保护设施

```
玩家不应能一口气全部拍完。  
   
⸻  
   
## 3. 建造时间口径  
V1 暂不做多日建造时间。  
规则：  
```
build 成功后立即扣资源。
build 成功后立即占槽。
build 成功后立即生成建筑编号。
当天 build 成功的生产建筑，若当天完成分配人手，则可在当天 end_day 结算中参与生产。
当天 build 成功的食堂，若当天完成分配人手，则可在当天 end_day 结算中参与加工。
当天 build 成功的医疗站，若当天完成分配人手，则可在当天 end_day 结算中参与医疗收治。

```
建筑是否当天生效，只看：  
```
是否已建成。
是否有人手。
是否满足温度 / 解锁 / 资源 / 区域条件。

```
不做：  
```
多日建造时间。
小时级施工进度。
施工队系统。
建造中建筑半效果。

```
说明：  
```
建筑成本已经承担取舍压力。
V1 不额外加入“建造需要 2 天”的施工时间系统。

```
   
⸻  
   
## 4. 食物生产与食堂加工顺序  
每日食物链结算顺序：  
```
狩猎小屋 / 温室等来源先产出生食。
食堂随后加工可用生食为熟食。
居民最后进行当日食物消耗。

```
因此：  
```
当天狩猎小屋产出的生食，可以在同一天被食堂加工。
当天食堂加工出的熟食，可以在同一天被居民食用。

```
示例：  
```
狩猎小屋当天产出 40 生食。
食堂当天将 40 生食加工为 80 熟食。
居民当日消耗 80 熟食。

```
边界：  
```
食堂只能加工当前可用生食。
若当天生食不足，则按实际可用生食加工。
若生食为 0，则食堂产出 0 熟食。
食堂不凭空生成食物。

```
说明：  
```
同日加工不是复制食物。
只是明确生产顺序，避免前期食物链出现假死锁。

```
   
⸻  
   
## 5. 建筑占槽口径  
建筑占槽沿用区域槽位系统。  
建筑规模：  
```
小型建筑：1 槽
中型建筑：2 槽
大型建筑：3 槽

```
V1 区域槽位容量统一为：  
```
内环：18 槽
中环：30 槽
外环：36 槽
外缘储放区：12 槽

```
正式代码区域名统一为：  
```
内环
中环
外环
外缘储放区

```
说明：  
```
“边缘区”只作为外环的玩家显示别名或旧文案兼容名。
代码字段不得新增独立的“边缘区”区域。
玩家面板可以显示为：外环（边缘区）。

```
特殊口径：  
```
地表资源点不是普通建筑槽。
避寒采集棚绑定地表资源点，不占普通建筑槽。
大型煤矿点 / 大型钢铁矿点是资源锚点，不直接占普通建筑槽。
绑定大型矿点建造的大型矿机，占外环建筑槽。
炉心临时仓储点不占普通仓库槽。

```
槽位说明：  
```
本版使用内环 18 / 中环 30 / 外环 36 / 外缘储放区 12 作为 V1 覆盖口径。
旧槽位表 14 / 22 / 28 / 10 作废。

```

## 6. 建筑区域字段标准化  
建筑表中使用两个字段：  
```
allowed_zones：可建区域
recommended_zone：推荐区域

```
显示给玩家时：  
```
“区域：外环”
表示只能建在外环。

“推荐区域：内环”
表示推荐建在内环，但并非唯一合法区域。

```
代码字段示例：  
```
基础住宅：
allowed_zones = ["inner", "middle"]
recommended_zone = "inner"

狩猎小屋：
allowed_zones = ["outer"]
recommended_zone = null

```
说明：  
```
推荐区域只影响提示，不作为硬限制。
区域或 allowed_zones 才是硬限制。
“边缘区”不得作为独立 allowed_zones 字段。

```

## 7. 岗位资格总规则  
V1 岗位分为三类。  
**A. 普通劳动岗位**  
允许：  
```
工人
工程师
儿童辅工：仅限对应炉律允许范围
学徒：仅限岗位或炉律明确允许时

```
示例：  
```
采集
狩猎小屋
食堂
部分温室岗位
部分普通生产岗位

```
说明：  
```
工程师可以下放到普通岗位。
儿童辅工不能绕过炉律限制。
学徒不是全体普通岗位通行证。

```
**B. 工程 / 研究岗位**  
允许：  
```
工程师
工程学徒

```
禁止：  
```
工人
普通儿童
儿童辅工
医疗学徒

```
示例：  
```
研究所
部分工程项目

```
说明：  
```
工程学徒最终岗位为研究所。
修造工坊 / 工坊不进入 V1 正式建筑表，因此不再作为工程学徒岗位。

```
**C. 医疗岗位**  
允许：  
```
工程师
医疗学徒

```
禁止：  
```
工人
普通儿童
儿童辅工
工程学徒

```
示例：  
```
医疗站
医院

```

```

## 12. 食堂  
```
建筑：食堂
占槽：2
可建区域：内环 / 中环
成本：20 木材，5 钢铁
岗位容量：5 工人
建筑上限：1
最低运行温度：待 008-3 确认

```
基础加工能力：  
```
每 1 生食 → 2 熟食
满员时每日最多加工 60 生食
即最多产出 120 熟食

```
人手不足时：  
```
按岗位比例降低加工上限。
向下取整。

```
示例：  
```
5 / 5 人：最多加工 60 生食 → 120 熟食
3 / 5 人：最多加工 36 生食 → 72 熟食

```
规则：  
```
食堂不生成食物。
食堂只能加工实际存在的生食。
当天产出的生食可在当天被食堂加工。
当天加工出的熟食可在当天被居民食用。

```
   
⸻  
   
## 13. 狩猎小屋  
```
建筑：狩猎小屋
占槽：2
区域：外环
显示别名：外环（边缘区）
成本：25 木材，5 钢铁
岗位容量：15 工人
建筑上限：当前地图可用猎区数量
开局可建：是
科技前置：无
城约前置：无
是否可 heat：否

```
基础产出：  
```
满员时每日产出 40 生食

```
人手不足时：  
```
按岗位比例产出。
向下取整。

```
示例：  
```
15 / 15 人：40 生食 / 日
10 / 15 人：26 生食 / 日
5 / 15 人：13 生食 / 日

```
天气规则：  
```
普通寒冷：可运行，但有风险。
暴雪：停工。
黑霜：停工。
第七霜落：停工。

```
食堂接线：  
```
1 座狩猎小屋满员产出 40 生食。
1 座食堂可将 40 生食加工为 80 熟食。
开局 80 人每日消耗 80 食物。
因此 1 个可用猎区 + 1 座狩猎小屋 + 1 座食堂，可以接住开局 80 人基础食物线。

```
说明：  
```
狩猎小屋是前期食物链核心。
狩猎小屋不是后期唯一食物来源。
中后期必须依赖食物科技、温室、储备、配给或其他食物效率提升。

```

## 15. 医疗站  
```
建筑：医疗站
占槽：1
可建区域：内环 / 中环
成本：25 木材，10 钢铁
岗位容量：5 工程师 / 医疗学徒
医疗容量：10 人
建筑上限：4
解锁：基础医疗法
最低运行温度：待 008-3 确认

```
规则：  
```
医疗站需要工程师或医疗学徒工作。
工程师 / 医疗学徒不足时，治疗能力按比例下降。
有效温度低于最低运行温度时，医疗站停治。
医疗站当天建成并分配工程师 / 医疗学徒后，当天 end_day 即可提供医疗容量。

```
人手示例：  
```
5 / 5：治疗容量 10
3 / 5：治疗容量 6
1 / 5：治疗容量 2

```
说明：  
```
临时救护第 1～3 天有效，第 4 天直接归零。
因此普通难度必须允许玩家在第 1～3 天内通过基础医疗法进入正式医疗路径。

```

避寒采集棚遵循全局拆除返还规则。

```
008-2 不单独写死“避寒采集棚不返还”。

```
建筑：学堂
占槽：2
可建区域：内环 / 中环
成本：30 木材，10 钢铁
岗位容量：2 工程师

解锁：儿童保护后续炉律

基础作用：  
```

```

学堂不提供全体儿童保护。

说明：  
```
学堂是培养系统，不是儿童保护所的完全替代品。
玩家不能建了学堂就把儿童保护所拆掉并永久无后果。
