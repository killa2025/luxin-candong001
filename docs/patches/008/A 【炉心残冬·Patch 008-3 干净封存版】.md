## 2. 日计划制总原则  
V1 采用每日回合制 / 日计划制。  
玩家白天提交今日计划，系统在 end_day 后统一结算。  
玩家不能在日结中途追加操作。  
不做：  
```
小时级煤耗
日内实时温度
上午 / 下午 / 夜晚分段操作
炉心烧半天后熄火的隐藏过程
日结中途补煤
日结中途补食物
日结中途临时改岗位

```
规则：  
```
玩家可以在 end_day 前调整计划。
玩家按下 end_day 后，系统统一结算。
结算中产生的资源变化，进入日结后结果。

```
   
⸻  
   
## 3. end_day 前预警  
end_day 前必须预警以下内容：  
```
今日炉心燃料是否足够。
今日食物是否明显不足。
今日医疗容量是否严重不足。
今日关键建筑是否低温停工。
是否存在大量无住所人口。
过载压力是否接近红线。
信任 / 恐慌是否接近危机。
第七霜落前准备是否明显不足。

```
预警分级：  
```
A 信息提示：不阻塞 end_day。
B 强警告：需要 confirm end_day。
C 硬阻塞：不能直接 confirm，必须先处理或二次确认特殊命令。

```
说明：  
```
预警只说明风险。
预警不能直接判定失败。
系统不得仅凭预测直接失败。

```

## 4. 炉心档位  
炉心命令：  
```
furnace off
furnace 1
furnace 2
furnace 3

```
规则：  
```
玩家可在 end_day 前反复调档。
调档不消耗行动点。
只按 end_day 前最终有效炉心档位结算。

```
防调档白嫖：  
```
不能先开高档再切低档，获得高档热量、低档煤耗。
当日只读取最终有效档位进行供暖与煤耗结算。

```
   
⸻  
   
## 5. 炉心煤耗  
V1 普通难度炉心基础煤耗：  
```
furnace off：0 煤 / 日
furnace 1：45 煤 / 日
furnace 2：85 煤 / 日
furnace 3：150 煤 / 日

```
说明：  
```
炉心 1 档用于开局最低供暖。
炉心 2 档用于中期寒潮压力。
炉心 3 档用于深冬、黑霜与第七霜落前后。

```
   
⸻

furnace 3：+42

```
说明：

炉心供暖修正进入区域有效温度。

```

## 7. 开局煤炭回填  
开局煤炭正式封存为：  
```
70

```
说明：  
```
炉心 1 档日耗 45。
开局煤炭 70 可完整支付第 1 天炉心 1 档。
日结后若无新增煤，剩余 25。
第 2 天若仍开 1 档且无新增煤，会出现 20 煤缺口。

```
作废旧口径：  
```
开局煤炭 180。
开局煤炭待 008-D 回填。
开局煤炭按 1～2 天模糊口径不落数值。

```

## 8. 今日完整日耗  
今日完整日耗公式：  
```
今日完整日耗 = 当前炉心档位基础煤耗 + 过载额外煤耗 - 炉心节煤科技修正

```
说明：  
```
heat 不计入今日完整日耗。
heat 成功时即时扣煤。
woodfuel 若后续启用，另由对应系统接入，不在本 Patch 补数值。

```
   
⸻  
   
## 9. 今日供暖支付与今日生产入库  
推荐日结资源顺序：  
```
1. 读取 end_day 前已有煤炭。
2. 进行今日供暖支付校验。
3. 支付炉心燃料 / 供暖消耗。
4. 结算建筑有效温度与停工。
5. 结算生产入库。
6. 结算食物加工与消耗。
7. 结算疾病、死亡、信任、恐慌。
8. 结算事件与日结日志。

```
关键规则：  
```
今日炉心燃料读取 end_day 前已有煤炭。
今日采集 / 今日生产的煤炭在日结后入库。
今日采煤主要保障明日。
今日采煤不能自动支付今日炉心缺口。

```
面板建议显示：  
```
今日供暖支付
今日生产入库
日结后库存

```
说明：  
```
这是避免“日结中途采煤救今日炉心”的核心口径。
代码窗不得把生产入库排在供暖支付之前。

```

## 10. 供暖不足日  
若当前煤炭不足以支付目标炉心档位完整一日，玩家仍确认 end_day，则进入供暖不足日。  
规则：  
```
不模拟小时级熄火。
不计算“烧了半天”。
不做日内中途补煤。

```
当日有效供暖档位：  
```
足够支付目标档位：按目标档位正常结算。
不足目标档位但足够较低完整档位：按可支付最高完整档位供暖。
不足 1 档完整日耗：按 furnace off 供暖。
炉心原本 off：按 furnace off 供暖。

```
说明：  
```
只要目标档位未被完整支付，当日仍标记为供暖不足日。
供暖不足日会提高寒冷暴露、冻伤、患病、建筑停工、信任下降、恐慌上升风险。

```
   
⸻  
   
## 11. 炉心关闭日  
若玩家最终设置：  
```
furnace off

```
则当日为炉心关闭日。  
效果方向：  
```
炉心供暖 +0。
住宅寒冷风险大幅上升。
无住所风险大幅上升。
关键建筑停工 / 停治 / 停产风险上升。
信任明显下降。
恐慌明显上升。
若造成寒冷死亡，可触发对应事件、标签或隐藏成就。

```
说明：  
```
炉心关闭不是普通节煤按钮。
炉心关闭是极高风险操作。

```

## 过载与炉心压力  
## 12. 过载命令  
过载命令：  
```
overload off
overload 1
overload 2

```
使用条件：  
```
必须炉心处于 1 / 2 / 3 档。
furnace off 不可过载。
炉心关闭时过载自动关闭。

```
规则：  
```
过载用于救急。
过载不是免费供暖。
过载会增加煤耗与炉心压力。

```
   
⸻  
   
## 13. 过载数值  
过载数值：  
```
overload off：建筑有效温度 +0，额外煤耗 +0，压力 +0
overload 1：建筑有效温度 +8，额外煤耗 +25，压力 +18
overload 2：建筑有效温度 +14，额外煤耗 +55，压力 +35

```
说明：  
```
过载修正进入建筑有效温度。
过载额外煤耗进入今日完整日耗。
过载压力进入炉心压力系统。

```
   
⸻  
   
## 14. 炉心压力  
炉心压力范围：  
```
0～79：正常
80～99：高压警告
>=100：红线

```
红线规则：  
```
炉心压力首次达到或超过 100 时，触发红线警告。
玩家获得最后一次关闭过载机会。
红线后若继续过载并再次 end_day，触发炉心崩毁。

```
说明：  
```
炉心崩毁是系统硬失败。
炉心压力高但未崩毁，可进入终局标签或终局状态评级。

```
   
⸻  
   
## 15. 炉心压力冷却  
压力冷却规则：  
```
overload off 且炉心运行：炉心压力 -20，最低降至 0。
furnace off：炉心压力 -25，最低降至 0。

```
说明：  
```
关闭炉心可以更快冷却压力，但会造成严重寒冷风险。
关闭过载但维持炉心运行，是更常规的降压方式。

```
   
⸻  
   
## heat 应急加热  
## 16. heat 命令  
heat 是单建筑当日救急供暖。  
命令方向：  
```
heat 建筑编号
heat 建筑名称

```
数值：  
```
单次耗煤：20
目标建筑当日建筑有效温度：+18
持续时间：当日
日结后自动关闭

```
规则：  
```
heat 成功后即时扣煤。
heat 不返煤。
同一建筑同日不可重复 heat。
heat 不按建筑大小、占槽、区域、类型变化。
heat 不重复计入今日完整日耗。

```
说明：  
```
heat 是救关键建筑的短时按钮，不是全城供暖。
住宅不可 heat。

```
   
⸻  
   
## 17. heat 开局与科技接口  
V1 中：  
```
heat 开局可用。

```
前提：  
```
目标建筑支持 heat。
目标建筑当前满足可加热条件。
玩家有足够煤炭支付 heat 成本。

```
应急加热装置口径：  
```
应急加热装置不是 heat 解锁科技。
应急加热装置是 heat 强化科技。

```
强化方向可包括：  
```
降低 heat 成本。
提高 heat 效率。
提高 heat 稳定性。
降低 heat 相关风险。
扩大 heat 可用表现。

```
具体强化数值未封存，不得由代码窗猜测。  
   
⸻

```
医疗站

酒馆

说明：  
```

建筑保温不等于 heat 权限。
应急加热装置也不会让研究所自动获得 heat。

```

## 19. heat 与建筑阈值  
heat 后重新计算建筑有效温度。  
规则：  
```
若 heat 后建筑有效温度达到最低运行温度阈值，则当日可运行。
若 heat 后仍低于阈值，建筑仍停工 / 停治 / 停产。

```
说明：  
```
heat 不是必定救回建筑。
heat 只是给目标建筑当日 +18。

```

## 供暖科技接口  
## 20. 炉心节煤  
炉心节煤科技修正：  
```
无节煤：1 / 2 / 3 档 = 45 / 85 / 150
节煤 I：1 / 2 / 3 档 = 40 / 75 / 135
节煤 II：1 / 2 / 3 档 = 35 / 68 / 120

```
规则：  
```
节煤 II 覆盖节煤 I。
节煤 I 与节煤 II 不叠加。

```
说明：  
```
节煤只降低炉心档位基础煤耗。
是否降低过载额外煤耗，需由后续科技表明确。

```
   
⸻  
   
## 21. 建筑保温  
建筑保温科技修正：  
```
无保温：+0
建筑保温 I：+6
建筑保温 II：+12

```
规则：  
```
建筑保温 II 覆盖为总 +12。
建筑保温 II 不在 I 的基础上再额外 +12。
住宅也可获得保温修正。
住宅仍不可 heat。

```
说明：  
```
建筑保温改变建筑有效温度。
建筑保温不改变 heat 权限。

```
   
⸻

## 温度与建筑运行  
## 26. 天气真实温度与玩家标签  
V1 使用后台真实整数温度 + 玩家整十级标签。  
示例：  
```
D17：-38，显示 -40 级寒潮。
D19：-40，显示 -40 级暴雪。
D28：-50，显示 -50 级深冬。
D36：-58，显示 -60 级深冬。
D42：-64，显示 -60 级黑霜。
D49：-66，显示 -70 级第七霜落。
D55：-76，显示 -80 级第七霜落。

```
规则：  
```
所有计算读取后台真实温度。
整十级标签只用于玩家理解。

```
禁止：  
```
用整十级标签参与建筑停工。
用整十级标签参与疾病、冻伤、死亡计算。
用整十级标签参与终局验收。

```
   
⸻  
   
## 27. 区域有效温度  
区域有效温度公式：  
```
区域有效温度 = 后台真实天气温度 + 炉心档位供暖修正 + 区域修正

```
区域修正：  
```
内环：+5
中环：+0
外环：-6
外缘储放区：不作为主要居住 / 工作温度区域，默认按外环或专表处理

```
说明：  
```
外环为正式代码区域名。
边缘区只作为外环显示别名。

```
   
⸻  
   
## 28. 建筑有效温度  
建筑有效温度公式：  
```
建筑有效温度 = 区域有效温度 + 建筑保温科技修正 + 炉心功率稳定修正 + 过载修正 + heat 修正 + 终局炉心稳定修正

```
规则：  
```
建筑有效温度必须为整数。
建筑有效温度用于判断关键建筑停工 / 停治 / 停产。

```
说明：  
```
住宅也有有效温度概念，但住宅不可 heat。
仓库、墓园、冷藏坑等不参与普通运行温度判断。

```
   
⸻  
   
## 29. A 类关键建筑最低运行温度阈值  
A 类关键建筑：  
```
医疗站：-35，低于该值停治，可 heat。
医院：-45，低于该值停治，可 heat。
食堂：-35，低于该值停工，不能加工生食，可 heat。
温室：-35，低于该值停产，可 heat。
儿童保护所：-30，低于该值保护失效或大幅下降，可 heat。
酒馆：-35，低于该值停业，可 heat。
大赌场：-35，低于该值停业，可 heat。

```
判断规则：  
```
若建筑有效温度 < 最低运行温度阈值，则停工 / 停治 / 停产 / 失效。
注意是“小于”，不是“小于等于”。

```
示例：  
```
医疗站建筑有效温度 = -35：仍可运行。
医疗站建筑有效温度 = -36：停治。

```
   
⸻

## 人口状态与疾病链  
## 33. 人口状态  
人口状态分为：  
```
健康：存活，可工作，消耗食物。
患病：存活，不可工作，消耗食物，可治疗。
重症：存活，不可工作，消耗食物，高死亡风险。
伤残：存活，不可常规工作，消耗食物，仍可能死亡。
死亡：不存活，不工作，不消耗食物，移出存活人口。

```
人口守恒：  
```
存活人口 = 健康 + 患病 + 重症 + 伤残
总人口记录 = 存活人口 + 累计死亡

```
说明：  
```
死亡人口不属于人口分组。
伤残不是安全状态，仍可能因饥饿、寒冷、疾病、第七霜落、医疗崩溃死亡。

```
   
⸻

疾病链：

健康 → 患病 → 重症 → 死亡 / 伤残

医疗容量不足
长期食物不足
长期生食
过劳 / 加班 / 长班制
第七霜落保护不足

```
   
⸻

## 寒冷暴露  
## 36. 寒冷暴露等级  
寒冷暴露等级：  
```
0：安全
1：寒冷
2：严寒
3：极寒
4：致命寒冷

```
寒冷暴露用于：  
```
新增患病
患病转重症
重症死亡
冻伤伤残
直接寒冷死亡
信任 / 恐慌变化
事件触发候选

```
   
⸻  
   
## 37. 住宅人口暴露  
住宅人口暴露按住宅有效温度判断：  
```
住宅有效温度 >= -25：暴露等级 0
-26～-35：暴露等级 1
-36～-45：暴露等级 2
-46～-55：暴露等级 3
< -55：暴露等级 4

```
说明：  
```
住宅不可 heat。
住宅可受炉心、区域、保温等影响。

```
   
⸻  
   
## 38. 无住所人口暴露  
无住所人口暴露温度：  
```
无住所暴露温度 = 后台真实天气温度 + floor(炉心档位供暖修正 / 2)

```
暴露等级：  
```
>= -20：暴露等级 1
-21～-35：暴露等级 2
-36～-50：暴露等级 3
< -50：暴露等级 4

```
额外修正：  
```
furnace off：+1 暴露等级
供暖不足日：+1 暴露等级
暴雪 / 黑霜：+1 暴露等级
第七霜落：+1 暴露等级

```
最高不超过 4。  
说明：  
```
无住所人口最低也有暴露等级。
无住所不会因为炉心开启而完全安全。

```
   
⸻  
   
## 39. 外勤 / 露天暴露  
外勤 / 露天暴露按后台真实天气温度判断：  
```
>= -30：暴露等级 1
-31～-45：暴露等级 2
-46～-60：暴露等级 3
< -60：暴露等级 4

```
天气下限：  
```
暴雪：至少暴露等级 3
黑霜：至少暴露等级 3
第七霜落：至少暴露等级 4

```
外勤防寒装备：  
```
暴露风险下降一级。
最低不降到 1。

```
避寒采集棚：  
```
绑定资源点采集人员外勤寒冷暴露风险下降一级。
最低不降到 0。

```
说明：  
```
外勤防寒装备不改变建筑温度。
避寒采集棚不改变区域温度。

```
   
⸻  
   
## 疾病与医疗结算  
## 40. 新增患病  
基础新增患病规则：  
```
暴露等级 0：每 20 人新增 0 患病
暴露等级 1：每 20 人新增 1 患病
暴露等级 2：每 20 人新增 2 患病
暴露等级 3：每 20 人新增 3 患病
暴露等级 4：每 20 人新增 4 患病

```
公式：  
```
新增患病 = floor(暴露人口 / 20) × 等级对应数值

```
小群体保护：  
```
若暴露人口不足 20，但暴露等级为 3 / 4，且暴露人口 >= 8，则至少新增 1 患病。

```
D1 保护：  
```
D1 开局无住所新增患病最多 6。
D1 不因常规寒冷暴露大规模死亡。

```
   
⸻  
   
## 41. 疾病推进顺序  
疾病推进顺序：  
```
1. 新增患病
2. 计算医疗容量
3. 分配治疗优先级
4. 患病恢复 / 恶化
5. 重症恢复 / 死亡 / 伤残
6. 冻伤伤残
7. 寒冷 / 饥饿 / 特殊死亡
8. 更新健康池
9. 更新信任 / 恐慌
10. 向事件系统提交候选

```
说明：  
```
顺序必须固定。
不得先治病再新增当日寒冷患病，除非后续另开 Patch 修改。

```
   
⸻  
   
## 42. 医疗容量  
临时救护：  
```
D1～D3：容量 5
D4 起：容量 0

```
医疗站：  
```
岗位容量：5 工程师 / 医疗学徒
满员医疗容量：10
建筑上限：4

```
医院：  
```
岗位容量：10 工程师 / 医疗学徒
满员医疗容量：30
建筑上限：2

```
医疗站容量公式：  
```
医疗站容量 = floor(10 × 实际医疗人员 / 5)

```
医院容量公式：  
```
医院容量 = floor(30 × 实际医疗人员 / 10)

```
   
⸻  
   
## 43. 医疗停治  
医疗站停治：  
```
医疗站建筑有效温度 < -35 → 容量 0

```
医院停治：  
```
医院建筑有效温度 < -45 → 容量 0

```
heat 接口：  
```
若 heat 后建筑有效温度达到阈值，则当日正常提供容量。
若 heat 后仍低于阈值，则仍停治。

```
   
⸻  
   
## 44. 治疗优先级  
治疗优先级：  
```
1. 重症
2. 患病
3. 冻伤急救接口

```
说明：  
```
重症优先占用医疗容量。
剩余容量再治疗患病。
冻伤急救接口由伤残系统读取剩余医疗缓冲。

```
   
⸻  
   
## 45. 治疗进度池  
治疗进度池只保存正向治疗进度，不保存负向死亡 / 恶化债务。  
患病治疗进度：  
```
每日收治患病加入进度。
每累计 3 点，恢复 1 名患病为健康。
剩余进度保留下一日。
若患病人口 = 0，则清零。
最多保留 2 点。

```
重症好转进度：  
```
每日收治重症加入进度。
每累计 4 点，1 名重症好转为患病。
剩余进度保留下一日。
若重症人口 = 0，则清零。
最多保留 3 点。

```
负向转换不保留长期隐藏余数：  
```
未收治患病转重症
未收治重症死亡
未收治重症伤残
直接寒冷死亡
饥饿死亡

```
   
⸻  
   
## 46. 患病结算  
收治患病恢复：  
```
恢复人数 = floor((收治患病 + 既有患病治疗进度) / 3)

```
普通日：  
```
收治患病普通日不转重症。

```
第七霜落且医疗严重不足：  
```
收治患病仍可少量转重症：floor(收治患病 / 12)

```
未收治患病转重症：  
```
暴露等级 0～1：floor(未收治患病 / 6)
暴露等级 2：floor(未收治患病 / 5)
暴露等级 3：floor(未收治患病 / 4)
暴露等级 4：floor(未收治患病 / 3)

```
额外修正：  
```
供暖不足日：额外 + floor(未收治患病 / 6)
第七霜落：额外 + floor(未收治患病 / 6)

```
保底：  
```
若未收治患病 >= 6 且暴露等级 3 / 4，至少 1 人转重症。

```
   
⸻  
   
## 47. 重症结算  
收治重症好转：  
```
好转人数 = floor((收治重症 + 既有重症好转进度) / 4)

```
收治重症死亡：  
```
floor(收治重症 / 10)

```
若医院已建成且运行：  
```
收治重症死亡减少为 floor(收治重症 / 15)

```
未收治重症死亡：  
```
floor(未收治重症 / 3)

```
未收治重症伤残：  
```
floor(未收治重症 / 5)

```
额外死亡：  
```
供暖不足日、黑霜或第七霜落：额外 + floor(未收治重症 / 6)

```
保底：  
```
若未收治重症 > 0，医疗容量 = 0，且暴露等级 3 / 4，则至少 1 名未收治重症死亡。

```
   
⸻  
   
## 冻伤、伤残与直接死亡  
## 48. 冻伤与伤残  
V1 不单独长期追踪轻冻伤池。  
规则：  
```
轻冻伤作为患病风险处理。
重冻伤直接进入伤残池。

```
新增伤残主要来自：  
```
极寒暴露
无住所严寒暴露
外勤 / 露天作业
暴雪外出
第七霜落保护不足
未收治重症

```
暴露导致伤残：  
```
暴露等级 0～1：0
暴露等级 2：第七霜落时 floor(暴露人口 / 60)
暴露等级 3：floor(暴露人口 / 40)
暴露等级 4：floor(暴露人口 / 25)

```
医疗缓冲冻伤：  
```
剩余医疗容量每 5 点抵消 1 名新增伤残。
不能抵消到负数。
不能治疗既有伤残。

```
   
⸻  
   
## 49. 直接寒冷死亡  
直接寒冷死亡触发条件之一：  
```
暴露等级 4
连续 2 天以上暴露等级 3
furnace off 且无住所人口 > 0
供暖不足日且无住所人口 > 0
第七霜落期间保护不足

```
寒冷死亡口径：  
```
暴露等级 4，无住所：floor(无住所人口 / 40)
暴露等级 4，住宅人口：floor(住宅低温人口 / 60)
连续 2 天等级 3：floor(相关暴露人口 / 60)
第七霜落等级 4：额外 + floor(相关暴露人口 / 50)

```
D1 保护：  
```
D1 常规寒冷死亡 = 0。
若玩家明确 furnace off / 确认供暖不足日，D1 直接寒冷死亡最多 1。

```
   
⸻  
   
## 饥饿、长期生食与过劳  
## 50. 饥饿  
饥饿分阶段：  
```
第 1 天食物不足：新增患病，信任下降，恐慌上升，不大规模死亡。
连续 2 天食物不足：患病增加，少量重症，可能少量死亡。
连续 3 天以上食物不足：死亡风险显著提高，提交饥荒危机候选。

```
推荐数值方向：  
```
饥饿人口每 20 人：
第 1 天：+1 患病
第 2 天：+2 患病，+1 重症
第 3 天起：+2 患病，+1 重症，+1 死亡

```
说明：  
```
饥饿死亡暂时计入自然死亡软上限。
若后续食物系统独立定义，以后续为准。

```
   
⸻  
   
## 51. 长期生食  
长期依赖生食风险：  
```
连续 2 天大量生食：新增患病 + floor(食用人口 / 30)
连续 4 天大量生食：额外新增重症 + floor(食用人口 / 60)

```
说明：  
```
生食可救急，但不是长期稳定食物。
熟食供应不足会影响信任、恐慌和疾病。

```
   
⸻  
   
## 52. 过劳接口  
过劳来源：  
```
加班日
长班制
高强度外勤
连续危险作业

```
推荐数值方向：  
```
单日长班：新增患病 + floor(参与人口 / 40)
连续 2 天长班：新增患病 + floor(参与人口 / 25)
连续 3 天以上：新增重症 + floor(参与人口 / 50)，信任下降，恐慌上升
暴露等级 3 / 4 时，长班额外新增患病 + floor(参与人口 / 40)

```
说明：  
```
过劳不单独开新系统。
过劳通过疾病、重症、信任、恐慌与事件候选体现。

```
   
⸻  
   
## 自然死亡软上限  
## 53. 自然死亡软上限覆盖范围  
自然死亡软上限暂时覆盖：  
```
疾病死亡
寒冷死亡
冻伤相关死亡
饥饿死亡
长期生食导致的疾病链死亡
过劳导致的疾病链死亡

```
自然死亡软上限不覆盖：  
```
炉心崩毁
人口归零后的失败判定
信任归零且未被终火誓约承接
恐慌达到失败条件且未被最高戒令压制
事件明确写死的特殊硬失败

```
说明：  
```
第七霜落终局验收不再作为额外硬失败轴。
第七霜落结束后的低综合分、多系统崩坏、低于城市连续性参考线，进入 008-4 终局状态评级，不由自然死亡软上限直接判负。

```
   
⸻  
   
## 54. 自然死亡软上限数值  
自然死亡软上限：  
```
D1：0，确认极端风险最多 1
D2～D15：3
D16～D35：5
D36～D48，非黑霜天气：7
黑霜天气日：8 + 人口规模修正，最高 14
D49～D55，第七霜落：12 + 人口规模修正，最高 22

```
人口规模修正：  
```
黑霜：floor(max(0, 存活人口 - 80) / 40)
第七霜落：floor(max(0, 存活人口 - 80) / 35)

```
说明：  
```
这不是固定死亡数。
风险不足不会死到上限。
超过上限的死亡压力转化为更多重症、伤残、恐慌上升、信任下降、事件候选。

```
