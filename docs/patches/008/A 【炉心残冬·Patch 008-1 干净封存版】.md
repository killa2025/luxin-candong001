## 2. V1 普通难度定位  
V1 默认难度为普通难度。  
普通难度目标：  
```
AI 玩家盲玩有机会通关。
允许玩家犯小错。
不要求完美数值优化。
不要求背板。
不要求每一天都极限压榨。
不能长期乱玩仍稳定通关。
不能无视炉心、住房、食物、医疗仍轻松活过第七霜落。

```
普通难度不是：  
```
教学无惩罚模式。
极限硬核模式。
纯剧情模式。
纯数值优化模式。

```

## 8. 食物显示口径  
食物分为：  
```
熟食
生食

```
默认消耗：  
```
每名存活人口每天消耗 1 份食物。
优先消耗熟食。
熟食不足时，可消耗生食。
熟食与生食都不足时，进入饥饿系统。

```
食堂加工口径：  
```
1 生食 = 2 熟食

```
显示规则：  
```
总食物为原始库存口径。
熟食、生食、可加工熟食等价应分开显示。
不应只显示一个“总食物”数字。

```
示例：  
```
熟食：160
生食：40
可加工熟食等价：80
原始库存合计：200

```
说明：  
```
原始库存合计用于库存理解。
可加工熟食等价用于食堂可运行时的潜在供餐判断。
若食堂停工或无法运行，生食不能自动视为熟食。

```
长期依赖生食会带来：  
```
信任下降风险
恐慌上升风险
患病风险
疾病链恶化风险

```
具体疾病、信任、恐慌结算由 008-3 负责。  
   
⸻  
   
## 9. 煤炭显示口径  
煤炭采用日计划制。  
显示重点：  
```
今日供暖支付
今日生产入库
日结后库存

```
规则：  
```
今日供暖支付读取 end_day 前已有煤炭。
今日采煤 / 今日生产煤炭在日结后入库，主要保障下一日。
今日采煤不能自动支付今日炉心缺口。

```
具体炉心煤耗、供暖、供暖不足日由 008-3 负责。  
   
⸻  
   
## 10. 温度显示口径  
温度使用：  
```
后台真实整数温度 + 玩家整十级标签

```
示例：  
```
基础温度：-38（-40 级寒潮）
基础温度：-56（-60 级深冬）
基础温度：-76（-80 级第七霜落）

```
规则：  
```
后台真实整数温度参与所有计算。
玩家整十级标签只用于理解。
建筑停工、寒冷暴露、疾病、死亡、终局验收不得读取整十级标签。

```
具体温度公式由 008-3 负责。

## 11. 系统预测不得直接判定失败  
系统可以预测风险，但不能仅凭预测直接判定失败。  
允许提示：  
```
今日供暖可能不足。
食物明显不足。
医疗容量严重不足。
关键建筑可能停工。
信任 / 恐慌接近危机。
第七霜落准备不足。
终局风险极高。

```
禁止：  
```
系统预测撑不过，所以提前失败。
第 48 天判断准备不足，直接失败。
第 49 天前因为终局准备不足，直接失败。
低综合分预测，直接失败。

```
真正系统硬失败只来自已封存硬条件：  
```
存活人口归零
信任归零且未被终火誓约承接
恐慌达到失败条件且未被最高戒令压制
炉心崩毁

```
说明：  
```
第七霜落终局验收不再作为额外硬失败轴。
第七霜落结束后的低综合分、多系统崩坏、低于城市连续性参考线，进入终局状态评级，不直接强制失败。
除 008-4 已封存的硬失败条件外，系统不得新增其他硬失败。

```

## 23. 开局压力结构  
普通难度开局压力来自：  
```
住房不足
医疗体系未建立
食物只能短期支撑
煤炭只能短期支撑
天气会逐步恶化
信任 / 恐慌不是满稳定状态

```
但开局不应立刻进入不可逆崩盘。  
设计目标：  
```
玩家第 1～3 天有明确补救窗口。
玩家需要优先处理住房、医疗、食物和煤炭。
玩家若完全无视其中任一系统，会在第一周后进入明显风险。

```
