第七霜落固定为：  
```

```
规则：  
```
第 49 天进入第七霜落。
第 55 天 end_day 后进入终局结算。
第七霜落不提前。
第七霜落不推迟。
第七霜落不是随机事件。

```
说明：  
```
第七霜落是终局天气阶段，不是普通事件。
第七霜落期间主要验收玩家前 48 天建立的城市系统。
```

## 8. 煤炭 / 炉心系统验收  
预评估终局炉心基础预算：  
```
当前炉心 3 档日耗 × 7

```
无节煤时：  
```
150 × 7 = 1050 煤

```
节煤 I：  
```
135 × 7 = 945 煤

```
节煤 II：  
```
120 × 7 = 840 煤

```
评分参考：  
```
充足：第七霜落期间基本无供暖不足日，炉心稳定运行，D55 后仍有煤炭余量，压力未接近崩毁。
稳定：有少量煤炭压力，但炉心大体稳定，供暖不足日极少，未进入长期红线。
勉强：存在多日供暖不足或高压运行，但未触发炉心崩毁。
不足：频繁供暖不足、炉心关闭或过载红线，城市靠硬扛撑过。
崩坏：炉心系统严重失能，但未触发炉心崩毁硬失败。

```
硬失败边界：  
```
炉心崩毁 = 硬失败。
煤炭 / 炉心评分崩坏 ≠ 额外硬失败。

```
   
⸻  
   
## 9. 食物系统验收  
终局食物预算：  
```
D49 存活人口 × 7

```
可用食物计算：  
```
熟食
+ 可加工生食折熟食
+ D49～D54 稳定预计熟食产出

```
折算规则：  
```
1 生食 = 2 熟食

```
限制：  
```
只有食堂可运行时，生食才可计入可加工熟食等价。
食堂停工时，生食不能自动按熟食结算。
狩猎小屋在暴雪 / 黑霜 / 第七霜落停工时，不得计入稳定产出。
温室低温停产时，不得计入稳定产出。

```
评分参考：  
```
充足：最后七天食物供应充足，基本无饥饿日。
稳定：有压力但未出现连续饥饿。
勉强：出现短期食物不足，但未发展成饥荒。
不足：多日食物不足，明显推动疾病、恐慌或死亡。
崩坏：长期饥饿或饥荒状态，但未因人口归零触发硬失败。

```
硬失败边界：  
```
食物系统崩坏不单独构成硬失败。
若食物崩坏导致人口归零，则由人口归零触发硬失败。

```
   
⸻  
   
## 10. 住房 / 温度系统验收  
住房覆盖率：  
```
住房覆盖率 = 住房容量 / 存活人口

```
上限按 100% 显示。  
温度验收读取：  
```
住宅有效温度
无住所人口暴露
关键建筑有效温度
供暖不足日
炉心关闭日
寒屋日

```
评分参考：  
```
充足：住房覆盖率高，住宅温度稳定，无大规模寒屋或无住所暴露。
稳定：存在局部寒冷压力，但未造成大规模疾病 / 死亡。
勉强：住房或温度多次不足，城市靠医疗和储备硬撑。
不足：大量无住所、寒屋、关键建筑低温，多次造成疾病和死亡。
崩坏：住房 / 温度系统严重失能，但仍有人活过第七霜落。

```
硬失败边界：  
```
住房 / 温度崩坏不单独构成硬失败。
若寒冷最终导致人口归零，则由人口归零触发硬失败。

```
   
⸻  
   
## 11. 医疗 / 疾病系统验收  
医疗覆盖率：  
```
医疗覆盖率 = 医疗容量 / max(1, 患病 + 重症)

```
重症率：  
```
重症率 = 重症 / 存活人口

```
评分参考：  
```
充足：医疗容量足够，重症率低，疾病链受控。
稳定：医疗紧张但未崩溃，重症可控。
勉强：医疗长期紧张，部分病人恶化或死亡。
不足：医疗容量明显不足，重症与死亡持续扩大。
崩坏：医疗系统严重失能，病患大量积压或医院停治，但仍有人活过终局。

```
硬失败边界：  
```
医疗崩坏不单独构成硬失败。
若医疗崩坏导致人口归零，则由人口归零触发硬失败。

```
   
⸻  
   
## 12. 信任 / 恐慌系统验收  
读取：  
```
D55 结算后信任值
D55 结算后恐慌值
信任危机日数量
恐慌危机日数量
是否触发终火誓约
是否触发最高戒令
是否发生旧城派重大后果

```
评分参考：  
```
充足：信任高，恐慌低，社会秩序稳定。
稳定：信任 / 恐慌有压力，但未长期危机。
勉强：多次接近危机，靠路线或承诺维持。
不足：信任低或恐慌高，社会明显破裂。
崩坏：信任或恐慌系统严重失衡，但被路线终章承接或尚未触发硬失败。

```
硬失败边界：  
```
信任 = 0 且未被终火誓约承接：硬失败。
恐慌 = 100 且未被最高戒令压制：硬失败。

```
说明：  
```
终火誓约只承接信任失败轴，不删除信任数值。
最高戒令只压制恐慌失败轴，不删除恐慌数值。
两者不保证好结局。
两者不免疫人口归零或炉心崩毁。

```
   
⸻  
   
## 13. 人口 / 死亡系统验收  
D49 记录基准：  
```
D49 基准存活人口
D49 基准健康人口
D49 基准患病人口
D49 基准重症人口
D49 基准伤残人口
D49 基准可工作人口

```
城市连续性参考线：  
```
城市连续性参考线 = max(40, floor(D49 基准存活人口 × 0.30))

```
说明：  
```
低于城市连续性参考线不触发硬失败。
低于城市连续性参考线只标记城市连续性崩坏。

```
评分参考：  
```
充足：死亡少，劳动力和人口结构保存良好。
稳定：有死亡，但人口结构仍可延续。
勉强：死亡明显，劳动力受损，但城市仍有基本连续性。
不足：大量死亡或伤残，城市未来非常困难。
崩坏：人口结构严重破坏，但仍有人活过第七霜落。

```
硬失败边界：  
```
存活人口 = 0：硬失败。
人口 / 死亡评分崩坏 ≠ 额外硬失败。
低于城市连续性参考线 ≠ 额外硬失败。

```

## 18. 终局状态分档  
根据六大系统总分判断终局状态：  
```
20～24：高质量胜利
15～19：标准胜利
10～14：惨胜
5～9：崩坏幸存
0～4：残火未灭

```
说明：  
```
崩坏幸存不是失败。
残火未灭不是失败。
它们表示城市以极低质量活过第七霜落。

```
   
⸻  
   
## 19. 高质量胜利限制  
即使总分达到高质量胜利区间，若出现以下情况，也不能判为高质量胜利：  
```
任一系统评分为崩坏。
死亡比例 > 20%。
信任 / 恐慌系统评分为崩坏。
煤炭 / 炉心系统评分低于勉强。
人口 / 死亡系统评分低于勉强。

```
限制规则：  
```
任一系统崩坏：最高标准胜利。
死亡比例 > 20%：最高标准胜利。
信任 / 恐慌崩坏：最高标准胜利。
煤炭 / 炉心仅不足或崩坏：最高惨胜。
人口 / 死亡仅不足或崩坏：最高惨胜。
崩坏项 >= 3：最高崩坏幸存。
崩坏项 >= 5：最高残火未灭。

```
说明：  
```
高质量胜利必须同时满足总分与关键限制。
不能靠其他系统高分掩盖核心崩坏。

```
   
⸻  
   
## 20. 标准胜利  
标准胜利条件：  
```
总分 15～19
或总分 20～24 但被高胜限制降级到标准胜利

```
状态说明：  
```
炉城活过第七霜落。
多数关键系统仍能维持。
城市付出代价，但没有彻底失序。

```
   
⸻  
   
## 21. 惨胜  
惨胜条件：  
```
总分 10～14
或高分局因核心系统不足被限制降级到惨胜

```
状态说明：  
```
炉城活下来了。
但死亡、疾病、寒冷、恐慌或资源断裂留下明显代价。
这不是失败，但也不是值得庆祝的胜利。

```
   
⸻  
   
## 22. 崩坏幸存  
崩坏幸存条件：  
```
总分 5～9
且未触发硬失败

```
状态说明：  
```
城市多个系统严重失能。
居民仍有人活着。
炉心仍未崩毁。
执政官仍能看到终局报告。

```
规则：  
```
崩坏幸存不是硬失败。
崩坏幸存进入 009 的存活结局文案池。

```
   
⸻

残火未灭条件：  
```
总分 0～4
且未触发硬失败
且 D55 后仍有存活人口

```
状态说明：  
```
城市几乎已经丧失完整运转能力。
但仍有存活人口。
炉心未崩毁。
系统不判定为硬失败。

```
规则：
