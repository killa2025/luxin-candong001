## 4. 终局流程总览  
终局报告按以下顺序生成：  
```
1. 判断是否触发系统硬失败或玩家主动结束。
2. 若系统硬失败，进入硬失败文案池。
3. 若玩家主动结束，进入主动结束文案池。
4. 若城市活过第七霜落，读取 008-4 终局状态。
5. 根据终局状态选择主文案池。
6. 收集重大标签候选。
7. 过滤普通小事。
8. 合并同类标签。
9. 按优先级选择 0～3 条重大状态附加句。
10. 选择 0～2 条重大城约 / 路线痕迹摘要。
11. 选择 1 条执政官拷问文案。
12. 可选显示隐藏成就名称。
13. 输出终局报告。

```
说明：  
```
玩家看到的是叙事式终局报告。
AI / 代码内部保留结构化标签。
默认终局页不展示大段数值表。
完整数值可放在调试 / 详情模式。

```

## 一、终局结果与硬失败标签  
## 5. 终局结果标签  
终局结果标签来自 008-4 / 009-1。  

| 标签                | 含义       |
| ----------------- | -------- |
| hard_fail         | 触发系统硬失败  |
| player_ended      | 玩家主动结束本局 |
| high_victory      | 高质量胜利    |
| standard_victory  | 标准胜利     |
| bitter_victory    | 惨胜       |
| collapse_survival | 崩坏幸存     |
| ember_survival    | 残火未灭     |
  
规则：  
```
collapse_survival 不是失败。
ember_survival 不是失败。
player_ended 不是系统硬失败。

```
   
⸻  
   
## 6. 硬失败类型  
系统硬失败类型：  

| 标签              | 条件                        |
| --------------- | ------------------------- |
| population_zero | 存活人口归零                    |
| core_collapse   | 炉心崩毁                      |
| trust_exile     | 信任归零，且未被终火誓约承接，执政官被流放     |
| panic_expelled  | 恐慌达到失败条件，且未被最高戒令压制，执政官被驱逐 |
  
说明：  
```
信任 / 恐慌失败是玩家失败，不等于炉城立刻灭亡。
炉城也许会有新的长官，也许就此陨落。
但这一切已经与当前执政官无关。

```
  
这个在“作废旧口径”里，**可以保留**，因为它是在说旧口径作废，不是正式字段。：  
```
trust_fail 作为玩家可见失败名。
panic_fail 作为玩家可见失败名。
信任失败 = 炉城立即毁灭。
恐慌失败 = 炉城立即毁灭。

```
   
⸻  
   
## 7. 玩家主动结束  
玩家主动结束使用：  
```
player_ended

```
规则：  
```
玩家主动结束不是系统硬失败。
玩家主动结束不进入 hard_fail。
玩家主动结束文案来自 009-1 的主动结束文案池。
玩家主动结束可以显示 0～1 条状态附加句。
玩家主动结束可以显示已解锁隐藏成就名称。

```
玩家主动结束含义：  
```
执政官亲手封存本局。
系统不替玩家判负。
风雪不替玩家判负。
城市状态按 player_ended 记录。

```

## 二、普通小事过滤与标签严重度  
## 8. 普通小事过滤规则  
以下情况默认不进入终局审判标签：  
```
死过 1 个人。
某天有人没被治疗。
有人露宿过 1 夜。
某天出现少量患病。
某天建筑短暂停工。
某天使用过 1 次 heat。
某天短暂食物不足。
某天煤炭紧张但没有连续崩坏。
某天信任 / 恐慌小幅波动。
某天触发普通状态提醒。
一次普通承诺失败但未造成长期后果。
一次普通承诺成功但不改变路线状态。

```
这些内容可以出现在：  
```
日结报告
事件反馈
日志
状态面板
详情页

```
但不进入最终执政官审判。  
进入终局标签必须满足至少一项：  
```
长期
反复
大规模
高比例
危机级
路线级
造成不可逆后果
改变终局状态
改写失败轴

```
说明：  
```
终局报告不是日记本。
小事归日志，大事归档案。

```
   
⸻  
   
## 9. 标签严重度  
所有候选标签分为四级。  

| 严重度      | 用途                  |
| -------- | ------------------- |
| minor    | 普通过程，不进入终局报告        |
| notable  | 可作为内部记录，通常不触发附加句    |
| major    | 可触发终局附加句            |
| defining | 可影响拷问文案、路线总结与 009-C |
  
默认进入终局报告候选：  
```
major
defining

```
默认不进入主报告：  
```
minor
notable

```
notable 可作为：  
```
009-C 路线文案素材
详情页记录
调试信息
内部状态判断

```

## 四、基础城约与路线标签  
## 16. 基础城约标签总规则  
基础城约来自 006A。  
009-2 只记录会影响终局审判的重大倾向。  
规则：  
```
签署某条炉律本身不一定进入终局审判。
长期使用、造成后果、改变城市制度方向，才进入标签。
普通医疗、普通食物、普通工时结果不再重复写摘要句。

```
009-B r3 后，重大城约摘要只保留：  
```
儿童线
死亡处理线
娱乐 / 减压线
誓言 / 铁腕路线一句式痕迹
旧城派处理一句式痕迹
隐藏成就名称记录

```
默认不作为 009-2 城约摘要显示：  
```
普通医疗处置线摘要
普通食物配给线摘要
普通工时线摘要

```
这些内容交给：  
```
009-1 生存摘要
六大系统标签
重大状态附加句
009-C 路线专属文案

```

## 八、重大城约 / 路线痕迹摘要  
## 38. r3 核心修正  
009-1 的终局报告已经包含叙事式生存摘要，例如：  
```
带领多少人走入寒冬。
墓园 / 冷藏坑 / 名册上留下多少名字。
第七霜落带走多少人。
仍有多少病患。
信任 / 恐慌如何托举或摔下城市。
炉心轰鸣状态。

```
```
煤炭与食物是否充足。
城市是否仍保留延续的可能。

```
因此 009-2 不应再重复写：  
```
医疗救了多少人。
医疗死了多少人。
食物够不够。
煤炭够不够。
死亡多少人。
病患多少人。

```
009-2 的重大城约 / 路线痕迹摘要只保留：  
```
不会被生存摘要自然覆盖的制度选择痕迹。

```

## 41. 儿童线摘要  
儿童线是重大制度选择，可以进入终局摘要。  
只保留两种主方向。  
**儿童辅工**  
```
你签署了儿童辅工。孩子们用细嫩的手搬运木料、整理仓库，甚至在最冷的日子里跟着成年人走近风雪。炉城因此多撑住了一些时刻，而这些时刻会永远记得他们的年纪。

```
**儿童保护**  
```
你修建了儿童保护所。孩子们在炉火旁学会数数、识字，拿起尺子、绷带和小小的手术刀。炉城没有急着把他们变成工人，而是把未来暂时藏进了更暖的屋子里。

```
说明：  
```
儿童伤害、儿童死亡、儿童高风险不在儿童摘要句里重复。

```
   
⸻  
   
## 42. 死亡处理线摘要  
死亡人数本身由 009-1 生存摘要记录。  
009-2 只记录城市如何安置死者。  
**墓园**  
```
你为死者留下了墓园。那些名字没有只停在数字里，人们仍能在风雪间找到一处地方，承认他们曾经活过。

```
**冷藏坑**  
```
你修建了冷藏坑。死者没有立刻离开炉城，他们被安置在冰冷的秩序里，等待城市决定如何继续面对他们。

```
说明：  
