```

**6.1 已确认机制**

```

```

```

```
状态：  
```

```
**6.3 woodfuel 支付规则**  
当日炉心基础煤耗结算时：  
```
required_coal = 当前炉心档位基础煤耗
available_coal = 当前煤炭库存
coal_gap = max(required_coal - available_coal, 0)

```
若 woodfuel 未确认：  
```
不燃烧木材。
煤炭不足按供暖不足规则处理。

```
若 woodfuel 已确认：  
```
wood_needed = coal_gap × 4
actual_wood_burned = min(wood_needed, woodfuel_daily_wood_limit, 当前木材库存)
woodfuel_equivalent = floor(actual_wood_burned / 4)

```
若：  
```
available_coal + woodfuel_equivalent >= required_coal

```
则：  
```
炉心基础供暖可完整支付。

```
若不足：  
```
仍按炉心基础供暖不足处理。

```
**6.4 woodfuel 平衡定位**  
```
woodfuel 可以救“差一点”的煤炭缺口。
woodfuel 不能救“完全没煤”的城市。
woodfuel 不能替代中期煤炭生产。
woodfuel 不能替代第七霜落前的煤炭储备。
woodfuel 不能支付 heat。
woodfuel 不能支付 overload。
woodfuel 不能计入煤炭承诺成功条件。

```
第七霜落期间：  
```
woodfuel 可以作为极限应急手段。
但不得写成第七霜落期间靠木材稳定翻盘。

```

**9.2 加班日暂定实现值 / 测试窗必测**  
```
overtime_output_multiplier = 2.00

```
解释：  
```

```
**9.3 加班日风险暂定实现值 / 测试窗必测**  
按参与加班人口 N 计算：  
```
overtime_sick_added = floor(N / 15)
overtime_accident_risk_points = ceil(N / 20)

```
若目标建筑处于寒冷暴露等级 3 / 4：  
```
额外新增患病 = floor(N / 30)

```
最低值：  
```
若 N > 0 且 floor(N / 15) = 0，则 overtime_sick_added = 1

```
**9.4 加班日信任 / 恐慌暂定实现值 / 测试窗必测**

```

**11.3 治疗效果暂定实现值 / 测试窗必测**  
使用医疗配给后，优先级为：  
```
1. 轻症
2. 重症

```
结算：  
```
每影响 1 名轻症：
该轻症立即治愈，转为健康。

轻症不足时：
剩余名额用于重症。
每影响 1 名重症：
该重症获得 1 日治疗推进。
不立即治愈。
不直接消除伤残。

```
实际消耗：  
```
actual_affected = min(当前轻症 + 当前重症, medical_ration_max_patients)
actual_food_cost = actual_affected × medical_ration_food_cost_per_patient

若 actual_food_cost > 当前熟食：
actual_affected = floor(当前熟食 / medical_ration_food_cost_per_patient)
actual_food_cost = actual_affected × medical_ration_food_cost_per_patient

```
