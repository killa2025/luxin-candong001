## 一、文件使用规则  
本文件用于整理建筑升级与中期生产链相关数值，供代码窗先接入可运行骨架，并供测试窗进行压力测试与平衡复核。  
本文件不回改任何已封存 Patch 正文。  
若本文件与以下文件冲突，按以下优先级处理：  
```
全局修正总控
>
A 干净封存版
>
无 A 版时对应的干净封存版
>
本批干净数值补表
>
小修、覆盖规则、返工清单、旧补表

```
本文件中数值分为：  
```
已封存 / 可直接实现：
来自全局总控、A 版封存正文或用户后续明确确认口径，当前无冲突，可交代码窗实现。

代码窗暂定实现值 / 测试窗必测：
允许代码窗先写入配置表或常量表，用于跑通游戏骨架。
不等于最终平衡封存。
必须由测试窗试玩、压力测试后确认保留、调整或封存。

后置 / 待确认 / 不进代码窗：

```

## 四、中期闭环目标  
D15～D35 应逐步形成以下闭环：  
```
煤炭：
地表煤堆 → 小型采煤机 → 小型采煤改良 → 炉心节煤 / 储备

木材：
地表木材堆 → 森林区资源点 → 伐木场 → 木材加工 II → 建筑升级 / woodfuel 应急

钢铁：
地表钢铁堆 → 小型采钢机 → 小型采钢改良 → 医院 / 高级科技 / 升级耗材

食物：
开局储备 → 狩猎小屋 + 食堂 → 温室 → 改良温室 / 食堂流程改良

住房：
基础住宅 → 改良住宅 → 高级住宅

医疗：
医疗站 → 医疗工具改良 / 医疗建筑保温 → 医院

```
避免死局原则：  
```
单一科技不能成为唯一活路。
单一资源不能完全卡死所有发展路线。
地表资源耗尽前，玩家应有时间接上至少一条中期生产链。
中期生产链不能太便宜，否则前期压力失效。
中期生产链不能太贵，否则玩家会在 D15～D25 断档。
第七霜落不能靠临时外环生产翻盘，必须靠此前储备。

```

## 六、通用 upgrade 规则  
**6.1 已封存 / 可直接实现**  
```
upgrade 是即时操作。
upgrade 需要前置科技。
upgrade 需要支付木材 / 钢铁等资源。
upgrade 成功后，建筑立即改变状态。
upgrade 不进入 end_day 队列。
upgrade 不需要等待日结完成。
upgrade 不产生“升级中”状态。
upgrade 不需要临时迁出居民。
upgrade 不临时停用建筑。
upgrade 默认不改变占槽，除非升级条目明确写明。

```
若玩家在 end_day 前完成 upgrade：  
```
该建筑从 upgrade 成功后立即使用新状态。
当日后续面板、岗位、容量、保温、生产结算按新状态读取。

```
upgrade 失败条件：  
```
缺少前置科技。
目标建筑不存在。
目标建筑已经是最高级。
资源不足。
建筑类型不支持该 upgrade。
目标建筑处于不可升级状态。

```
失败时：  
```
不扣资源。
不创建升级中状态。
不改变建筑。

```

## 十、森林区与伐木场  
**10.1 命名边界**  
用户已确认并采用：  
```
森林区 = 地图资源点 / 大型木材资源锚点。
森林区对标大型煤矿点 / 大型钢铁矿点。
当前地图开局有 2 座固定森林区。

伐木场 = 科技解锁建筑。
伐木场绑定森林区。
伐木场数量受森林区数量限制。

```
不得写成：  
```
删除伐木场建筑。
森林区替代伐木场建筑。
森林区本体有建筑成本。
```
