# 《炉心残冬》第三批干净数值补表：006B 工程科技树成本 / 研究天数  
版本定位：第三批干净数值补表 适用对象：代码窗、测试窗 处理方式：查错式合并、去重、删除旧口径、修正科技 Tier 回流、保留可跑骨架的暂定测试值 不修改对象：全局修正总控、A 干净封存版、无 A 版时对应的干净封存版  
   
⸻  
   
## 一、文件使用规则  
本文件用于整理 006B 工程科技树的研究成本与研究天数，供代码窗先接入可运行骨架，并供测试窗进行压力测试与平衡复核。  
本文件不回改任何已封存 Patch 正文。  
若本文件与以下文件冲突，按以下优先级处理：  
```
全局修正总控
>
A 干净封存版
>
无 A 版时对应的干净封存版
>
本批干净数值补表
>
小修、覆盖规则、返工清单、旧补表

```
本文件中所有科技成本与研究天数，统一状态为：  
```
代码窗暂定实现值 / 测试窗必测

```
含义：  
```
代码窗可以先按本表写入配置表或常量表，用于跑通 research 骨架。
这些数值不是最终平衡封存。
测试窗必须测试研究节奏、资源压力、前置链是否过强或过弱。
测试后可保留、调整或封存。
代码窗不得把这些数值标记为“最终封存平衡值”。

```
   
⸻

## 五、研究节奏规则  
**5.1 基础研究进度**  
代码窗可实现规则：  
```
每个科技使用 research_days 表示基础研究天数。
每天 end_day 末尾推进 1 天研究进度。
当累计研究进度 >= research_days 时，科技完成。
科技完成在该日 end_day 末尾提交。

```
示例：  
```
某科技 research_days = 3。
D5 白天开始研究。
D5 end_day 推进 1。
D6 end_day 推进 2。
D7 end_day 推进 3，并在 D7 end_day 末尾完成。

```
   
⸻  
   
**5.2 第二研究所加速**  
已封存方向：  
```
第二研究所可以加速研究。
第二研究所不开放第二研究队列。

```
代码窗暂定实现值 / 测试窗必测：  
```
second_research_center_speed_multiplier = 1.50

```
规则：  
```
若存在第 2 座可运行研究所：
当前唯一研究队列研究速度 ×1.50。
不产生第二研究队列。
不允许同时研究两项科技。

```
最低天数限制：  
```
若科技基础 research_days = 1，则仍为 1 天。
若科技基础 research_days >= 2，加速后最短不得低于 2 天。

```
推荐计算：  
```
若 base_research_days = 1：
effective_research_days = 1

若 base_research_days >= 2：
effective_research_days = max(2, ceil(base_research_days / 1.50))

```
示例：  

| 基础研究天数 | 第二研究所后 |
| ------ | ------ |
| 1      | 1      |
| 2      | 2      |
| 3      | 2      |
| 4      | 3      |
| 5      | 4      |
| 6      | 4      |
  
状态：  
```
代码窗暂定实现值 / 测试窗必测

```
理由：  
```
第二研究所需要额外建筑、槽位、人手与资源。
如果加速太弱，玩家没有建第二研究所的动力。
如果开第二队列，科技膨胀过快。
因此 V1 先用单队列加速，不开放并行研究。

```

## 十七、本批结论  
本批可交代码窗实现的确定规则：  
```
研究所开局可建。
研究所不需要科技 / 炉律解锁。
research 命令需要至少 1 座已建成研究所。
研究推进至少需要 1 名工程师。
工程学徒不能替代工程师满足最低要求。
默认同一时间只有 1 个研究队列。
第二研究所只加速，不开放第二研究队列。
科技研究开始时支付木材 / 钢铁。
取消研究不退款。
科技完成后只解锁资格或效果，不自动建造、不自动升级。
升级成本缺失时，upgrade 不可执行，显示 PENDING_UPGRADE_COST。

```
本批允许代码窗先写入骨架、但必须交测试窗复核的暂定实现值：  
```
全部 006B 科技木材成本。
全部 006B 科技钢铁成本。
全部 006B 科技研究天数。
second_research_center_speed_multiplier = 1.50。

```
第三批最重要修正：  
```
006B-2 科技 Tier 已恢复 A 版口径。
煤层支架 / 钢材筛选已改为 木20 / 钢5 / 2天。
所有成本 / 研究天数均标为“代码窗暂定实现值 / 测试窗必测”，不写成最终封存。

```
下一批建议写：  
```
《炉心残冬》第四批干净数值补表：建筑升级与中期生产链

```
第四批应重点处理：  
```

小型采煤机 / 小型采钢机接线。
```
