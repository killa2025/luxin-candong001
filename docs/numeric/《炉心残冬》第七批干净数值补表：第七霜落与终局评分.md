## 九、第七霜落煤炭 / 炉心压力  
**9.1 炉心基础预算**  
已封存参考：  
```
无节煤：炉心 3 档 150 × 7 = 1050 煤
节煤 I：炉心 3 档 135 × 7 = 945 煤
节煤 I + II：炉心 3 档 120 × 7 = 840 煤

```
说明：  
```
这是预评估终局炉心基础预算。
实际消耗还会受到炉心档位、overload、heat、woodfuel 等影响。

```
**9.2 煤炭压力规则**  
代码窗可实现：  
```
第七霜落期间炉心煤耗仍按炉心档位读取。
第七霜落不额外增加炉心基础煤耗倍率。
第七霜落不免费降低煤耗。
heat 成本仍按已封存 heat 规则读取。
overload 额外煤耗仍按已封存 overload 规则读取。
woodfuel 仍可作为应急燃料，但不能替代煤炭储备评分。

```
原因：  
```
008-3 已经通过极端真实温度制造压力。
若再额外加煤耗倍率，容易把终局变成单纯煤炭硬门槛。

```
**9.3 煤炭 / 炉心每日标签阈值**  
代码窗暂定实现值 / 测试窗必测：  
```
frost_coal_shortage_day:
```

## 十、第七霜落食物压力  
已封存 / 可直接实现：  
```
第七霜落期间每人每日仍默认消耗 1 份食物。
熟食优先。
熟食不足可吃生食。
熟食与生食都不足进入饥饿系统。
1 生食 = 2 熟食。

```
食物预算：  
```
终局食物预算 = D49 存活人口 × 7

```
可用食物计算：  
```
熟食
+ 可加工生食折熟食
+ D49～D54 稳定预计熟食产出

```
限制：  
```
只有食堂可运行时，生食才可计入可加工熟食等价。
食堂停工时，生食不能自动按熟食结算。
狩猎小屋在第七霜落停工，不得计入稳定产出。
温室低温停产时，不得计入稳定产出。

```
代码窗暂定实现值 / 测试窗必测：  
```
food_shortage_day:
当日无法按当前配给满足全部人口食物消耗。

starvation_day:
当日熟食 + 生食均不足，出现饥饿人口。

famine_pressure_day:
连续 2 天出现 food_shortage_day，或任意 1 天出现 starvation_day。

```
终局标签接口：  
```
famine_survivor：
第七霜落期间出现 starvation_day，但城市存活。

famine_city：
第七霜落期间 starvation_day >= 3，或存在饥饿死亡。

```

## 十三、第七霜落死亡压力与软上限  
008-3 已封存方向：  
```
第七霜落期间存在自然死亡软上限。
第七霜落终局验收不再作为额外硬失败轴。
低综合分、多系统崩坏、低于城市连续性参考线进入终局状态评级。

```
代码窗可实现：  
```
frost_daily_natural_death_soft_cap = min(
  22,
  12 + floor(max(0, alive_population - 80) / 35)
)

```
边界：  
```
自然死亡软上限不是硬保护。
极端危机可以突破软上限。
软上限不阻止人口归零硬失败。
软上限不让医疗 / 饥饿 / 寒冷失效。

```
可突破软上限的极端情况：  
```
炉心关闭。
第七霜落期间关键建筑大面积停工。
连续饥饿。
医疗完全崩溃。
大规模无住所暴露。
炉心崩毁。
人口已接近归零。

```
死亡状态标签：  
```
mass_death_day:
deaths_today >= max(5, floor(alive_population_start_of_day * 0.08))

grave_city_tag_threshold:
total_deaths >= max(20, floor(total_population_ever * 0.25))

city_continuity_broken:
D55 alive_population < max(40, floor(D49_alive_population * 0.30))

```

```
mass_death：

grave_city：

city_continuity_broken：
D55 存活人口低于城市连续性参考线，但未归零。

```
