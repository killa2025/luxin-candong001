# 《炉心残冬》第五批干净数值补表：007 事件 / 承诺 / 固定增员  
版本定位：第五批干净数值补表 适用对象：代码窗、测试窗 处理方式：查错式合并、去重、删除旧口径、修正事件与失败条件边界 不修改对象：全局修正总控、A 干净封存版、无 A 版时对应的干净封存版  
   
⸻  
   
## 一、文件使用规则  
本文件用于整理 007 事件系统、承诺任务与固定增员相关数值，供代码窗先接入可运行骨架，并供测试窗进行压力测试与平衡复核。  
本文件不回改任何已封存 Patch 正文。  
若本文件与以下文件冲突，按以下优先级处理：  
```
全局修正总控
>
A 干净封存版
>
无 A 版时对应的干净封存版
>
本批干净数值补表
>
小修、覆盖规则、返工清单、旧补表

```
本文件中数值分为：  
```
已封存 / 可直接实现：
来自全局总控、A 版封存正文或用户后续明确确认口径，当前无冲突，可交代码窗实现。

代码窗暂定实现值 / 测试窗必测：
允许代码窗先写入配置表或常量表，用于跑通游戏骨架。
不等于最终平衡封存。
必须由测试窗试玩、压力测试后确认保留、调整或封存。

后置 / 待确认 / 不进代码窗：
当前不写入代码实现，不由代码窗自行猜测。

```

## 三、本批已修正的问题  
**3.1 事件不得变成隐藏硬失败**  
本批统一删除或压制以下风险：  
```
事件触发 = 失败。
承诺失败 = 失败。
固定增员拒绝 = 失败。
危机倒计时 = 独立硬失败轴。
第七霜落预警 = 第七霜落硬失败。
低综合分 = 失败。
多系统崩坏 = 独立失败。

```
统一规则：  
```
事件只提示风险、提供选择、生成承诺、结算一次性后果、推动事件链。
事件不能绕过系统硬失败条件直接判负。

```
硬失败仍只来自：  
```
存活人口 = 0。
炉心崩毁。
信任 = 0 且未被终火誓约承接。
恐慌 = 100 且未被最高戒令压制。

```
   
⸻  
   
**3.2 承诺失败不得直接杀人 / 爆炉 / 出走 / 判负**  
本批保留承诺失败惩罚，但明确：  
```
承诺失败不得直接造成人口死亡。
承诺失败不得直接触发炉心崩毁。
承诺失败不得直接触发旧城派最终出走。
承诺失败不得直接提前第七霜落。
承诺失败不得直接判负。

```
承诺失败可以：  
```
降低信任。
提高恐慌。
推动事件链升级。
提高后续危机事件出现概率。
触发 status 警告。

```
但最终仍由对应系统结算：  
```
死亡由疾病 / 饥饿 / 寒冷 / 炉心 / 医疗系统结算。
炉心崩毁由炉心压力系统结算。
旧城派出走由旧城派系统结算。
信任 / 恐慌失败由 008 / 009 硬失败口径结算。

```
   
⸻  
   
**3.3 固定增员人数可给代码窗跑骨架，但不是最终封死**  
007 A 版正文只封存固定增员日期和规则，不封存具体人数。 本批为代码骨架补出可运行测试值。  
状态统一为：  
```
代码窗暂定实现值 / 测试窗必测

```
说明：  
```
代码窗可以按本表先实现第 6 / 19 / 37 天固定增员。
测试窗必须重点测试人口、住房、食物、医疗、儿童、旧城派压力。
测试后可调整人数、结构和接纳奖惩。
不得把本批固定增员人数标成最终平衡封存。

```
   
⸻  
   
## 四、已封存事件系统规则  
**4.1 事件类型**  
007 事件系统包含：  
```
重大事件
普通事件
固定日期事件
状态触发事件
承诺到期结算
非阻塞 status 提示
隐藏成就提示

```
规则：  
```
承诺到期结算独立于事件队列。
隐藏成就提示不进入事件队列。
非阻塞 status 提示不占事件名额。
固定日期事件优先级高于普通民生事件。

```
   
⸻  
   
**4.2 每日事件执行顺序**  
代码窗可实现：  
```
新一天开始
→ 先结算昨日到期承诺成功 / 失败
→ 移除已完成或失败的承诺任务
→ 检查固定日期事件
→ 检查重大事件
→ 检查普通事件
→ 显示 status 提示
→ 玩家操作
→ end_day

```
说明：  
```
承诺到期结算不占事件名额。
承诺结算先于当日新事件检查。
固定增员事件不会被普通事件覆盖。

```
   
⸻  
   
**4.3 同日事件数量限制**  
代码窗可实现：  
```
若当天有重大事件：
最多显示 1 个重大事件 + 1 个普通事件。

若当天没有重大事件：
最多显示 2 个普通事件。

固定日期事件若为重大事件：
必须处理后才能 end_day。

```
被压下事件：  
```
不偷偷结算。
不偷偷执行选项。
不占事件名额。
可转为 status 提示。
后续仍满足条件时可再进入事件候选。

```
   
⸻  
   
**4.4 事件不能替玩家执行命令**  
事件不得自动执行：  
```
build
upgrade
research
heat
overload
woodfuel
ration
worktime
triage
medical_ration
sign_law

```
事件可以提示：  
```
你可以考虑建造食堂。
你可以考虑切换配给。
你可以考虑关闭过载。
你可以考虑启用 woodfuel。
你可以考虑使用 heat。

```
但不能替玩家操作。  
   
⸻

## 六、基础民生事件阈值  
**6.1 空锅请愿**  
事件类型：  
```
食物短缺普通事件

```
代码窗暂定实现值 / 测试窗必测：  
```
food_warning_days_threshold = 2.0
empty_pot_event_days_threshold = 1.5
empty_pot_status_delay_days = 1
empty_pot_cooldown_days = 4

```
计算：  
```
可食天数 = (熟食 + 生食) / 存活人口

```
触发：  
```
若可食天数 < 2.0：
显示 status 食物预警。

若可食天数 < 1.5，且该状态持续至少 1 天：
触发“空锅请愿”。

```
选项最低数值：  
```
A：承诺补足口粮
立即效果：信任 0，恐慌 0
生成 food 承诺。

B：维持当前配给
trust_change = -1
panic_change = +1

C：提示调整配给 / 当日临时压低分配
trust_change = -2
panic_change = +1
不永久切换 ration。

```
边界：  
```
不凭空生成食物。
不绕过 ration 命令。
不免除当日饥饿日结。
不永久修改配给模式。

```
   
⸻

**6.3 病床告急**  
事件类型：  
```
医疗压力普通事件。
医疗系统完全崩溃时可升级为重大事件。

```
代码窗暂定实现值 / 测试窗必测：  
```
medical_gap_warning_threshold = 1
medical_gap_event_threshold = 5
medical_gap_major_threshold = 10
medical_gap_cooldown_days = 4

```
计算：  
```
医疗缺口 = max(当前患病 + 当前重症 - 当前有效医疗容量, 0)

```
触发：  
```
医疗缺口 >= 1：
显示 status 医疗预警。

医疗缺口 >= 5：
触发“病床告急”。

医疗容量 = 0 且 当前患病 + 当前重症 >= 5：
可作为重大事件候选。

医疗缺口 >= 10：
可升级为重大事件候选。

```
选项最低数值：  
```
A：承诺扩容医疗
立即效果：信任 0，恐慌 0
生成 medical 承诺。

B：临时腾挪床位
前置：已解锁相关医疗炉律或存在可运行医疗建筑
trust_change = +1
panic_change = -1
当日未收治风险下降 20%
不新增医疗容量。

C：维持现状
trust_change = -2
panic_change = +2
医疗恶化仍按日结系统处理。

```
   
⸻

## 九、承诺任务总规则  
**9.1 已封存规则**  
```
承诺任务不是持续增益。
承诺任务不会每天提高信任。
承诺任务不会每天降低恐慌。
承诺任务不会暂停原本的日结惩罚。
承诺任务不会自动生成资源、住房、医疗容量或煤炭。
承诺任务不会自动修复旧城派、医疗、食物、住房或炉心压力问题。

```
承诺上限：  
```
同一时间最多 2 个 active 承诺。
同类承诺同一时间最多 1 个。
承诺截止日当天仍可完成。
承诺完成后立即释放承诺槽。

```
   
⸻  
   
**9.2 承诺期限分档**  
代码窗暂定实现值 / 测试窗必测：  
```
轻度承诺：3 天
中度承诺：4 天
重度承诺：5 天
终局前承诺：不得超过第 48 天

```
说明：  
```
第 42 天后，普通承诺截止日不得超过第 48 天。
若承诺期限会超过第 48 天，则自动缩短到第 48 天。
若无法生成有效截止日，则承诺选项不可用。
第 49 天进入第七霜落后，不再开启新的普通承诺任务。

```
   
⸻  
   
**9.3 承诺奖惩分档**  
代码窗暂定实现值 / 测试窗必测：  
```
ordinary_promise_success_trust = +2
ordinary_promise_success_panic = -1

ordinary_promise_failure_trust = -6
ordinary_promise_failure_panic = +6

serious_promise_success_trust = +3
serious_promise_success_panic = -2

serious_promise_failure_trust = -8
serious_promise_failure_panic = +8

critical_promise_success_trust = +4
critical_promise_success_panic = -3

critical_promise_failure_trust = -10
critical_promise_failure_panic = +10

```
适用：  
```
ordinary：
食物、医疗、住房、煤炭、遗体、儿童等普通危机承诺。

serious：
炉心压力、信任、恐慌、重症积压、严重住房暴露等高压承诺。

critical：
第 42 天后终局前短期承诺、炉心压力 100 附近承诺、旧城派临界承诺。

```
护栏：  
```
承诺失败不得直接造成人口死亡。
承诺失败不得直接触发炉心崩毁。
承诺失败不得直接触发旧城派最终出走。
承诺失败不得直接提前第七霜落。

```
