# Patch 005：006A 炉律、配给、工时、医疗与死亡处理实现记录

## 施工依据与编号边界

本轮按 `docs/handoff/代码窗的.md` 的施工矩阵，将“代码 Patch 005”解释为 006A 炉律、配给、工时与医疗行动。它与 `docs/patches/001-005/` 中机制设计文档自身使用的 Patch 005 编号不是同一套编号；本轮没有实现天气 Patch，也没有开始代码 Patch 006。

主要读取范围为 006A 总收口与 006A-1～6、全局总控、全局修正登记、全局解锁索引、第一及第二批数值补表，以及既有 Patch 001～004 交付记录。冲突时按总控、覆盖登记和用户确认口径执行。

## 已完成范围

- 新增严格加载的 `data/laws.json` 与不可变 `LawRules` 配置，登记 006A 炉律前置、互斥、解锁、普通炉律 3 天冷却和已允许测试的数值；
- 新增 `game.sign_law`，签署只解锁建筑、行动或模式，不代替玩家执行后续建造和行动；火盆集会按覆盖口径在签署后自动启用，不占槽且不进入 `build`；
- 新增 `game.set_ration`，支持普通饭、粗粮汤、白米稀粥及当日应急口粮；应急口粮需要确认，日结后恢复此前模式及连续天数；
- 新增 `game.set_worktime` 与 `game.overtime`；长班制持续到手动关闭，加班日指定单座合法且当天预计可运行的建筑、每日最多一次、需要确认并在日结后关闭；加班启动后禁止通过部分或全部撤岗绕过结算；
- 建筑与食物结算使用整数分子、分母及既有余数累计接口应用配给和产出倍率，不引入浮点数；
- 配给的当日实际执行依赖食堂运行状态；命令合法性与风险预览共用当天固定天气、预计炉心实际档位、区域温度、保温、`heat`、岗位及停工阈值的预计运行判断；食堂不存在、无人或低温停工时，粗粮汤 / 白米稀粥降级为普通饭，应急口粮在命令阶段拒绝且不消耗冷却；食物短缺同时保存缺粮单位与未进食人口；
- 配给及长班连续天数只按实际完成的日结模式更新，同日切换模式不能提前清零或绕过连续惩罚；
- 新增 `game.medical_ration`，仅消耗熟食，按实际可处理病患结算轻症治愈和重症治疗进度；
- 新增 `game.triage` 的结构化命令、确认、目标建筑与运行状态校验。由于结果和冷却未封存，合法目标仍返回 `triage_balance_not_sealed` 且事务不改变状态；
- 新增墓园 / 冻土坑互斥路线、未处理遗体汇总、墓园 / 冻土坑已建成后的遗体处理及 `game.memorial`；死亡人口不会因处理遗体而被取消；
- 将医疗容量、配给后果、遗体压力、信任恐慌和当日行动收口接入固定 `end_day` 阶段；
- 存档数据版本升级到 5，加入社会政策与医疗状态，提供 v4 → v5 迁移、严格解码和完整配置级状态校验；迁移会按当前天数、病患及可运行医疗站 / 医院重新推导医疗容量与压力；篡改的未解锁、非法类型、无人或倍率不一致加班状态会在解码或配置级校验阶段被拒绝；
- 炉律配置拒绝 `required_all` 前置闭环，并要求互斥关系双向一致；
- 建筑炉律前置只认已签署炉律，`active_law_ids` 不能绕过签署资格；
- `observe()` 提供已签署、可签署、锁定项、运行中模式、医疗容量、死亡处理和 deferred unlock 等机器可读状态。
