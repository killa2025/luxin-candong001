# PATCH-004：建筑、建造、升级、供热与应急燃料实现记录

## 施工口径

本轮按 `代码窗的.md` 的 Patch 004 施工矩阵，读取第二批、第四批干净数值补表、A 008-2、A 008-3、全局总控、总收口、解锁索引及第 2 猎区 / woodfuel 完整覆盖版。旧文件与后续总收口冲突时，以总收口中的用户覆盖口径执行。

用户明确确认口径：开局已有 4 座基础住宅全部位于内环，每座为独立正式建筑，各占 1 个内环普通槽位，开局共占 4 / 18 槽。每座分配独立 `building_id`，属于开局预建建筑，不重复支付建造成本或等待建造时间，可在满足条件后逐座升级。该确认不限制后续住宅只能建于内环，也不新增其他住宅机制。

用户本轮明确确认口径：部分岗位产出使用规则中的精确整数分子 / 分母，并在存档中保存整数余数；每日按“累计分子整除分母、保存模数”结算，不使用浮点数，也不从展示小数反推比例。基础 `heat` 全城每日共 2 次，同一建筑每日最多 1 次，每日切换时重置；该次数是用户确认值，不标记为从示例文案推导出的既有封存规则。

用户本轮明确确认口径：`heat` 判断读取本日天气和当前燃料可完整维持的实际炉心档位，判断时不预扣 20 煤；执行前必须保留该炉心档位所需燃料，并确认仍有额外 20 煤。已确认的 `woodfuel` 只按正式换算参与炉心档位计算，`heat` 不得挪用炉心燃料或导致实际档位下降。

审核报告提出“学堂能否 `heat`”需确认；经复核，第二批干净数值补表已明确写明“学堂可 heat”，因此按既有封存来源执行，不列为新增用户确认。

## 已完成范围

- 新增机器可读建筑目录、12 个地表资源点、四区域槽位、资源点绑定、建造成本、岗位、温度阈值、产出与上限配置；
- 将开局 4 座基础住宅实体化为 `residence-start-001`～`004`，内环槽位从 4 / 18 开始；
- 新增结构化命令 `game.build`、`game.upgrade`、`game.assign`、`game.unassign`、`game.assign_resource`、`game.unassign_resource`、`game.heat`、`game.woodfuel`；
- `build` 检查建筑是否允许手动建造、区域、槽位、资源、上限、炉律 / 科技前置和资源锚点绑定；
- 第一座狩猎小屋成功建成后自动开放第 2 猎区，只把狩猎小屋上限提高到 2，不自动建第二座建筑或生成食物；
- 住宅与温室升级逐座、即时完成，不创建升级队列、升级中状态、停用期或居民迁出；
- 岗位分配采用“设置目标人数”语义，检查人口类型、岗位资格、岗位容量和全城可用人口；工程师可下放普通劳动与地表采集岗位，但普通工人不能替代研究等专业岗位；
- 地表 4 个煤堆、5 个木材堆、3 个钢铁堆分别保存资源类型、剩余储量、岗位容量、岗位分配、整数余数与耗尽状态；日结产出并扣减储量，耗尽后自动清空岗位且不可再次分配。避寒采集棚只登记风险保护绑定，不增加产量；
- `heat` 开局可用，单次即时扣除 20 煤，全城每日最多 2 次且同一建筑当日最多一次，默认 +18；命令会拒绝已达到运行阈值的建筑，并为当前实际炉心档位保留燃料；应急加热装置接口为 +24，日结后关闭；
- `woodfuel` 必须明确确认，只补炉心基础供暖缺口，不进入煤炭库存、不支付 `heat` 或 `overload`，日结后关闭；当前测试值为 4 木 = 1 煤等效、每日最多 80 木、最小单位 4 木、无冷却；
- 建筑有效温度按区域温度、保温接口与 `heat` 计算；已封存阈值使用严格“小于阈值才停工”；
- 同日建成并分配岗位的生产建筑可在当日结算，顺序为地表采集与生产建筑产出、食堂加工、居民消耗；当日产煤不能反向支付当天炉心；
- 部分岗位产出使用精确整数分子 / 分母与存档余数累计，建筑和地表资源点均不使用浮点数；
- 命令执行前与日结早期执行配置感知校验，核对目录派生字段、岗位总量、区域容量、猎区 / 森林区、资源绑定和地表点状态，损坏状态不会进入正式结算；
- 存档版本升级为 v4，提供 v1 → v2 → v3 → v4 确定性迁移；v2 非连续或自定义住宅 ID 会保留，缺失住宅使用未占用的规范 ID 补齐。

## 手动建造与自动生效边界

- 基础住宅、食堂、狩猎小屋、研究所、小仓库、避寒采集棚等明确为手动 `build`；
- 医疗站、儿童保护所、学堂、墓园、冷藏坑、小酒馆、大赌场等仍需对应炉律先解锁，再由玩家手动 `build`；
- 守炉堂、巡查所属于后续 006C 路线入口签署后自动生效建筑，不占槽、不进入 `build` 目录；
- 火盆集会签署后启用、不占槽、不 `build`，由 Patch 005 炉律系统接线；
- 科技只提供建造或升级资格，不自动建造、不自动升级。Patch 004 只消费已研究科技 ID，不实现研究流程。
