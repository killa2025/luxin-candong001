# PATCH-003：人口、资源、食物、住房与炉心基础实现记录

## 施工口径

仓库没有独立的代码 Patch 003 任务单。本轮范围按 `代码窗的.md` 的施工矩阵确定，并以全局控制文件、A 干净封存版和数值补表为权威输入。除矩阵指定的 001、002-A/B/C、003 外，为核对开局状态、固定天气和炉心温度接口，额外只读核对了 Patch 005、Patch 008-1、Patch 008-2、Patch 008-3；没有实现这些后续 Patch 的机制。

## 已完成范围

- 建立人口、住房、饥饿和当日生存结算的机器可读状态；
- 建立 Patch 003 封存开局：80 人、4 座基础住所、40 个住房容量、封存资源、信任与恐慌初值；
- 建立共享仓储占用量和超容量状态接口，不猜测超容量处理规则；
- 建立 55 天固定真实温度表、内中外环温度修正和日结温度结果；
- 建立炉心 0～3 档结构化命令 `game.set_furnace`；
- 日结时先按日初煤量支付取暖，燃料不足时回落到可完整支付的最高档，不允许当天产出的煤反向支付当天取暖；
- 食物按熟食优先、原料食物随后消耗，记录未进食人口；
- 每日日结重新按存活人口和总住房容量派生有房、无房人口；
- 存档版本升级为 v2，提供 v1 到 v2 的确定性迁移和新增状态的不变量校验；
- 保持随机种子、随机状态和回放接口不变。本 Patch 没有新增随机抽取。

## 风险确认口径

- 炉心关闭、燃料不足和食物不足是 B 级强风险，需要走 Patch 002 的日结确认流程；
- 无房人口和仓储超容量是 A 级信息提示；
- `game.set_furnace` 允许明确选择 0 档。关闭炉心不是系统替用户决策，真正日结前仍会给出 B 级确认。
