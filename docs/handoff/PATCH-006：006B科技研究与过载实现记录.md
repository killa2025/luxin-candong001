# Patch 006：006B 科技研究与过载实现记录

## 完成范围

- 新增 `data/technologies.json` 与严格配置加载器，登记 37 项 006B 科技、T0～T5 层级、前置、成本、研究天数、效果状态、单研究队列、第二研究所倍率及过载压力规则；科技成本、研究天数和 ×1.50 倍率按总收口许可声明为 `TEST_NUMERIC`。
- 新增 `game.research`、`game.cancel_research` 与 `game.set_overload` 结构化命令。研究开始时一次支付木材和钢铁；取消不退款；研究所必须已建成且至少分配 1 名正式工程师，工程学徒不能替代最低要求。
- 研究只在 `ADVANCE_AND_COMMIT_RESEARCH` 日结阶段推进和完成；全城始终只有一个队列，第二座符合条件的研究所只把基础推进速度精确乘以 1.50，不创建第二队列。研究所加班继续使用 Patch 005 的医疗 / 研究 ×1.50 接口。
- 接通已明确效果：炉心节煤 I / II、建筑与住房保温、应急加热强化、生产链产量、食堂加工、医疗站容量、仓储扩容、科技建筑与逐座升级解锁。
- 接通过载 1 / 2 的科技解锁、额外煤耗、有效温度、压力增长、稳定科技减压、关闭过载降压、炉心关闭自动关闭过载、压力 80 强警告、压力 100 红线及红线后继续过载导致炉心崩毁。
- 接通第 49～55 日终局炉心稳定的运行中温度 +3 与过载压力增长额外 -5；不实现终局评分、结局或 Patch 008 / 009 的其他系统。
- 存档版本升级为 v7，新增精确研究进度单位、研究总量、过载档位及红线状态；v6 → v7 只迁移无活动研究的历史存档，无法可靠换算的旧活动研究会被拒绝。

## 关键口径

- 研究完成不自动建造建筑、不自动升级建筑、不自动开启过载。
- `woodfuel` 只补炉心基础供暖，不能支付 `heat` 或过载额外煤耗。
- 炉心关闭时过载自动关闭；过载燃料不足时只失去当日过载效果，不允许木材代付。
- 仓储扩容完成时，既有小仓库每座从 +300 容量同步提升为 +600；后续新建小仓库直接使用 +600，配置级存档校验使用同一口径。
- 运行配置和状态校验拒绝未知科技、重复科技、缺失层级解锁、缺失前置、非法研究总量、未解锁过载及伪造红线状态；命令和日结失败均保持原状态且不生成自动存档。
