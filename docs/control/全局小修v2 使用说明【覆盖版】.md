全局小修**v2 使用说明【覆盖版】**  
  
状态：覆盖原《全局小修v2 使用说明》  
用途：作为 006B / 006C 后续合并与回灌的专项覆盖依据  
优先级：若其他“合并全局小修收敛版”或《v1 全局修正登记》与本文件存在冲突，以本文件为准。  
  
⸻  
  
一、文件定位  
  
本文件只用于记录已确认的全局覆盖口径。  
  
本文件不重写 Patch 正文。  
本文件不新增机制。  
本文件不补数值。  
本文件不替代码窗猜测未封存内容。  
  
⸻  
  
二、处理原则  
  
1. 本文件写明的新口径覆盖旧 Patch 正文中的旧口径。  
2. 本文件没有提到的内容，仍按原 Patch 正文处理。  
3. 未封存 / 待确认 / 待复核 / 已审核但未封存数值不得进入代码窗。  
4. 只有已封存数值可以进入代码窗。  
5. 建筑与升级默认拍下后立即生效。  
6. 不新增施工期。  
7. 不新增修缮期。  
8. 不新增升级中等待。  
9. 不新增升级期间停用机制。  
10. 不新增升级期间居民迁出机制。  
11. 守炉堂 / 巡查所不占槽、自动生效，但必须显示在面板上。  
12. 守炉堂 / 巡查所至少 1 名可工作人口值守即可运行。  
13. 006B 面板与 006C 面板必须区分：  
    * 解锁资格  
    * 已建成或已生效  
    * 可执行行动  
    * 冷却状态  
    * 占槽状态  
    * 是否可 heat  
    * 数值是否待确认  
14. 正式代码区域名统一为：  
    * 内环  
    * 中环  
    * 外环  
    * 外缘储放区  
15. “边缘区”只可作为外环显示别名，不作为新增代码区域字段。  
16. 本登记编号可能沿用历史跳号。缺号不代表文件缺页。  
  
⸻  
  
全局小修**2：006B / 006C 专项小修**  
  
**1. 建筑与升级立即生效**  
  
V1 中，建筑与升级按已封口径：  
  
拍下后立即可运行 / 立即生效。  
  
**不得新增**  
  
施工期  
修缮期  
升级中等待  
临时施工状态  
升级期间停用机制  
升级期间居民迁出机制  
  
**科技只提供资格**  
  
unlock_building：只解锁建造资格。  
unlock_upgrade：只解锁升级资格。  
passive：研究完成后自动参与结算。  
  
**玩家仍需主动执行**  
  
build <建筑>  
upgrade <建筑编号> <升级目标>  
  
**升级耗材**  
  
若升级耗材未封存，统一写：  
  
升级耗材：待确认  
  
代码窗不得自行猜测升级耗材。  
  
**旧残留作废**  
  
旧登记或旧小修中若出现以下字段，视为旧残留，不进入代码窗：  
  
升级耗时  
是否需要停用建筑  
是否需要临时迁出居民  
升级期间停用建筑  
升级期间居民迁出  
升级中等待  
升级修缮期  
  
**可保留的升级字段**  
  
upgrade 表可以保留：  
  
升级前建筑  
升级后建筑  
前置科技 / 前置炉律  
木材成本  
钢铁成本  
是否改变占槽  
是否改变容量  
是否改变保温 / 产出 / 效率  
升级后效果  
  
如果这些内容未封存，必须标记为待确认。  
  
⸻  
  
**2. 006C 核心建筑口径**  
  
006C 誓言 / 铁腕路线核心建筑采用特殊建筑口径。  
  
核心建筑：  
  
守炉堂  
巡查所  
  
**规则**  
  
守炉堂 / 巡查所不占普通建筑槽位。  
守炉堂 / 巡查所不参与普通建筑槽位竞争。  
守炉堂 / 巡查所不需要玩家手动 build。  
守炉堂 / 巡查所不需要施工期。  
守炉堂 / 巡查所不需要 D+1 天等待。  
  
路线入口签署后：  
  
誓言路线入口签署后：守炉堂自动生效。  
铁腕路线入口签署后：巡查所自动生效。  
  
不占槽不等于隐藏。  
自动生效不等于不显示。  
  
⸻  
  
**3. 006C 核心建筑运行人数**  
  
守炉堂 / 巡查所运行人数按后置口径：  
  
守炉堂 / 巡查所至少 1 名可工作人口值守即可运行。  
普通儿童不可作为工作人员。  
儿童辅工 / 学徒不能独立运行。  
若使用儿童辅工 / 学徒辅助，至少需要成人兜底。  
  
**作废旧口径**  
  
守炉堂 / 巡查所最低运行人数固定 3 人。  
守炉堂 / 巡查所必须 3 人才能运行。  
  
⸻  
  
**4. 006C 主动行动边界**  
  
006C 路线入口签署后，核心建筑立即生效并显示在面板上。  
  
但主动行动仍需满足：  
  
对应路线已开放  
对应行动已由炉律或路线解锁  
核心建筑处于生效状态  
行动不在冷却中  
满足行动目标条件  
  
核心建筑自动生效 ≠ 主动行动自动执行。  
  
主动行动必须由玩家手动使用。  
  
⸻  
  
**5. 006B / 006C 面板共同原则**  
  
面板必须区分：  
  
解锁资格  
已建成 / 已生效  
可执行行动  
冷却状态  
占槽状态  
是否可 heat  
数值是否待确认  
  
不要把：  
  
科技已完成  
  
误显示为：  
  
建筑已自动建好  
  
不要把：  
  
路线入口已签署  
  
误显示为：  
  
主动行动已自动执行  
  
不要把：  
  
不占槽  
  
误显示为：  
  
不存在 / 不显示  
  
不要把：  
  
未封存数值  
  
显示成正式成本。  
  
不要把：  
  
边缘区  
  
显示成新增代码区域字段。  
  
正确处理是：  
  
代码区域：外环  
玩家显示别名：边缘区

**10. 额外医疗配给数值身份**  
  
额外医疗配给只消耗熟食，不新增医疗物资字段。

已登记口径  
待后续平衡表复核  
未完全封存  
不得进入代码窗最终实现清单

不得改写这些数值。

如果当前交付要求是“只交已完全封存数值”，则这组数值进入“待复核 / 待确认”区，不进入最终代码实现清单。  
如果后续用户明确确认 006A-5 医疗行动数值可进入实现，再按这组数值执行。

**13. 本覆盖版结论**  
  
本覆盖版确定：
