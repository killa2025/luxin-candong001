# 《炉心残冬》后续数值表全局修正：升级即时完成与森林区 / 伐木场命名边界  
本修正覆盖此前关于 upgrade 耗时、森林区与伐木场命名的旧口径。  
   
⸻  
    
## 一、upgrade 统一口径  
后续所有建筑升级统一采用即时完成。  
规则：  
```
upgrade 为即时操作。
玩家支付升级所需资源后，建筑立即升级完成。
升级不进入 end_day 队列。
升级不需要等待到日结末尾。
升级不产生“升级中”状态。
升级不需要临时迁出居民。
升级不临时停用建筑。

```
适用范围：  
```
基础住宅 → 改良住宅
改良住宅 → 高级住宅
温室 → 改良温室
后续所有已确认为 upgrade 的建筑升级

```
保留规则：  
```
upgrade 仍需要前置科技。
upgrade 仍需要支付木材 / 钢铁等资源。
科技只解锁 upgrade 资格，不自动升级建筑。
upgrade 仍需玩家手动执行。

```
作废旧口径：  
```
upgrade 需要 1 天。
upgrade 完成在 end_day 末尾提交。
升级期间建筑维持基础功能。
升级期间建筑停用。
住宅升级需要临时迁出居民。
温室升级期间停产。

```
   
⸻  
   
## 二、森林区 / 伐木场统一口径  
后续统一区分：  
```
森林区 = 地图资源点 / 大型木材资源锚点。
伐木场 = 科技解锁的木材生产建筑。

```
森林区规则：  
```
森林区是地图开局固定的大型木材资源点。
森林区对标大型煤矿点 / 大型钢铁矿点。
森林区本身不是建筑。
森林区本身没有建造成本。
森林区本身不占普通建筑槽。
森林区数量由地图决定。
当前暂定地图中有 2 座固定森林区。

```
伐木场规则：  
```
伐木场是科技树解锁建筑。
伐木场用于开发森林区。
伐木场应绑定森林区。
伐木场数量受森林区数量限制。
伐木场才有建造成本、岗位、产出、运行温度、停工规则。

```
科技接线：  
```
木材加工 I：解锁伐木场 / 伐木场基础生产接线。
木材加工 II：强化伐木场产出。

```
作废旧口径：  
```
删除伐木场建筑。
森林区替代伐木场建筑。
森林区本身有建筑成本。
森林区本身占普通建筑槽。
森林区本身安排工人生产。
木材加工 I 直接让森林区产木。
木材加工 II 直接强化森林区本体。

```

## 四、代码窗不得做  
```
不得为 upgrade 创建 day queue。
不得让 upgrade 等到 end_day 才完成。
不得创建升级中状态，除非后续另开高级难度规则。
不得因住宅升级临时制造无住所人口。
不得因温室升级临时停产。

不得删除伐木场建筑。
不得把森林区当成伐木场建筑。
不得给森林区本体设置建筑成本。
不得给森林区本体设置普通建筑槽位。
不得给森林区本体安排岗位。
不得让木材加工 I / II 直接作用于森林区本体。

```
