## 一、总收口用途

代码窗看本文件时，应明确：  
```
哪些可以直接实现。

哪些必须留给测试窗复核。
哪些不得进入代码窗。
哪些是旧口径作废。

```
测试窗看本文件时，应明确：  
```

哪些值不是最终封存。
哪些系统必须做压力测试。
哪些测试结果会决定后续是否调整数值。

```
本文件不替代七批补表正文。 七批补表仍是详细实现来源。 本文件只做代码窗 / 测试窗交付总控。

## 六、测试窗交付原则  
测试窗重点不是“查有没有实现”，而是测试暂定数值是否合理。  
测试窗应重点判断：  
```
数值是否导致前期死局。
数值是否导致无脑最优解。
数值是否让关键路线没人走。
数值是否让第七霜落失去压力。
数值是否让某个系统完全替代另一个系统。
数值是否与封存硬失败口径冲突。

```
测试后可输出三类结论：  
```
保留：数值可封存。
调整：给出建议改值与理由。
作废：该暂定值导致系统问题，不建议继续使用。

```
