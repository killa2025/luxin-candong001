版本定位：干净全局解锁索引表 适用对象：代码窗、测试窗、后续合并窗 处理方式：只做索引收口，不重写 Patch 正文，不新增机制，不补未确认 TODO 不修改对象：全局修正总控、A 干净封存版、无 A 版时对应的干净封存版

## 一、索引用途  
本文件用于回答三类问题：  
```
1. 某个建筑由什么解锁？
2. 某个动作 / 命令由什么解锁？
3. 某个模式 / 路线 / 终局接口由什么解锁？

```
代码窗使用本文件时，应按“解锁关系”接线，不得自行猜：  
```
建筑前置
动作前置
模式前置
科技前置
炉律前置
路线互斥
是否开局默认可用
是否需要 build
是否占槽
是否可 heat
是否为暂定测试值

```
   
⸻  
   
## 二、来源优先级  
若索引表与旧补表、旧小修、旧封存正文冲突，按以下优先级处理：  
```
全局修正总控
>
A 干净封存版
>
无 A 版时对应的干净封存版
>
用户后续明确确认覆盖项
>
七批干净数值补表
>
旧小修 / 覆盖规则 / 返工清单 / 旧补表

```
说明：  
```
已封存正文原则上不回改。
用户后续明确确认的新口径，不强行回改旧封存正文，而是在本索引表和干净总收口中作为最新执行口径。

```
   
⸻  
   
## 三、状态说明  
本表使用四种状态。  
```
已封存 / 可直接实现：
来源明确、无冲突，可交代码窗实现。

用户确认覆盖项：
旧封存正文中未确认或旧口径不同，但用户后续明确确认，作为最新执行口径。

代码窗暂定实现值 / 测试窗必测：
代码窗可先写入骨架或配置表，但不是最终平衡封存。
测试窗试玩后再决定保留、调整或封存。

后置 / 待确认 / 不进代码窗：
当前不能实现，代码窗不得猜。

```
