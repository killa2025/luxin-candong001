## 二、来源与覆盖优先级  
六轮表中的“来源优先级”和“实现优先级”需要分开理解。  
**2.1 来源身份与覆盖口径来源**  
用于判断某项内容的来源身份，以及它是否属于最新覆盖口径：  
1. 全局修正总控明确引用的最新口径。  
2. A 干净封存版。  
3. 没有 A 版时，对应干净封存版。  
4. 七批干净数值补表。  
5. 旧小修、旧补表、旧返工记录。  
**全局修正总控主要用于确认最新执行口径；除非其中明确给出玩家可见文案，否则不替代 A 干净封存版作为正文原文来源。**  
**2.2 实际代码执行优先级**  
当旧封存原文与后续正式覆盖口径冲突时：  
1. 全局修正总控。  
2. 全局小修覆盖版。  
3. 已明确记录为“用户确认覆盖项”的后续口径。  
4. A 干净封存版。  
5. 无 A 版时的干净封存版。  
6. 七批干净数值补表。  
7. 旧小修、旧补表、旧口径。  
处理方式：  
* 不必反复回改旧封存原文。  
* 文案表中保留其来源身份。  
* 代码和运行时必须读取最新覆盖口径。  
* 用户确认覆盖项不能被旧 A 版再次覆盖回去。  
* 代码窗暂定实现值 / 测试窗必测不能伪装为已封存。  
   
⸻  
   
## 三、六轮统一导入边界  
**3.1 进入正式 TextRegistry**  
仅导入：  
* 玩家可见  
* 通用  
同时必须满足：  
* 状态为 已封存原文；或  
* 状态为 用户确认覆盖项；或  
* 明确允许作为暂定运行时内容，并由代码字段标记为测试值。  
**3.2 不进入正式 TextRegistry**  
以下内容默认不进入玩家 runtime 文案库：  
* 系统内部  
* TODO_TEXT  
* TODO_VALUE  
* TODO_SYSTEM  
* 待确认  
* 待判定  
* 后置  
* 作废旧口径  
* Deprecated 区  
* TODO 汇总索引  
* 内部数值说明  
* 内部公式  
* 内部评分阈值  
* 文档索引键  
* 使用 xxx 的模式占位 text_id  
系统内部条目可以进入：  
* 配置文档  
* 开发注释  
* 测试断言  
* 规则索引  
* PendingRegistry  
但不能当成玩家文案直接显示。  
**3.3 可见性枚举统一**  
六轮正式可见性只保留四种：  
* 玩家可见  
* 通用  
* 系统内部  
* 待判定  
第 6 轮中的：  
```
玩家可见（运行时模板）

```
统一改为：  
```
玩家可见

```
并通过“文案类型 / 备注”标记它是运行时模板，不能新增第五种可见性枚举。  
涉及：  
* ending.report.template  
* ending.report.death_record_sentence  
**可见性为“玩家可见”只表示该 text_id 未来属于玩家可见类型；若状态仍为 TODO_TEXT、待确认或未封存，则不得进入正式 TextRegistry，只能进入 PendingRegistry。**

## 七、文案池导入规则  
第 6 轮存在大量：  
* body_pool  
* closing_pool  
* additional.*.pool  
* interrogation.*.pool  
* report.*.pool  
这些条目不是“一整段要全部显示的长文案”。  
**7.1 禁止行为**  
代码窗不得：  
* 把整行文案池从头到尾一次性显示。  
* 仅按双换行机械切分候选。  
* 不看日期、死亡、路线、医疗和社会标签直接随机。  
* 选择会声称本局发生了实际未发生事件的段落。  
**7.2 正确处理**  
文案池必须拆为候选数组，每个候选至少保留：  
* candidate_id  
* text  
* required_tags  
* forbidden_tags  
* day_min / day_max，如需要  
* min_deaths / max_deaths，如需要  
* route_requirements，如需要  
当前六轮只完成了文案池正文和一部分筛选备注，**尚未完成每个候选段落的完整条件标注**。  
因此：  
文案池正文可以交代码窗保存，但自动随机选择功能暂时属于 PENDING_UI / PENDING_RULE；不能直接全池随机。  
硬失败文案池已经补了第七霜落日期限制，但其他结局池同样必须避免事实错误，例如：  
* 总死亡为 0 时，不能选择“有人死去”的胜利段落。  
* 没有病床溢出时，不能选择“太多人没能等到床位”。  
* 没有长期过载时，不能选择“过载让城市多活了一夜”。  
* 没有生食经历时，不能选择生食寒意相关句。  
* 没有儿童死亡时，不能生成儿童死亡审判。

| ---------- | ------------------------------------ |
| 当前封存正文     | 保留，不回改                               |

| 后续处理       | 需要拆成儿童保护所通用句、医疗学徒句、工程学徒句，或由 009-C 补齐 |
| 当前缺口       | TODO_TEXT / 009-C                    |
  
在新文案封存前，代码窗可选择：  
* 不显示该摘要；或  
* 仅在实际状态完全符合正文时显示。  
由于两种学徒路线互斥，现有正文原则上没有完整可达条件，因此建议暂不显示。
